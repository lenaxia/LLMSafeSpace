## Key Decisions

- **Claim-dispatch-release over record-then-dispatch** for webhook idempotency. The validator (C2) proved the record-then-dispatch order permanently stranded events on transient dispatch failure. The release-on-failure pattern preserves idempotency (prior successes return "duplicate") while allowing retry after transient failures.
- **Migration backfill existing orgs to active**. The DEFAULT 'pending_activation' would have locked out every pre-existing org member the moment the migration landed (validator H1). Backfill is one-shot (documented; migrations run exactly once).
- **Consolidated verifyOrgAdmin into verifyOwner**. Post-D6 they were functionally identical (owner OR org-admin). Kept verifyOwner as the single canonical access check; removed the duplicate (Rule 5).
- **Aligned secrets adapter to D6**. The workspaceOwnerVerifierAdapter used IsOrgMember, creating a security inconsistency between the workspace and secrets surfaces for the same workspace (validator H2).
