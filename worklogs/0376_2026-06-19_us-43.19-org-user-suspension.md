**Date:** 2026-06-19

## Objective

D19/D20: when an org or user is suspended, hard-operational suspension kicks in ‚Äî kill all pods (data preserved via PVCs), block API access, stop compute/token metering, keep storage metering. The auth middleware is the single gate that blocks a suspended user from every authenticated endpoint.

## Work Completed

### Migration 000037: `users.status` column
- `ALTER TABLE users ADD COLUMN status TEXT NOT NULL DEFAULT 'active' CHECK (status IN ('active', 'suspended'))`
- Backfill: `UPDATE users SET status = CASE WHEN active THEN 'active' ELSE 'suspended' END`
- Partial index `idx_users_status WHERE status != 'active'`
- Mirrored in `charts/llmsafespace/migrations/`

### Go types + DB queries
- `pkg/types/auth.go`: `UserStatus` type + `UserStatusActive`/`UserStatusSuspended` constants, `Status` field on `User`, `Status *UserStatus` on `UserUpdates`, `UserListEntry` DTO.
- `database.go`: all user SELECTs read `status`; `UpdateUser` accepts `Status`; new `SetUserStatus(ctx, userID, status)`.

### Auth middleware suspension gate
- `auth.go:947-956`: `AuthMiddleware` fetches user, checks `Status == suspended` ‚Üí 401.
- `Login` and `OptionalAuthMiddleware` also check `status`.

### Suspend/unsuspend endpoints (PlatformAdminHandler)
- `POST /api/v1/admin/orgs/:id/{suspend,unsuspend}` ‚Äî calls `UpdateOrgStatus`.
- `POST /api/v1/admin/users/:id/{suspend,unsuspend}` ‚Äî calls `SetUserStatus`. Last-admin deadlock prevention via `OrgsWhereUserIsLastActiveAdmin`. `?force=true` override. Audit-logged.
- All behind `AdminGuard()`.

### Controller org-status coupling (D20)
- `GET /api/v1/internal/orgs/:orgID/status` ‚Äî internal endpoint, gated by `X-Internal-Token` header.
- `controller/internal/workspace/org_status_client.go` ‚Äî 30s TTL cache, per-org mutex dedup, serve-stale-on-refresh-failure, fail-open on cold-cache+API-down.
- `controller/internal/workspace/org_suspend.go` ‚Äî `applyOrgSuspension` transitions Active‚ÜíSuspending when org is suspended; never auto-resumes.
- `--api-service-url` flag added to `controller/main.go`.

## Key Decisions

1. **`users.active` retained alongside `users.status`** ‚Äî backward compat for existing readers; `status` is authoritative. **THIS IS NOW TECH DEBT (F6)** ‚Äî see Blockers.
2. **Fail-open on cold-cache+API-down** ‚Äî chosen to avoid unwarranted pod suspension during API outages. **THIS IS WRONG FOR SECURITY SUSPENSIONS (F1, F5)** ‚Äî see Blockers.
3. **Internal endpoint uses opt-in `X-Internal-Token`** ‚Äî modeled after `/metrics`. **THE CLAIMED NetworkPolicy DOES NOT EXIST (F5)** ‚Äî see Blockers.

---

## Blockers

**5 findings from worklog 0372 must be fixed before this story is genuinely complete:**

- **F1 (CRITICAL):** Org-suspension is dead code in production. `--api-service-url` defaults to `""`; the Helm chart never passes it; `OrgStatusClient` is nil ‚Üí `applyOrgSuspension` no-ops. D20's "kill all pods" never happens. **Fix: wire the flag + token in `controller-deployment.yaml`.**
- **F3 (HIGH):** Auth middleware suspension check fails open on DB error. `GetUser` returning any error skips the check; request proceeds. **Fix: fail closed ‚Äî abort 401/503 on `GetUser` error.**
- **F4 (HIGH):** No token/API-key revocation on suspend. Existing JWTs remain valid; enforcement depends solely on the fail-open per-request check. **Fix: revoke active sessions on `SuspendUser`.**
- **F5 (HIGH):** Internal endpoint is unauthenticated by default (token opt-in). The handler comment falsely claims a NetworkPolicy boundary ‚Äî none exists in the chart. **Fix: invert default (require token), add `api-network-policy.yaml`, use `subtle.ConstantTimeCompare`.**
- **F7 (MEDIUM):** Last-admin check is a TOCTOU race ‚Äî SELECT then UPDATE with no transaction/`FOR UPDATE`. **Fix: wrap in a transaction with row locking.**
- **F6 (MEDIUM, latent):** `users.active`/`status` desync ‚Äî `SetUserStatus` deliberately doesn't touch `active`. Latent today; future divergence vector. **Fix: stop writing `active` independently or have `SetUserStatus` mirror.**
