## Objective

Workspace config (MaxActiveSessions + AutoApprovePermissions) is fetched from the Workspace CRD on first access and cached. Moving to Redis ensures all replicas share the same config view.

## Work Completed

- Overrode `GetWorkspaceConfig` → Redis `GET` + JSON unmarshal, fail-through to CRD on error
- Overrode `SetWorkspaceConfig` → Redis `SET` with 5min TTL, JSON-serialized Config
- Overrode `InvalidateWorkspaceConfig` → Redis `DEL`
- `DefaultConfigTTL = 5*time.Minute` (shorter than password — config changes via CRD updates)
- Updated `InvalidateAll` and type doc to reflect wsConfig is now Redis-backed
- Removed stale `DelegatesWorkspaceConfigToInMemory` test
