# Worklog: US-45.2 — Redis-backed activeSess

**Date:** 2026-06-17
**Session:** Implement US-45.2 (move active-session tracking from per-replica in-memory maps to a shared Redis SET, eliminating multi-replica state drift — the root cause of the 2026-06-16 stuck-session incident).
**Status:** Complete

---

## Objective

Make active-session tracking correct under multi-replica API deployments by externalizing the activeSess state from each replica's process-local map to a shared Redis SET. Uses Lua scripts for atomic check-and-add and remove-and-delete-if-empty. This is the core fix for the bug class documented in `STUCK-SESSIONS-RECOVERY.md`.

## Key Decisions

1. **Embed `*InMemoryStore` as private field, not anonymous embedding.** Validator flagged this as the safer choice: anonymous embedding would promote all InMemoryStore methods, creating a subtle shadowing trap if anyone removed an explicit Redis override. Explicit delegation methods make the migration path auditable.

2. **Package-level metrics with `sync.Once`.** Per-store metrics would panic on duplicate registration (Prometheus default registry rejects duplicates). Tests construct multiple stores; the package-level pattern matches `metrics.Service` precedent.

3. **Fail-open for writes, fail-closed for reads.** CheckAndAdd returns true on Redis error (allow the request — better than blocking legit traffic). Reads return false/0/nil (don't trap users in 409 based on possibly-stale state). Asymmetric but intentional; documented in code and tests.

4. **30-min TTL auto-recovery.** Stuck entries (process crash, network partition) automatically expire. The 2026-06-16 incident involved sessions stuck indefinitely; the TTL is the architectural guarantee this class cannot recur.

5. **Hash-tagged key `ws:{workspace_id}:active`.** The `{workspace_id}` braces force all keys for a workspace to land on the same Redis shard — enables future cluster-mode migration with zero code change. Zero cost today (single-shard Redis).

6. **Forward-fix production wiring (no feature flag).** Per design migration strategy: `app.go` auto-wires RedisStore when cache service is available. Once merged, production is permanently on Redis for activeSess. Rollback is via fix-forward PR, not env-var toggle.
