## Findings Addressed

### BUG 1 (P0, Critical): AsyncAuditLogger didn't implement CredentialStore
- **Root cause:** `PrepareSecretsForInjection` type-asserts `s.store.(CredentialStore)`. In production `s.store` is `*AsyncAuditLogger` which only implemented `SecretStore`. Assertion always failed → legacy fallback → new path dead.
- **Fix:** Added 4 delegation methods to `AsyncAuditLogger`: `GetWorkspaceCredentials`, `UpsertFreeTierCredential`, `SeedWorkspaceCredentials`, `HasUserProviderCredential`.
- **Test:** `TestAsyncAuditLogger_ImplementsCredentialStore` + `TestPrepareSecretsForInjection_ViaAsyncAuditLogger` prove the path activates.

### BUG 2 (P1): No startup backfill for existing workspaces
- **Root cause:** `ensureFreeTierCredential` only upserted the credential row + auto-apply rule but never backfilled `workspace_credential_bindings` for existing workspaces.
- **Fix:** Added `BackfillFreeTierBindings()` on `PgSecretStore` (single SQL INSERT...SELECT with ON CONFLICT DO NOTHING). Called after successful `UpsertFreeTierCredential` at startup. Returns count of rows inserted for logging.

### BUG 3 (P0, Security): No ownership verification in Bind/Unbind
- **Root cause:** Any authenticated user could bind any credential to any workspace by guessing UUIDs.
- **Fix:** Added `WorkspaceOwnerChecker` func type + `CredentialStateWriter` interface. Bind/Unbind now verify: (1) user owns the credential via `GetUserCredential(ctx, userID, credID)`, (2) user owns the workspace via `wsOwnerCheck`. Wired with `workspaceOwnerVerifierAdapter` in app.go.

### Input Validation
- Provider name: trimmed, rejected if empty after trim.
- Name: trimmed.
- Applied to both admin and user Create handlers.

### Duplicate Constraint → 409
- Added `isDuplicateErr` helper that checks for PostgreSQL 23505.
- Both admin and user Create handlers now return 409 Conflict instead of 500 on duplicate provider.

### MarkCredentialChanged (Epic 27a integration)
- Bind and Unbind now call `CredentialStateWriter.MarkCredentialChanged` after successful operation.
- Wired with `database.Service` which implements `MarkCredentialChanged`.
