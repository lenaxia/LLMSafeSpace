## Key Decisions

1. **Split Phase 1 into two commits, not one.** The epic plan said ship
   US-46.1/.2/.14/.15 in a single PR. US-46.2 changes controller production code
   (driver wiring) and adds a regression test; US-46.14/.15 are pure
   doc/file-moves. Keeping the code change in its own commit makes review and
   any revert cheaper. Rationale: lower blast radius, clearer git history.

2. **Renumbered epic-43-relay-admin-ux to epic-48, not epic-47.** The US-46.1
   story text proposed `epic-47`, but `epic-47-frontend-architecture-consolidation`
   was created after the story was written. The next genuinely free number is
   48. Rationale: the story's suggestion was based on stale data; uniqueness is
   the actual requirement.

3. **Kept internal `US-38.x` / `US-43.x` story IDs inside the renamed READMEs.**
   Those IDs are referenced by append-only worklogs (e.g. 0341, 0343) and by the
   story-breakdown tables inside each README. Renumbering every story ID would
   break those references and violate the "append-only history" rule. The
   directory number is the canonical handle; internal story IDs are historical
   labels. Rationale: consistency over cosmetic tidiness.

4. **Removed `ErrNotImplemented` entirely, not just from the stub.** Rule 5
   (zero tech debt) — it had exactly one consumer (the deleted stub) and its test.
   Leaving a dead sentinel would itself be tech debt. Rationale: the only
   legitimate use of `ErrNotImplemented` would be a future stub, and a future
   stub is exactly what US-46.2 exists to prevent.

5. **Did not renumber or rewrite the pre-existing worklog collision mess on
   `origin/main`.** Per explicit user instruction ("leave it, work around").
   `origin/main` HEAD currently has three duplicate `us-46.2-keep-annotatemodels-guard`
   worklogs at 0347/0348/0349 (a separate autofix issue). This worklog is added
   at 0350 (next free on origin/main) and stages only Epic 46 files; the dirty
   worklog churn in the working tree is left untouched.
