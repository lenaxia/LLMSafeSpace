## Objective

Migrate the last two per-workspace state sections from InMemoryStore to Redis, completing the Epic 45 state externalization. After this PR, the embedded InMemoryStore serves no production state — it's retained only for defense-in-depth in InvalidateAll until US-45.9 removes it.

## Work Completed

### US-45.7: priorPhase
- `GetPriorPhase` → Redis `GET`, fail-through on error (returns false → treated as first-invocation)
- `SetPriorPhase` → Redis `SET` with 24h TTL
- `DeletePriorPhase` → Redis `DEL`
- **priorPhase survives InvalidateAll** per US-45.1 contract — onPhaseChange relies on it to distinguish first-invocation from Active→Active reconcile. Verified by `TestRedisStore_InvalidateAll_ClearsRedisPriorPhase`.
- **Deviation from spec**: spec said Redis HASH with `{prior, current, transitionedAt}` + Lua compare-and-swap. The existing code only uses prior phase (not current or transitionedAt), and the watcher serializes events per-resource — no compare-and-swap needed. STRING key is simpler and correct.

### US-45.8: parentBackfilled
- `GetParentBackfilled` → Redis `EXISTS`
- `SetParentBackfilled` → Redis `SET` with 24h TTL
- `DeleteParentBackfilled` → Redis `DEL`
- **Deviation from spec**: spec said SETNX (atomic claim). The existing code pattern is check-then-set (GetParentBackfilled then SetParentBackfilled inside the same function), not atomic claim — the watcher serializes per-resource, so SETNX is unnecessary. `SET` is simpler and matches the existing semantics. If a future change adds concurrent backfill callers, SETNX would be the correct upgrade.

### All Redis constants now defined:

- `DefaultDeletedTTL` = 30min (US-45.3)
- `DefaultPasswordTTL` = 1h (US-45.4)
- `DefaultConfigTTL` = 5min (US-45.6)
- `DefaultPriorPhaseTTL` = 24h (US-45.7)
- `DefaultBackfilledTTL` = 24h (US-45.8)
