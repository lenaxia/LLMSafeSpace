# Worklog: Epic 46 Phase 2 — US-46.5 Propagate Context Everywhere

**Date:** 2026-06-19
**Session:** Audit and fix context.TODO/context.Background propagation in production code
**Status:** Complete

---

## Objective

Implement US-46.5: eliminate `context.TODO()` from production code and fix
illegitimate `context.Background()` usage that drops request deadlines.

---

## Work Completed

### Audit (Rule 7: validate before acting)

The spec claimed **187 occurrences** of `context.TODO()` + `context.Background()`.
Actual reality after merge of prior stories:

| Metric | Spec claim | Actual |
|--------|-----------|--------|
| `context.TODO()` | part of 187 | **0** (already eliminated) |
| `context.Background()` | part of 187 | **188** |
| Total | 187 | **188** |

Of the 188 `context.Background()` sites:
- **96** in `sdks/canary/` — all legitimate (canary test programs with `func main()`)
- **91** in production code (`api/`, `pkg/`, `controller/`, `cmd/`)

Thorough classification of the 91 production sites (via sub-agent audit):

| Classification | Count | Description |
|---------------|-------|-------------|
| **ILLEGITIMATE** | **3** | Local ctx available but unused for a sub-call |
| **ILLEGITIMATE (ambiguous)** | **1** | Fire-and-forget intent; documented as legitimate |
| LEGITIMATE-BG | 37 | main/init/startup/shutdown/background goroutine |
| OUT-OF-SCOPE | 50 | Function doesn't take ctx; adding it is a signature change (spec excludes) |

### Fixes applied (3 illegitimate sites)

All in `api/internal/handlers/proxy_session_index.go`:

| Line | Function | Before | After |
|------|----------|--------|-------|
| 62 | `fetchAndPersistTitle` | `context.Background()` | `ctx` (local, line 24) |
| 67 | `fetchAndPersistTitle` | `context.Background()` | `ctx` (local, line 24) |
| 139 | `runParentBackfill` | `context.Background()` | `ctx` (local, line 86) |

Each function creates a `ctx` via `context.WithTimeout(context.Background(), ...)`
at the top, uses it for the upstream pod fetch, then silently switched to
`context.Background()` for the DB write — dropping the timeout deadline. Now
all calls in each function share the same ctx.

### Ambiguous site documented (1 site)

`proxy_handlers.go:250` (`DeleteSession`): uses `context.Background()` for
`sessionIndex.DeleteSession`. The intent is fire-and-forget — the agent has
already deleted the session; the index delete must complete even if the client
disconnects, to avoid an inconsistent state. Added a comment + `//nolint:contextcheck`
matching the established pattern in `agent_reload.go:536`.

### Lint enforcement added

Added a `context.TODO check` step to the CI Lint job (`.github/workflows/ci.yml`)
that greps for `context.TODO()` in production code and fails if any are found.
This prevents regressions — `context.TODO()` is never correct in shipped code
(it literally means "I don't know what context belongs here").

`contextcheck` linter remains enabled in `.golangci.yml` for broader
`context.Background()` detection (it has a known blind spot for the pattern
fixed in this story — a function that creates its own root ctx via
`WithTimeout(Background())` then switches to `Background()` for a sub-call).

## Key Decisions

1. **`context.TODO()` acceptance criterion already met.** Zero occurrences
   existed at branch time (cleaned up in prior merges). The spec's 187 count
   was stale. Rationale: validate before acting (Rule 7).

2. **Only 3 sites needed fixing, not 187.** The audit sub-agent classified
   every site: 37 legitimate (startup/shutdown/background goroutines),
   50 out-of-scope (functions without ctx param — spec explicitly excludes
   signature changes). Rationale: the spec's "187" conflated TODO + Background
   + test code + SDK canary programs.

3. **Documented the ambiguous DeleteSession site rather than "fixing" it.**
   The audit flagged it as illegitimate because the gin context is available,
   but the INTENT is to survive client disconnect (the agent already deleted
   the session; the index must follow through). Changing to `c.Request.Context()`
   would introduce a regression where client disconnect leaves the index
   inconsistent. Rationale: preserve current behavior; document intent.

4. **Lint check targets `context.TODO()` specifically.** A blanket
   `context.Background()` ban is impossible — too many legitimate uses
   (main, init, startup, shutdown, background goroutines). `context.TODO()`
   is never correct in shipped code, so a zero-tolerance grep is feasible.
   Rationale: enforce what's enforceable; leave Background to code review.
