# Worklog: Epic 30 US-30.1 — Schema + Legacy Credential Deletion

**Date:** 2026-06-05
**Session:** Implement US-30.1 — new unified credential model schema, delete `pkg/credentials`, drop `credential_sets`
**Status:** Complete

---

## Objective

Implement the schema migration and code deletion for Epic 30's unified credential model (US-30.1). This creates `provider_credentials`, `workspace_credential_bindings`, `credential_auto_apply`, and `credential_backfill_jobs` tables while dropping the legacy `credential_sets` table and deleting the entire `pkg/credentials` package.

---

## Work Completed

### Migration 000015_unified_credential_model

- Created `api/migrations/000015_unified_credential_model.up.sql`:
  - `DROP TABLE IF EXISTS credential_sets` — verified no FK references (A3)
  - `CREATE TABLE provider_credentials` with UNIQUE(owner_type, owner_id, provider)
  - `CREATE TABLE workspace_credential_bindings` with FK cascades to both provider_credentials and workspaces
  - `CREATE TABLE credential_auto_apply` with partial unique indexes for NULL-safe uniqueness
  - `CREATE TABLE credential_backfill_jobs` for async backfill state
- Created matching down migration

### Legacy Code Deletion

- Deleted `pkg/credentials/` entirely (service.go, types.go, crypto.go, all tests)
- Deleted `api/internal/handlers/credentials.go` + tests
- Deleted `api/internal/services/database/credentials.go` + tests
- Removed `credKeySet`, `credSvc`, `credentialsHandler` wiring from `api/internal/app/app.go`
- Removed `loadCredentialKeySet()` function
- Removed unused imports (`crypto/rand`, `encoding/hex`, `os`)
- Removed `CredentialsHandler` field from `RouterConfig`
- Removed `registerCredentialRoutes` function from router
- Removed `/admin/credentials` routes from `sdks/openapi.yaml`
- Removed credential-set schemas from OpenAPI spec

### New CredentialStore Interface + Implementation

- Created `pkg/secrets/credential_store.go`: `CredentialStore` interface, `CredentialBinding` type, `AdminKeyDeriver` type
- Added `OwnerTypeAdmin` to existing `OwnerType` constants in `provider.go`
- Created `pkg/secrets/pg_credential_store.go`: implements `GetWorkspaceCredentials`, `UpsertFreeTierCredential`, `SeedWorkspaceCredentials`, `HasUserProviderCredential`
- Created integration test file `pkg/secrets/credential_store_integration_test.go` (5 test cases covering happy paths, idempotency, priority ordering, empty results)

### Fix Required by Repolint

- Ran `make chart-sync-migrations` to sync migration 000015 into `charts/llmsafespace/migrations/`
- Fixed OpenAPI contract test that referenced `CredentialsHandler`

## Key Decisions

1. **`OwnerTypeAdmin` added to existing `provider.go`** rather than declaring a new type to avoid redeclaration conflicts.
2. **Integration tests are `//go:build integration`** tagged — they require a live Postgres instance and the new schema applied. Unit tests continue to pass with `-short`.
3. **OpenAPI spec updated to remove legacy routes** — the contract test enforces spec↔router alignment, so removal was required for tests to pass.
