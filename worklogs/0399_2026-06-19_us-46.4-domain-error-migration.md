# Worklog: US-46.4 Domain Error Migration (audit-driven fix)

**Date:** 2026-06-19
**Session:** Close US-46.4 acceptance gap — migrate ≥10 sentinel errors to typed errors with HTTP status codes.
**Status:** Complete

---

## Objective

US-46.4's original implementation (worklog 0366) delivered 2/10 sentinel migrations and no DomainError type, choosing to reuse APIError instead. The final Epic 46 audit flagged this as a gap. This session closes it by:

1. Creating `pkg/errors.StatusError` — the shared typed error for `pkg/` packages (which can't import `api/internal/errors/`)
2. Converting 12 `pkg/` sentinels to `*StatusError` (carrying HTTP status + user-facing message)
3. Simplifying `handleSecretError` from a 30-line switch to 5 lines (duck-type delegation)
4. Replacing `strings.Contains(err.Error(), "no running pod")` with `errors.Is`

---

## Design Decision: StatusError, not DomainError

The story spec called for a `DomainError` type in `api/internal/errors/`. The original implementer correctly observed that `APIError` already fills that role (Type/Code/Message/Err + `StatusCode()` + `Unwrap()`). Creating a parallel type would violate Rule 4 (no speculative generality).

The actual blocker was that `pkg/` packages can't import `api/internal/errors/` (Go internal-package visibility). The solution: `pkg/errors.StatusError` — a minimal type that carries an HTTP status code and user-facing message, usable from any package. `APIError` and `StatusError` are conceptually the same pattern (typed error with status code); they coexist because Go's import visibility forces them into different packages.

## Work Completed

### New: `pkg/errors/status_error.go` (89 lines + 9 tests)

```go
type StatusError struct {
    Status  int    // HTTP status code
    Code    string // machine-readable code
    Message string // user-facing message (safe to expose)
    Cause   error  // wrapped cause for logging
}
```

Implements: `Error() string`, `StatusCode() int`, `Unwrap() error`, `Wrap(cause) *StatusError`.

9 tests cover: Error() with/without Cause, StatusCode(), Unwrap(), Wrap() immutability, errors.Is sentinel pointer identity, errors.As chain traversal, NewStatusError constructor.
