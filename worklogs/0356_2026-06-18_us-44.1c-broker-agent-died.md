# Worklog: US-44.1c — Broker-side agent_died Bridge

**Date:** 2026-06-18
**Session:** Implement US-44.1c — completes the user-facing goal of US-44.1 (terminal SSE events). The proxy already emitted `agent_died` on the proxied SSE body (US-44.1a), but the frontend listens on the broker stream — so users saw a silent close. US-44.1c bridges the broker so the frontend renders a dismissible "⚠ Agent was terminated" warning.
**Status:** Complete (implementation + skeptical validation to zero findings; pending PR review/merge)

## Objective

Implement US-44.1c from `design/stories/epic-44-session-reliability-transparency/README.md`. When opencode dies mid-SSE-stream, the SSE tracker fires `onAgentDied(workspaceID)` on stream-end-after-data; `ProxyHandler.onAgentDied` publishes `WorkspaceSSEEvent{Type:"agent_died", WorkspaceID, Data:{"reason":"unknown"}}` to the workspace broker; the frontend `ChatPage` renders a dismissible warning banner. This unblocks US-44.4's deferred frontend items (OOM terminal message).

Followed the README-LLM.md Multi-Agent Workflow: TDD implementation → independent skeptical validator (separate sub-agent) → remediation loop to zero findings → build/test/lint → commit/PR/worklog.

## Key Decisions

1. **Accept the false-positive tradeoff (death vs normal restart).** The tracker cannot distinguish a real opencode death (OOM/SIGKILL) from a normal in-place restart (secrets reload) — both end the SSE stream after data. `agent_died` fires on both. This is the same accepted tradeoff as US-44.1a (pinned by `TestProxy_US44_1_SSECleanClose_AcceptableFalsePositive`); mitigated by a dismissible banner and the durable `reconcileSessionState` recovery signal on the next successful reconnect.
2. **`scanner.Err()` guard aligns with US-44.1a.** Non-EOF read errors (TCP RST, `bufio.ErrTooLong`) after data do NOT fire `agent_died` — a network blip is not a death. A real death produces clean EOF (`scanner.Err()` returns nil) so genuine deaths still fire. The rare RST-on-death case is missed (documented, accepted).
3. **Transient delivery (no replay).** `PublishToWorkspace` does not buffer; a frontend reconnecting after `agent_died` will miss it. Acceptable for a dismissible banner — the workspace phase stream and session-status events carry the durable state, and `reconcileSessionState` clears stale active sessions on reconnect.
4. **Injectable `idleTimeout`** as a test-only knob (production uses the 5min const default). Enables deterministic coverage of the idle-timeout negative path without a production refactor or a 5-minute test wait.
5. **`bytesReceived` counts `len(scanner.Text())`** (newline-stripped) rather than raw bytes — diverges from US-44.1a's `resp.Body.Read` count but immaterial since only `> 0` is checked (documented).
