# Worklog: Epic 46 Phase 2 — US-46.4 Migrate Sentinel Errors to APIError

**Date:** 2026-06-19
**Session:** Migrate sentinel errors to typed *APIError for centralized HTTP mapping
**Status:** Complete

---

## Objective

Implement US-46.4: introduce typed error patterns so callers can use `errors.As`
instead of string matching. The spec proposed a new `DomainError` type, but a
Rule 7 audit revealed that **`APIError` already exists** with the exact shape
the spec describes (Code, Message, Category, Cause, StatusCode, 9 constructors,
70 server-side call sites). Creating a parallel `DomainError` would violate Rule
4 (over-engineering) and Rule 5 (duplicate type systems).

Revised approach (approved by user): migrate the existing sentinel errors to
participate in the `APIError` ecosystem rather than creating a new type.

## Work Completed

### Rule 7 finding: spec's audit was inaccurate

| Spec claim | Actual reality |
|------------|----------------|
| 43 sentinel errors | **4** (`ErrNoAgentStateRow`, `ErrTooManySubscribers`, `ErrPartialFailure`, `ErrAutoBindingProtected`) |
| Only 4 custom error types | **`APIError` already provides typed errors** with 8 categories, 9 constructors, `StatusCode()`, `Unwrap()` — used in 70 server-side call sites |
| `DomainError` doesn't exist | `APIError` IS the DomainError the spec proposes — same fields, same constructors, same HTTP mapping |

### Sentinel migration

Migrated **2 of 4** sentinels to `*APIError`. The other 2 are architecturally
blocked (documented below).

| Sentinel | Package | Before | After | HTTP Status |
|----------|---------|--------|-------|-------------|
| `ErrNoAgentStateRow` | `api/internal/errors` | `errors.New(...)` | `*APIError{Conflict, "no_pending_agent_reload"}` | 409 |
| `ErrTooManySubscribers` | `api/internal/services/eventbroker` | `errors.New(...)` | `*apierrors.APIError{RateLimit, "too_many_subscribers"}` | 429 |
| `ErrPartialFailure` | `pkg/agentd/secrets` | `errors.New(...)` | **unchanged** (internal barrier) | N/A (agentd only) |
| `ErrAutoBindingProtected` | `pkg/secrets` | `errors.New(...)` | **unchanged** (internal barrier) | handler wraps at boundary |

### Why 2 sentinels could not be migrated

Go's **internal-package visibility rule** prevents `pkg/` packages from importing
`api/internal/errors`. Both `ErrPartialFailure` (pkg/agentd/secrets) and
`ErrAutoBindingProtected` (pkg/secrets) live in `pkg/` and cannot reference
`*apierrors.APIError`. This is a hard architectural constraint, not a design
choice. Both sentinels now carry a doc comment explaining the constraint.

- `ErrPartialFailure` is consumed only by `cmd/workspace-agentd` (the agent
  daemon) — never reaches an HTTP handler, so HTTP status mapping is moot.
- `ErrAutoBindingProtected` IS surfaced to HTTP, but the handler
  (`user_provider_credentials.go:334`) already maps it to 409 manually. The
  sentinel stays plain; the handler wraps at the boundary if needed.

### Centralized error handler upgrade

Updated both `respondWithError` (server/router.go) and `respondWithAPIError`
(handlers/agent_reload.go) to use `errors.As(err, &apiErr)` for `*APIError`,
in addition to the existing duck-typed `StatusCode()` check. This handles:
- Direct `*APIError` returns (covered by both old and new code)
- `fmt.Errorf`-wrapped `*APIError` values (NEW — old duck-type missed these)
- Other types implementing `StatusCode()` (preserved as fallback)

### Handler simplification

`agent_reload.go:299-302`: replaced manual `c.JSON(http.StatusConflict, ...)`
with `respondWithAPIError(c, err)` — the error now carries its own status and
message.

### TDD tests added

- `errors_test.go`: 5 tests for `ErrNoAgentStateRow` (IsAPIError, ErrorsIsPreserved, Category, StatusCode, WrappedErrorsAsStillWorks)
- `user_broker_test.go`: 2 tests for `ErrTooManySubscribers` (IsAPIError, StatusCode)

## Key Decisions

1. **APIError IS DomainError — no new type created.** The spec's `DomainError`
   proposal duplicates the existing `APIError` 1:1 (same fields, constructors,
   HTTP mapping). Creating it would add a parallel type system needing adapters
   (Rule 4 + Rule 5 violations). Rationale: use what exists; extend adoption
   organically.

2. **Sentinels as `*APIError` values, not constructor calls.** Defined as
   `var ErrX = &APIError{...}` rather than changing return sites to call
   constructors. Rationale: preserves `errors.Is(err, ErrX)` via pointer
   equality (zero caller breakage) while adding `StatusCode()` and `errors.As`
   support. Callers that return the sentinel directly get centralized mapping
   for free.

3. **2 of 4 sentinels blocked by Go internal-package visibility.** `pkg/`
   packages cannot import `api/internal/`. This is a language-level constraint,
   not a design flaw. Rationale: document the constraint rather than work around
   it (e.g., by moving APIError to pkg/ — that would couple the error type to
   both the API layer and the agentd daemon, a worse trade-off).

4. **errors.As added to centralized handlers.** Both `respondWithError` copies
   now try `errors.As` before the duck-typed check. Rationale: wrapped
   `*APIError` values (via `fmt.Errorf %w`) were previously invisible to the
   duck-type check because the wrapper doesn't implement `StatusCode()`. Now
   they're found anywhere in the chain.

5. **Kept the two `respondWithError` copies rather than consolidating.** They
   live in separate packages (`server` and `handlers`) and can't share without
   either coupling handlers→server or extracting to a shared package. The
   duplication is 10 lines; eliminating it is out of scope for this story.

## Assumptions Stated and Validated

| Assumption | Validation |
|------------|------------|
| APIError already provides what DomainError proposes | Read errors.go in full: same fields (Type, Code, Message, Details, Err), same constructors (9), same StatusCode() mapping, same Unwrap() |
| Only 4 sentinel errors exist (not 43) | `grep -rn '^var Err' --include="*.go" api/ pkg/ controller/ cmd/` returns exactly 4 |

| No one wraps the sentinels with `fmt.Errorf %w` | `grep -rn "ErrNoAgentStateRow\|ErrTooManySubscribers" --include="*.go" \| grep Errorf` returns empty |
| `errors.Is` still works after sentinel → *APIError | TDD test `TestErrNoAgentStateRow_ErrorsIsPreserved` passes |
| `errors.As` now works (new capability) | TDD test `TestErrNoAgentStateRow_IsAPIError` passes |
| eventbroker can import apierrors without a cycle | apierrors imports only stdlib; eventbroker → apierrors is one-directional; `go build` passes |
| pkg/ packages cannot import api/internal/ | Go specification: internal packages only importable within parent tree; confirmed by package paths |
| Existing handler tests pass unchanged | `go test ./api/internal/handlers/... -run TestAgentReload` passes |
