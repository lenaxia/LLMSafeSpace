# Worklog 0298 — Epic 42: AWS EC2 Driver Implementation

**Date:** 2026-06-15
**Epic:** 42
**Story:** US-42.13
**Scope:** Replace AWS driver stub with real EC2 implementation

## Summary

Replaced the `ErrNotImplemented` stub AWS driver with a full EC2
implementation using the AWS SDK Go v2. AWS is the primary relay provider
(weight 1000, most reliable — no idle reclamation, no capacity issues).

## Changes

### `controller/internal/relay/aws_driver.go` (369 lines)
- `NewAWSDriver(k8sClient, namespace, credentialSecret)` constructor
- `loadAWSConfig`: reads accessKeyId/secretAccessKey from K8s Secret,
  falls back to default credential chain (IRSA, env, instance metadata)
- `Provision`: resolves latest Ubuntu ARM64 AMI via DescribeImages,
  calls RunInstances with cloud-init userdata, waits for running state
- `Destroy`: calls TerminateInstances
- `GetStatus`: calls DescribeInstances, maps EC2 state to VMState
- `ListInstances`: filters by tag `managed-by=llmsafespace-relay`
- `resolveAMI`: queries Canonical AMI catalog (owner 099720109477)
  for Ubuntu 22.04 ARM64 images, picks most recent
- `waitForRunning`: polls DescribeInstances with context cancellation
- Error classification: InsufficientInstanceCapacity/InstanceLimitExceeded/
  RequestLimitExceeded/Throttling → ErrCapacity;
  InvalidParameterValue/InvalidAMIID/UnauthorizedOperation/AccessDenied
  → ErrConfig

### `controller/internal/relay/aws_driver_test.go` (8 tests)
- State mapping tests (all 6 EC2 states)
- Error classification tests (capacity, config, generic, nil)
- Credential fallback test
- Constructor default/custom secret name tests

### `controller/internal/controller/controller.go`
- Wire `NewAWSDriver` with `aws-relay-irwa` credential secret
