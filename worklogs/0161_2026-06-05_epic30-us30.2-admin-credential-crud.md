## Work Completed

### Handler: `api/internal/handlers/admin_provider_credentials.go`
- `AdminProviderCredentialsHandler` with `Create`, `List`, `Get`, `Update`, `Delete` methods
- `AdminCredentialStore` interface (satisfied by `PgSecretStore`)
- Server-KEK nil check → 503 when master secret not configured
- BaseURL decrypted from ciphertext for GET/List responses (omitted if KEK unavailable)
- API key never returned in any response

### DB Layer: `pkg/secrets/pg_credential_store.go`
- `AdminCredentialRow` type (shared between handler and DB layer)
- `CreateAdminCredential`, `ListAdminCredentials`, `GetAdminCredential`, `UpdateAdminCredential`, `DeleteAdminCredential`
- All scoped to `owner_type='admin'` rows

### Wiring: `app.go` + `router.go`
- Handler wired in the secrets `{}` block using `pgStore` and `deriveServerKey`
- Routes registered with `AuthMiddleware()` + `AdminGuard()`

### Tests (TDD)
- 7 test cases: create success, create missing apiKey, create nil KEK, list, get not-found, delete, update success
- All use fake in-memory store with real `secrets.EncryptSecret` for ciphertext
