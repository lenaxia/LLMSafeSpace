  `sessionRevoker` interface in `app.go` — both `RevokeAllUserSessions(ctx, …)`.

- Live callers: `server/router.go` (`/auth/logout`) → `c.Request.Context()`;
  `handlers/password_reset.go` → its in-scope `ctx`.

### Tests

- New `TestRevokeToken_PropagatesContext` (TDD): sentinel ctx; RevokeToken
  writes both `token:<hash>` and `token:<jti>` via `Set` — the ctx matcher is
  load-bearing (a `context.Background()` regression leaves the sentinel
  absent, the Sets become unexpected, the mock fails).
- Existing auth/handlers/server/middleware-tests updated to pass ctx.

## Key Decisions

- **Mock ignores ctx in matching.** Preserves existing `.On("RevokeToken", token)`
  setups (ValidateToken / database-check mock pattern).
- **5s cap retained.** `context.WithTimeout(ctx, 5s)` keeps the per-call bound
  while gaining cancellation propagation.
