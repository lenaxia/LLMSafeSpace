# Worklog: US-33.1 — Fix cardinality bugs in API service metrics

**Date:** 2026-06-19
**Session:** Remove unbounded-cardinality / dead metrics from the API service metrics package per epic-33 design.
**Status:** Complete

---

## Objective

Implement US-33.1: drop `user_id` from `api_active_connections`; delete the API's `workspaces_created_total` and `workspace_resource_usage` metrics; update the `MetricsService` interface, the hand-maintained mock, all call sites, and tests.

---

## Work Completed

### 1. `api_active_connections` — dropped `user_id` label
- `api/internal/services/metrics/metrics.go`: gauge label set `[]string{"type", "user_id"}` → `[]string{"type"}`.
- `IncrementActiveConnections` / `DecrementActiveConnections` signatures `(connType, userID string)` → `(connType string)`.
- `api/internal/middleware/metrics.go:WebSocketMetricsMiddleware`: removed the now-dead `userID` extraction block (Rule 5) and updated both calls to the single-arg signature.

### 2. `workspaces_created_total` — deleted
- Removed the `workspacesCreated` struct field, the `promauto.NewCounterVec` registration, and the `RecordWorkspaceCreation` method. The controller's authoritative `llmsafespaces_workspaces_created_total{runtime, security_level}` is untouched (distinct name).

### 3. `workspace_resource_usage` — deleted
- Removed the `resourceUsage` struct field, the `promauto.NewGaugeVec` registration, and the `RecordResourceUsage` method.

### Interface + mock
- `api/internal/interfaces/interfaces.go`: `MetricsService` lost `RecordResourceUsage`; connection methods changed to single-arg. (`RecordWorkspaceCreation` was never on the interface — it had no callers at all.)
- `api/internal/mocks/metrics.go`: removed `RecordWorkspaceCreation` and `RecordResourceUsage` stubs; changed connection method stubs to single-arg.

### Call sites / tests updated
- `api/internal/middleware/tests/metrics_test.go`: `mockMetrics.On("IncrementActiveConnections", "chat", mock.Anything)` → `"chat"` only.
- Router test files (`.Maybe()` expectations trimmed to one `mock.Anything`): `router_admin_platform_list_test.go`, `router_workspace_access_test.go`, `router_workspace_test.go`, `router_frontend_workspace_test.go`, `router_admin_audit_test.go`.
- `api/internal/services/metrics/metrics_test.go`: removed `TestRecordResourceUsage`; dropped `workspacesCreated`/`resourceUsage` asserts from `TestNew_AllCountersInitialised`; updated `TestIncrementDecrementActiveConnections` to the new signature; **added** `TestActiveConnections_OnlyTypeLabel` which gathers from `prometheus.DefaultGatherer` and asserts exactly one label named `type` (regression guard for `user_id`).

### `RecordRelayInjector` — confirmed absent
Grep of `api/internal/services/metrics/` returns nothing for `RecordRelayInjector` / `relayInjectorTotal` (already removed in worklog 0231). Skipped per scope.

---

## Key Decisions

- **Removed the dead `userID` extraction in `WebSocketMetricsMiddleware`** rather than leaving it assigned-but-unused. Go would fail to compile on the unused var, and Rule 5 forbids carrying dead code forward. The value had no other consumer.
- **Did not modify historical docs/worklogs** that still reference the old metric names (`design/archive/`, `design/story2.1`, worklogs 0175/0231, epic-27b README). These are append-only historical records (Rule 7); the epic-33 README is the spec describing this very change.
- **No chart/dashboard/rule edits needed**: grep of `charts/` returned zero references to the API's `workspaces_created_total`, `workspace_resource_usage`, or `api_active_connections`.
- Kept changes minimal — did not refactor unrelated metrics.

---

## Assumptions (Rule 7) and validation

1. `RecordRelayInjector` is absent from API metrics → validated by grep (zero hits).
2. Controller's `llmsafespaces_workspaces_created_total` is the surviving authoritative metric → validated by distinct name; untouched (whole-repo `go build ./...` passed).
3. No chart references to the deleted/changed metrics → validated by `grep -rn ... charts/` (NONE).
4. `RecordWorkspaceCreation` / `RecordResourceUsage` have no production callers → validated by grep (only definitions + interface/mock/test); clean deletion.
5. Mock is hand-maintained (no `go:generate`) → validated by grep; hand-edited.
6. `promauto` would panic on duplicate registration in `New()` → `TestMain` calls `New()` and the suite passes, proving no duplicate registrations remain.
