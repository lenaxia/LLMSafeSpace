```json
        {"label": "Rust", "description": "Ideal for performance-critical systems, CLIs, and WebAssembly"},

    "callID": "call_00_L2Vxenr4keDzvhpYXz9J6233"
```

**Key observations:**
- `id` format: `que_` prefix + opaque string
- `sessionID` format: `ses_` prefix + opaque string
- `questions` is an array (supports multiple questions per request)
- Each question: `question` (string), `header` (string), `options` (array of `{label, description}`)
- **`custom` field is ABSENT** â€” not present at all
- **`tool` field IS present** with `messageID` and `callID`

#### Test 9: SSE event capture during question flow

Captured full SSE stream (17,847 bytes) to `/tmp/sse_events.txt`.

**SSE wire format**: `data: {JSON}\n\n` â€” standard SSE with no `id:` or `event:` fields.

**Event envelope**: `{"type": "<event-type>", "properties": {<payload>}}`


**`question.replied` event** (fired when answer submitted via REST):

```json
{
  "type": "question.replied",
  "properties": {
    "sessionID": "ses_18b28260affeoxXrX1iwPH8wFg",
    "requestID": "que_e74d81666001aP5o617w3vfdMb",
    "answers": [["dark"]]
  }
}
```

**All SSE event types observed during the flow:**

| Event Type | Description |
|-----------|-------------|
| `server.connected` | SSE connection established, properties `{}` |
| `server.heartbeat` | Keepalive, properties `{}` |
| `message.updated` | Message metadata changes (role, cost, tokens, finish) |
| `message.part.updated` | Part state transitions (text, tool, reasoning, step-start, step-finish) |
| `message.part.delta` | Streaming text/reasoning deltas (incremental field updates) |
| `session.updated` | Session metadata (title, timestamps, summary) |
| `session.status` | Session busy/idle state: `{"sessionID":"...","status":{"type":"busy"}}` |
| `session.diff` | File changes (empty array if none) |
| `question.asked` | New question pending |
| `question.replied` | Question answered |

## Complete Request/Response Contract Summary

| Endpoint | Method | Request Body | Response (200) |
|----------|--------|-------------|----------------|
| `/question` | GET | - | `QuestionRequest[]` |
| `/question/:id/reply` | POST | `{"answers":[["answer"]]}` | `true` |
| `/question/:id/reject` | POST | - | `true` |
| `/permission` | GET | - | `PermissionRequest[]` |
| `/permission/:id/reply` | POST | `{"reply":"once"}` | `true` |
| `/session` | GET | - | `Session[]` |
| `/session` | POST | - | `Session` |
| `/session/:id/prompt_async` | POST | `{"parts":[{"type":"text","text":"..."}]}` | (empty body) |
| `/event` | GET (SSE) | - | `data: {"type":"...","properties":{...}}\n\n` |
| `/config` | GET | - | Config object |
