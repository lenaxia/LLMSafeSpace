# Worklog: Epic 30 US-30.5 — Rewrite PrepareSecretsForInjection

**Date:** 2026-06-05
**Session:** Multi-source credential merge in injection pipeline
**Status:** Complete

---

## Objective

Rewrite `PrepareSecretsForInjection` to query `workspace_credential_bindings` (joined with `provider_credentials`), decrypt by owner type (user DEK or admin server KEK), deduplicate by provider in priority order, and merge with non-LLM secrets from `user_secrets`.

---

## Work Completed

- Added `deriveAdminKey AdminKeyDeriver` field to `SecretService`
- `SetAdminKeyDeriver(d)` setter — when non-nil, activates new path
- Renamed old implementation to `prepareSecretsLegacy` — fallback when admin key not wired
- New `PrepareSecretsForInjection`: type-asserts store to `CredentialStore`, queries bindings, decrypts by owner type, deduplicates by provider
- `decryptBinding`: dispatches to user DEK or server KEK
- `buildNonLLMSecrets`: reads `user_secrets` bindings, filters out LLM type, lazy DEK fetch
- `buildSecretsJSON`: produces the `[]InjectedSecret` JSON materializer format
- Model allowlist filtering: restricts models per-credential before merge
- Wired `secretService.SetAdminKeyDeriver(deriveServerKey)` in `app.go`

---

## Key Design Decisions

1. **Type assertion to CredentialStore** rather than widening the `store` field type — avoids changing `NewSecretService` signature, backwards compatible
2. **Lazy DEK fetch in buildNonLLMSecrets** — workspaces with only admin credentials don't need an active session
3. **Failed decryption skips without marking provider as seen** — allows fallback to lower-priority credential for same provider
