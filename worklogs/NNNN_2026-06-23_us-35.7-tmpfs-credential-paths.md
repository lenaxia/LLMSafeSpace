# Worklog: US-35.7 Tmpfs Credential Paths Implementation

**Date:** 2026-06-23
**Session:** Implemented US-35.7 ‚Äî redirected all credential-bearing files to RAM-backed tmpfs to close the PVC-at-rest plaintext gap.
**Status:** Complete

---

## Objective

Close the PVC-at-rest credential exposure: while a workspace is Suspended, plaintext credentials written by `materialize` persisted on the PVC and were recoverable by anyone who could attach it. US-35.7 redirects all credential files to tmpfs (RAM), which is wiped unconditionally when the pod dies.

## Key Decisions

1. **Two path resolvers updated:** `loadMaterializeConfig()` (production) AND `DefaultPaths()` (tests). The AI review on the design PR caught that updating only `DefaultPaths()` would silently leave `reset()` operating on PVC paths.
2. **No `touch auth.json`:** dangling symlink is correct ‚Äî `O_CREAT` follows it and creates the target on first write. A zero-byte file would break `JSON.parse`.
3. **`rm -rf` before `ln -s`:** structurally required ‚Äî `ln -s` into an existing directory creates the symlink inside it.
4. **Init container gets PVC mounts:** without `/home/sandbox` and `/workspace` RW mounts, `ReadOnlyRootFilesystem: true` causes `ln -s` to silently fail.
