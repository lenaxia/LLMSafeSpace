## Production code changes (TDD)

All changes follow red → green → refactor with meaningful tests (not framework tests).

### 1. `/livez` and `/readyz` endpoints

**Files:** `api/internal/server/router.go`, `api/internal/interfaces/interfaces.go`, `api/internal/mocks/{database,cache}.go`, `api/internal/server/router_health_test.go`

- Added `Ping(ctx context.Context) error` to `DatabaseService` and `CacheService` interfaces (production services already had these methods; only the interfaces lacked them)
- Added `/livez` (always 200) and `/readyz` (pings both deps with 2s timeout, 503 on failure)
- Preserved `/health` as a documented legacy alias for `/livez`
- Added `Ping` to mocks; compile-time interface checks force the mock to stay in sync

**Tests written (8 total, 1 low-value test removed during audit):**
- Happy path with both deps healthy
- Database down → 503 + body identifies which dep failed
- Cache down → 503 + body identifies which dep failed
- Both down → 503 + body lists both
- Nil database/cache (defensive) → 503, no panic
- `/health` legacy alias → 200
- `/livez` always 200
- **Probes bypass auth middleware** (regression for kubelet probes)
- **Probes excluded from RecordRequest** (regression for Prometheus cardinality)

**Removed:** `TestReadyz_PassesContextToPing` — tested implementation detail (that `context.WithTimeout` was used). Real timeout failures would surface in `TestReadyz_CacheDown` instead.

### 2. Middleware skip lists for probe paths

**Files:** `api/internal/middleware/{metrics,logging,auth}.go`

Default skip lists hardcoded `/health` and `/metrics`. Adding new probe paths without updating these would cause:
- **Auth middleware**: 401 for kubelet probes (no Authorization header), perpetually unready pods
- **Metrics middleware**: probe traffic (~8,640 reqs/pod/day per probe type) polluting Prometheus path cardinality
- **Logging middleware**: log spam at the same rate

Added `/livez` and `/readyz` to all three skip lists. Two new regression tests in `router_health_test.go` directly verify this behavior with strict mocks (auth that 401s without header; metrics service that asserts no `RecordRequest` call for probe paths).

### 3. `--watch-namespaces` flag in controller

**Files:** `controller/main.go`, `controller/watch_namespaces.go`, `controller/watch_namespaces_test.go`

Added the flag and wired it to `cache.Options.DefaultNamespaces`. Extracted parser as a pure function for testability.

**Tests written (7 total, 1 low-value test removed during audit):**
- Empty string → nil (cluster-wide, controller-runtime default)
- `"*"` → nil (explicit form; without this, "*" would be treated as a literal namespace name and silently watch nothing)
- Single namespace
- Multi namespace (comma-separated)
- Whitespace trimming (YAML leaves whitespace around values)
- Empty entries ignored (`"ns1,,ns2"`)
- All-empty input returns nil, not empty map (different semantics in controller-runtime: empty map watches NOTHING; nil map watches EVERYTHING)

**Removed:** `TestParseWatchNamespaces_DuplicatesCollapse` — tested Go map semantics, not my logic.

### Key chart features

**RBAC scoping** — `rbac.scope` toggles between `cluster` (ClusterRole + ClusterRoleBinding) and `namespace` (Role + RoleBinding scoped to release ns). Verified both modes render correctly.

**Webhook TLS via cert-manager** — Self-signed Issuer + Certificate produces a `<fullname>-webhook-cert` Secret mounted into the controller at `/tmp/k8s-webhook-server/serving-certs` (controller-runtime's default cert dir). The `cert-manager.io/inject-ca-from` annotation populates `caBundle` automatically.

**Webhook can be disabled** — `webhooks.enabled=false` skips ValidatingWebhookConfiguration, Certificate, and Issuer. NOTES.txt prints a security warning when disabled.

**Watch-namespaces wired** — Chart's `controller.watchNamespaces` value passes through to the controller's new `--watch-namespaces` flag.

**Probes use new endpoints** — `livenessProbe: /livez`, `readinessProbe: /readyz` (real DB+Redis check).

**Migrations as Helm hook** — Bundled as ConfigMap (Files.Glob is chart-relative, so migrations were copied into `charts/llmsafespace/migrations/`); Job uses official `migrate/migrate:v4.17.1` image.
