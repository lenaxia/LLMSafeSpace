# Worklog: US-45.1 — stateStore Abstraction Layer

**Date:** 2026-06-17
**Session:** Implement US-45.1 (extract per-workspace in-memory state from ProxyHandler into a `wsstate.Store` interface, no behavior change). Second foundational PR for Epics 44/45.
**Status:** Complete

---

## Objective

Externalize the six per-workspace state maps that `ProxyHandler` held as private fields (`activeSess`, `deletedSessions`, `pwCache`, `wsConfig`, `priorPhase`, `parentBackfilled`) behind a typed `wsstate.Store` interface with an in-memory implementation. This is the foundation for US-45.2 through US-45.8 (Redis-backed implementations) which eliminate multi-replica state drift — the root cause of the 2026-06-16 stuck-session incident. Per the story acceptance: NO behavior change.

## Key Decisions

1. **Package-local interface over `interfaces.WorkspaceStateStore`.** The original spec said define the interface in `api/internal/interfaces/`. The implementer chose `wsstate.Store` (package-local) because the only consumer is `ProxyHandler`, and the consumer-defined interface pattern is more idiomatic Go. Deviation documented in the design README.

2. **`h.state()` lazy getter for nil-safety.** Many existing tests construct `&ProxyHandler{...}` literally for narrow unit tests, bypassing `NewProxyHandler`. Without lazy init, every stateStore access would panic. The getter initializes on first call. Trade-off: a latent data race if a future test uses literals + goroutines. Acceptable because production code never hits the lazy path.

3. **`(!hadPrior)` replaces `prior == ""`.** The old code used `prior == ""` as the first-invocation signal. The new `(phase, ok)` tuple from `GetPriorPhase` makes `!hadPrior` the equivalent. The validator independently verified no phase emitted by the controller is ever `""`, so the two formulations are semantically equivalent in practice.

4. **All production callers go through `h.state()` not `h.stateStore`.** This is a small redundancy (one nil-check per call) but ensures tests using raw literals never panic. The alternative — `sync.Once` init — has its own cost.

5. **Per-data-type mutexes, not one big mutex.** A hot operation (active-session check on every prompt) must not block unrelated operations (password cache read on every request). The cost is 6 mutex fields; the benefit is no cross-contention.
