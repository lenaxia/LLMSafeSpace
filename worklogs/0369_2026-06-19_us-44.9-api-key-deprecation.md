# Worklog: US-44.9 — Aggressive api-key Deprecation

**Date:** 2026-06-19
**Session:** Implement US-44.9 — the final Epic 44 story. Deprecate the api-key secret type with a sunset date, UI banner, migration guide, and backend creation gate.
**Status:** Complete (pending PR review/merge)

---

## Objective

Deprecate the `api-key` secret type (legacy, superseded by `llm-provider` and `env-secret`). Add a sunset date (2026-12-19, 6 months from Epic 44 ship), a UI deprecation banner with migration guidance, a migration guide doc, and a backend gate that rejects new api-key creation after the sunset date. Existing api-key secrets continue to work.

---

## Work Completed

### Backend sunset gate (`pkg/secrets/`)
- `types.go`: `APIKeySunsetDate = "2026-12-19"` constant + testable `isAPIKeySunset` function variable.
- `secret_service.go`: gate in `CreateSecret` (after `ValidSecretTypes` check, before name validation) — rejects `type=api-key` after sunset with `ErrInvalidSecretType` → HTTP 400. Existing secrets unaffected (UpdateSecret doesn't re-check type).
- `api_key_sunset_test.go`: 4 TDD tests (pre-sunset allows, post-sunset rejects, non-api-key unaffected, predicate matches fixed date).

### Frontend banner + bug fix
- `SecretsTab.tsx`: `API_KEY_SUNSET_DATE` const + `formatSunsetDate` helper + updated banner text to match acceptance criteria ("⚠️ api-key secrets are deprecated. Migrate to llm-provider... api-key will be removed on December 19, 2026.").
- `SecretsTab.test.tsx`: updated banner-text assertions.
- `WorkspaceSettingsDrawer.tsx` (Rule 5 bug fix): `SECRET_TYPE_LABELS` was missing `llm-provider` (secrets silently dropped from per-workspace bind drawer) and mislabeling `api-key` as "LLM Providers". Fixed: added `llm-provider`, relabeled `api-key` as "API Keys (legacy)". 3 regression tests added.

### Migration guide
- `docs/migration/api-key-to-llm-provider.md`: why migrate, how to migrate (LLM APIs → llm-provider; other APIs → env-secret), existing secrets continue to work.

---

## Key Decisions

1. **Fixed sunset date as a constant** (not a runtime setting) — the date is a one-time 6-month deadline, not an operator-tunable knob.
2. **Create-only gate** — existing api-key secrets remain functional; only new creation is blocked after sunset.
3. **Testable predicate** (`isAPIKeySunset` as a function variable) — tests swap it via `t.Cleanup` without waiting for the real date.
4. **`WorkspaceSettingsDrawer` bug fixed** per Rule 5 — pre-existing mislabeling/missing type entries.
