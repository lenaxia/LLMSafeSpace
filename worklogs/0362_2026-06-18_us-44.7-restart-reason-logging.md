## Objective

Implement US-44.7 from `design/stories/epic-44-session-reliability-transparency/README.md`. When opencode restarts (crash, OOM, env-secret/api-key change), agentd writes a JSON marker file (`/workspace/.opencode-restart-reason`) to the PVC before the restart, recording the reason, timestamp, and changed secret names. The reason is logged in real-time (at write time) and re-surfaced on the next pod boot (with stale-marker detection to avoid misattribution).

Followed the Multi-Agent Workflow: TDD → independent skeptical validator (2 rounds → zero findings) → commit/PR.

## Work Completed

### New file `cmd/workspace-agentd/restart_reason.go` (177 lines)
- `RestartReasonMarkerPath = "/workspace/.opencode-restart-reason"` (PVC-backed, persists across pod restarts — consistent with the existing `OOMMarkerPath`).
- `restartReason` struct: `{Reason, Timestamp, SecretNames}` (matches spec format exactly; omitempty on SecretNames).
- `writeRestartReasonMarker(path, reason, secretNames)` — writes JSON marker (MkdirAll + 0600).
- `readRestartReasonMarker(path, core)` — reads + unmarshals; returns false on missing/corrupt (no boot failure).
- `logRestartReason(markerPath, core)` — boot-time: reads, logs (Info if fresh / Debug+stale-note if >10min old), deletes. Stale detection prevents misattribution when a pod boots for an unrelated reason (node drain) days after a crash.
- `logRestartReasonAtWrite(reason, secretNames, core)` — real-time: logs "opencode restart scheduled" at Info (Warn for oom) immediately after each marker write. This is the PRIMARY logging path — gives operators real-time visibility without waiting for a pod reboot.
- `classifySecretRestartReason(batch)` — classifies a secret batch: api-key present → `"api_key_changed"` (precedence); env-secret present → `"env_secrets_changed"`; else `""`. Collects secret names (prefers `Metadata["var_name"]` for env-secrets).

### Integration into 4 restart trigger points
- **env_secrets_changed / api_key_changed** (`secrets.go`): writes marker + logs at write time BEFORE `makeSessionAwareRestartDecision` (once up-front; survives deferral).
- **crash** (`main.go` supervisor crash path): writes marker + logs alongside `RecordRestart("crash")`.
- **oom** (`oom_detection.go` `handleOOMExit`): writes restart-reason marker alongside the existing OOM-specific marker (both retained — OOM marker has exitCode/memoryLimit; restart-reason marker is the generalized one).
- **user_requested**: DEFERRED — no trigger exists today. `/v1/agent/reload` only disposes the opencode instance (in-process), not the OS process. The API server consumes `drain` itself and forwards a param-less request. Creating the trigger requires extending the API↔agentd protocol — separate concern.

### Boot-time read (`main.go:833`)
`logRestartReason(RestartReasonMarkerPath, log.Core())` called before `proc.start()` — surfaces the previous run's restart reason (if any) before the new opencode boots.

## Key Decisions

1. **Marker path `/workspace/` (not `/home/workspace/`).** The design doc says `/home/workspace/` which doesn't exist in the container. The existing OOM marker uses `/workspace/` (PVC subPath:workspace, persists across restarts). Corrected with documentation.
2. **Dual logging: write-time (primary) + boot-time (secondary).** The validator caught that boot-only logging misses in-pod restarts (crash/oom supervisor respawns never re-run `main()`). Write-time logging gives real-time visibility; boot-time read with 10-minute stale detection handles cross-pod attribution without misattribution.
3. **`user_requested` deferred.** No trigger exists — `/agent/reload` only disposes the instance. Creating a process-restart trigger requires API↔agentd protocol changes (a separate concern from logging reasons). The initial implementation attempted to wire it but the validator proved the drain branch was unreachable dead code; reverted for honesty.
4. **Both OOM markers coexist.** The OOM-specific marker (`/workspace/.opencode-oom-marker` with exitCode/memoryLimit) and the generalized restart-reason marker (`/workspace/.opencode-restart-reason` with reason/timestamp/secretNames) serve different dashboards. Neither replaces the other.
5. **Logger injection via `zapcore.Core`.** Three functions whose output is test-asserted take `zapcore.Core` instead of reading the package-global `log`. Enables `zaptest/observer`-based assertions on emitted level/message/fields.
