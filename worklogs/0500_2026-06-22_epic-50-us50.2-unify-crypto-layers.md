# Worklog: Epic 50 — US-50.2 Unify Crypto Layers under RootKeyProvider

**Date:** 2026-06-22
**Session:** Implement US-50.2 (Phase 2 FOUNDATIONAL) — eliminate `AdminKeyDeriver`, unify all credential encrypt/decrypt under `RootKeyProvider`.
**Status:** Complete (implementation + validation); PR pending.

---

## Objective

Eliminate the `AdminKeyDeriver func(label string) []byte` callback (Layer 2) and
unify it with `RootKeyProvider` (Layer 1) so every credential encrypt/decrypt in
the system flows through one interface. This is the foundational refactor that
enables US-50.3 (key_version), US-50.4 (multi-key rotation), and US-50.12
(decrypt audit logging).

## Key Decisions

1. **Decrypt closures, not raw keys.** `decryptBinding` and `credential_ops.go`
   accept `func(ctx, []byte) ([]byte, error)` closures. Admin/org pass
   `provider.Decrypt`; user passes a DEK-wrapping closure. This unifies all paths
   without forcing user credentials through `RootKeyProvider` (they use a
   password-derived DEK, not the server KEK).
2. **`apiKeyProv` unifies both else branches.** Instead of separate
   `rkp != nil ? rkp : StaticKeyProvider(dekMasterKey())` logic in two places,
   a single `apiKeyProv` variable is computed once and used by both the auth
   service and the key service.
3. **Deprecated (not removed) `AdminKeyDeriver`.** Per the design's rollback-safety
   requirement, the type is retained for one release so a production bug can be
   patched by wiring the old callback path back.
