## Key Decisions

1. **Workspace types in pkg/apis/v1**: The `LLMSafespaceV1Interface` needed `Workspaces()` but `Workspace` type only existed in `controller/internal/resources/`. Added a minimal, API-usage-only version to `pkg/apis/llmsafespace/v1/` following the same pattern as Sandbox. The controller's version remains authoritative; the pkg/v1 version is the REST client transport type.

2. **credentials via k8s.Clientset()**: Credentials use the standard k8s `CoreV1().Secrets()` API rather than a custom CRD client, matching the design spec (§5.7). This is the correct approach since Secrets are a core k8s resource.

3. **SetCredentials test strategy**: Used `k8sfake.NewSimpleClientset()` for full integration-level testing of credential create/update flows in unit tests. This avoids mocking the entire k8s client chain and gives better confidence.

4. **ListWorkspaces doesn't fetch CRD phase**: Per spec ("CRD status not fetched for list — too expensive"). Phase comes from DB mirror field.

5. **DeleteWorkspace keeps DB record, sets phase to "terminating"**: The controller owns the actual deletion cleanup; the API only initiates it and mirrors the state.
