## Key Decisions

1. **No `OPENCODE_SHA256` ARG**: §9.8 pattern fetches the co-published `.sha256` file from the same release rather than embedding a hash. This is the spec's stated pattern; it means the SHA256 rotates with each release without Dockerfile changes.

2. **`test -n "${OPENCODE_VERSION}"` guard**: Added before the download. Since `ARG OPENCODE_VERSION=1.0.0` has a default, the guard is belt-and-suspenders — it ensures an explicit override is provided (the default `1.0.0` may not be a real release). Operators must pass `--build-arg OPENCODE_VERSION=<real-version>`.

3. **Base image not digest-pinned**: The spec uses `<pinned-digest>` as a placeholder. Cannot compute the actual digest without pulling the image. Used `debian:bookworm-slim` (tag only) with a comment instructing how to pin it. This is a known deviation; Renovate bot or CI can pin it once the first build runs.

4. **Inline smoke test**: `smoke-test.sh` does not exist. Used `RUN which redact && which bash && which curl && which git && which jq && which opencode` per the delegation spec's fallback.

5. **No PATH-shadowing wrappers**: Confirmed deferred to V2.1 per §13.4.

6. **No V1 apparmor/seccomp profiles**: Deferred to V2.1 per story spec.

7. **`DEBIAN_FRONTEND=noninteractive`**: Set as ENV to suppress apt prompts during build.

8. **`--chmod=755` on COPY**: Used for both entrypoint scripts and redact binary, per §13.2 pattern.
