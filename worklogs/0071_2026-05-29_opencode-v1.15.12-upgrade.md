### Change 1: Opencode Version Bump

### Change 2: Fix `persistTitleFromEvent` (pre-existing bug)

**File:** `api/internal/handlers/proxy.go` (line 871)

The function was parsing `properties.id` and `properties.title` but the actual `session.updated` event has these nested at `properties.info.id` and `properties.info.title`. This meant SSE-driven title persistence never worked — titles were only persisted via the `fetchAndPersistTitle` HTTP GET fallback.

**Before:** Two parsers (flat + nested) both looking for wrong field paths. Neither ever matched.
**After:** Single parser matching the actual event shape: `properties.info.id` and `properties.info.title`. Works for both v1.2.27 and v1.15.12 event formats.

### Change 3: Add `ID` field to `sseEvent` struct

**File:** `api/internal/handlers/session_tracker.go` (line 25)

Added `ID string \`json:"id"\`` to the `sseEvent` struct. This captures the event ID that opencode v1.15 includes on every event (`evt_<ascending-id>`). Enables future logging/debugging of event ordering and deduplication.

### Change 4: Strip workspace/directory query params

**File:** `api/internal/handlers/proxy.go` (`stripVerboseQuery` function)

Added `values.Del("workspace")` and `values.Del("directory")` alongside the existing `values.Del("verbose")`. LLMSafeSpace manages workspace routing at the pod level via `OPENCODE_WORKSPACE_ID` env var — these query params are never needed and opencode v1.15 validates them strictly (returns 400 for invalid format).

### Change 5: Future-proofing comment for compression

**File:** `api/internal/handlers/proxy.go` (line 350)

Added a comment above `stripPatch := false` documenting that if `stripPatch` is ever re-enabled, `Accept-Encoding` must be stripped from the upstream request because opencode v1.15+ compresses JSON responses >1KB.

---

## Key Decisions

| Decision | Rationale |
|----------|-----------|
| Single parser in `persistTitleFromEvent` instead of two fallbacks | The nested/global format is never received on the instance `/event` stream. One parser matching the actual shape is clearer than two parsers matching nothing. |
| Strip `workspace`/`directory` rather than validate them | Simpler, defensive, and these params serve no purpose in our single-workspace-per-pod model. |
| Comment-only for compression guard | `stripPatch` is intentionally dead code. A runtime guard would be dead code too. A comment is the right level of documentation. |
