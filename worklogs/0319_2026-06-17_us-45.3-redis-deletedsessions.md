# Worklog: US-45.3 — Redis-backed deletedSessions

**Date:** 2026-06-17
**Session:** Move session tombstones from per-replica in-memory map to Redis, preventing late-event zombie resurrection across replicas.
**Status:** Complete

---

## Objective

Session tombstones prevent late SSE events from opencode from re-inserting a deleted session into `session_index`. Moving them to Redis ensures the suppression is cluster-wide — previously a session deleted on replica A could be resurrected by a late event arriving on replica B.

---

## Work Completed

### Implementation (`redis.go`)
- Overrode `MarkSessionDeleted` → Redis `SET` with per-key TTL (30min)
- Overrode `IsSessionDeleted` → Redis `EXISTS`, **fail-CLOSED** (returns `true` on error — assume deleted to prevent resurrection)
- Overrode `ClearDeletedSessions` → Redis `SCAN` + `DEL` for `ws:{workspace_id}:deleted:*` pattern
- Updated `InvalidateAll` to clear both Redis-backed sections (activeSess + deletedSessions) and InMemoryStore-backed sections explicitly, rather than calling `inMemory.InvalidateAll()` which would no-op the Redis sections

### Key design decisions
1. **Fail-CLOSED** (opposite of activeSess fail-OPEN): `IsSessionDeleted` returns `true` on Redis error. Rationale per design: "If we can't verify, assume deleted; user can recreate session" — data integrity > availability.
2. **Per-key TTL** (not shared SET TTL like activeSess): each tombstone expires independently. No threshold-based eviction needed — the old InMemoryStore's 500/250 batch eviction is obsolete when Redis handles per-key expiry natively.
3. **Hash-tagged key** `ws:{workspace_id}:deleted:{session_id}` — forces co-location with activeSess key on the same Redis shard.

### Tests (`redis_deleted_test.go` — 12 tests)
- Happy paths: mark-then-isDeleted, multiple workspaces tracked separately, idempotent re-mark
- TTL: per-key TTL set, auto-expire after TTL
- ClearDeletedSessions: scoped to workspace, no-op on empty
- Fail-closed: IsSessionDeleted returns true on Redis down; MarkDeleted and ClearDeletedSessions don't panic
- InvalidateAll integration: tombstones cleared via Redis
