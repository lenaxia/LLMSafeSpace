# Worklog: Session Title Fetched from OpenCode Agent

**Date:** 2026-05-26
**Session:** Fetch opencode session title via proxy and display it in the UX
**Status:** Complete

---

## Objective

Surface the session title generated by the opencode agent in the sidebar and anywhere else sessions are listed. Previously, the sidebar showed a fallback timestamp string for all sessions because `SessionListItem.title` was only populated if it had been explicitly set via the rename API — the auto-generated opencode title was never fetched.

---

## Work Completed

### Design decision

Evaluated two approaches:
1. Proxy `GET /session/:id` directly through the existing proxy handler (option 1)
2. Add a dedicated API endpoint that wraps the opencode session call (option 2)

Chose option 1 (proxy through) — consistent with the existing proxy handler pattern, no duplication, and the UX already knows the session ID. The future multi-agent adapter layer (if/when added) would replace the proxy at that point, not now.

### Backend — `api/internal/handlers/proxy.go`

Added `GetSession` handler: proxies `GET /session/:sessionId` on the opencode agent (`/session/<id>`) back to the caller. No patch-stripping needed (JSON, not a message list). Follows the identical pattern of `GetHistory`, `AbortSession`, etc.

### Backend — `api/internal/server/router.go`

Registered `GET /:id/sessions/:sessionId` → `proxyHandler.GetSession` in `registerProxyRoutes`. Route sits between `GetHistory` and `AbortSession` to keep the session-scoped GET routes together.

### Frontend — `src/api/types.ts`

Added `OpenCodeSession` interface matching the shape returned by opencode's `GET /session/:id`:
```ts
{ id, title?, parentID?, share? }
```

### Frontend — `src/api/workspaces.ts`

Added `getSession(workspaceId, sessionId)` → `GET /workspaces/:id/sessions/:sessionId`.

### Frontend — `src/hooks/useSessionTitle.ts` (new file)

New TanStack Query hook. Behaviour:
- Fetches on mount when `workspaceId`, `sessionId`, and `active` (workspace is Active) are all set.
- Re-fetches when `streaming` transitions `true → false` — this is when opencode generates/updates the title after the first exchange.
- `retry: false` — a brand-new session may not exist in opencode yet; silent 404 is correct.
- `refetchOnWindowFocus: false` — title only changes after messages, not on tab switch.
- When a non-empty title is received (or changes), invalidates `["sessions", workspaceId]` so the sidebar list refreshes automatically without a manual poll.

### Frontend — `src/pages/ChatPage.tsx`

Added `useSessionTitle(activeWorkspaceId, sessionId, isReady, streaming)` call after `useChatStream`. The hook is self-contained — no return value needed in the page; all side effects (invalidation) happen inside the hook.

### Frontend — `src/lib/names.ts`

Added `sessionDisplayTitle(title, lastMessageAt)` — single canonical title fallback used everywhere:
- Returns `title` if set
- Falls back to `"Chat <relative-time>"` if `lastMessageAt` is available
- Falls back to `"New chat"` otherwise

Fixes the pre-existing inconsistency: `Sidebar.tsx` used `"Chat <relative>"` while `SessionItem.tsx` used `"Session at <HH:MM>"`.

### Frontend — `src/components/layout/Sidebar.tsx`

- Replaced inline title fallback with `sessionDisplayTitle`.
- Removed unused `formatRelativeTime` import.

### Frontend — `src/components/session/SessionItem.tsx`

- Replaced inline title fallback with `sessionDisplayTitle`.
- Kept `formatRelativeTime` import (still used for the timestamp label on line 31).

---

## Key Decisions

- **No dedicated aggregation endpoint** — not needed now; if fan-out across sandboxes is required later, add it then.
- **`retry: false` on `useSessionTitle`** — a new session has no opencode session object yet; retrying would hammer the API for no benefit. The re-fetch on `streaming → false` handles the "title just became available" case.
- **Invalidate, don't mutate cache directly** — invalidating `["sessions", workspaceId]` lets the existing `WorkspaceSessionList` query re-fetch naturally, keeping a single source of truth for the session list.
