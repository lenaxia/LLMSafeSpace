# Worklog: Epic 50 — US-50.4 Multi-Key Provider

**Date:** 2026-06-22
**Session:** Implement US-50.4 — multi-key support in StaticKeyProvider for zero-downtime rotation.
**Status:** Complete.

## Objective
Enable the local provider to decrypt ciphertexts from multiple key versions during the rotation transition window (design D4).

## Work
- `root_key.go`: StaticKeyProvider gains `[]keyEntry` (sorted by version desc). Encrypt uses entries[0] (active/highest). Decrypt tries each key newest-to-oldest. New `NewStaticKeyProviderMultiVersion(activeVersion, keyByVersion)` constructor. `ActiveVersion()` returns the highest version (for US-50.3's write path). Existing `NewStaticKeyProvider(key)` wraps to single-entry at version 1 (backward compatible).
- `us504_multikey_test.go` (new): 5 design-mandated tests (decrypt-by-trial, encrypt-uses-highest, active-version-returns-highest, wrong-key-errors, single-key-backward-compat).
