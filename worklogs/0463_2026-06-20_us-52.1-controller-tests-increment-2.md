## Objective

Close the side-effect gaps left open by increment 1. The existing
controller_test.go tests assert phase transitions but not the **load-bearing
deletions and state-clearing** that make those transitions safe. A regression
that skipped a PVC delete or left recovery state set would pass the existing
tests but leak resources or cause zombie recovery on resume.

## Key Decisions

- **Test load-bearing side effects, not just phase transitions.** The existing
  tests verify "phase changed to Suspended" but not "recovery state was cleared"
  or "PVC was deleted." These are the assertions that catch real regressions.
- **Direct method call for handleDeletion no-finalizer branch** ŌĆö the fake
  client refuses to seed deletionTimestamp without finalizers (K8s invariant),
  so the early-return guard is unreachable through Reconcile(). Called
  handleDeletion directly (it only checks the finalizer, not the timestamp) ŌĆö
  faithful to the branch under test.
- **No envtest in this increment** ŌĆö envtest requires setup-envtest binary +
  etcd/kube-apiserver binaries, unavailable here. Deferred to a CI-only
  increment.
