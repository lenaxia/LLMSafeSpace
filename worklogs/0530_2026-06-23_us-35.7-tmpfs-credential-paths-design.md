# Worklog: US-35.7 Tmpfs Credential Paths Design

**Date:** 2026-06-23
**Session:** Designed US-35.7 (tmpfs redirect + symlink farm) to close the PVC-at-rest plaintext gap left by Epic 35.
**Status:** Complete (design approved, implementation pending)

---

## Objective

Close the PVC-at-rest credential exposure gap identified in Epic 35's "Known Gap" section. While a workspace is Suspended, plaintext credentials written by `materialize` persist on the PVC and are recoverable by anyone who can attach it.

---

## Work Completed

Authored `design/stories/epic-35-secretless-credential-injection/US-35.7-tmpfs-credential-paths.md` covering:

- **Direct redirect (Groups A+B):** `agent-config.json` and `secrets-env` move from PVC `/tmp/*` to a new RW tmpfs volume `sandbox-runtime` (96Mi). Both are env-var-indirected — zero lifecycle impact.
- **Symlink farm (Groups C+D):** SSH keys, git credentials, secret-files, and auth.json redirected to tmpfs via symlinks created by the init container. PVC retains only symlink inodes.
- **Why not delete-on-suspend:** SIGKILL-unsafe. Tmpfs dies with the pod unconditionally.
- **Why not a per-workspace proxy:** same-pod RCE defeats it (process memory, localhost interception). The at-rest gap is real and solvable; the runtime-RCE gap is fundamental.

Conducted an adversarial stress-test that found two issues before the AI review:
1. Init container needs RW PVC mounts (`/home/sandbox` + `/workspace`) — `ReadOnlyRootFilesystem` would silently fail symlink creation
2. `touch auth.json` creates invalid JSON — dangling symlink + `O_CREAT` is correct

AI review found one critical issue: the design targeted `DefaultPaths()` which is never called by production code. The real resolver is `loadMaterializeConfig()` in `cmd/workspace-agentd/secrets.go:301-312`. Fixed in second commit — both resolvers now updated.

---

## Key Decisions

1. **Symlink farm over `XDG_DATA_HOME` redirect:** redirecting `XDG_DATA_HOME` to tmpfs would move opencode's session DB to RAM, regressing session persistence. The auth.json symlink achieves the same goal without the regression.
2. **`rm -rf` before `ln -s`:** structurally required — `ln -s` into an existing directory creates the symlink inside it. Not migration; just setup.
3. **No pre-creation of auth.json/git-credentials:** dangling symlinks; `O_CREAT` follows the symlink and creates the target on first write.
4. **Two resolvers must be updated:** `loadMaterializeConfig()` (production) + `DefaultPaths()` (tests). Updating only one causes silent regression.
