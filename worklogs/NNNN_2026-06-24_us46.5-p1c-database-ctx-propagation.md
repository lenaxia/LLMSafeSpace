# Worklog: US-46.5 P1-c — propagate ctx through database checks (#224)

**Date:** 2026-06-24
**Session:** Third migration from the #224 ctx-propagation audit (P1-c)
**Status:** Complete

---

## Objective

Execute P1-c: thread `context.Context` through `database.CheckResourceOwnership`
and `CheckPermission` (switch `QueryRow` → `QueryRowContext`) so request
deadlines/cancellation reach these ownership/permission queries.

---

## Work Completed

- `database.go`: `CheckResourceOwnership(ctx, …)` + `CheckPermission(ctx, …)`;
  both now use `s.DB.QueryRowContext(ctx, …)`. Removed the two `//nolint:noctx`
  pragmas + their stale "caller-side limitation" comments (the limitation is
  now resolved).
- `interfaces.DatabaseService`: both methods take ctx.
- `mocks/database.go` (`MockDatabaseService`): accept ctx, call `m.Called(…)`
  without ctx so existing `.On("CheckResourceOwnership", userID, …)` setups
  keep matching (ValidateToken mock pattern).
- Live caller `handlers/usage.go` → `c.Request.Context()`.
- Dead caller `auth.CheckResourceAccess` (0 production callers; interface-
  satisfying only, has its own test) → `context.Background()` with a comment
  noting it's pending deletion/ctx-propagation. (Deleting it would ripple into
  the AuthService interface + 2 mocks + its test; kept minimal for P1-c.)
- Test helpers implementing `DatabaseService` (`apiKeyAwareDB`, `fullMockDB`,
  `mockDB`, `recordingDB`) updated to the new signatures.

### Tests

- New `TestCheckResourceOwnership_PropagatesContext` (TDD): a pre-cancelled ctx
  must make `QueryRowContext` fail fast (`database/sql` checks `ctx.Err()`)
  WITHOUT running the query — verified empirically that the error is
  `context.Canceled` and `mock.ExpectationsWereMet()` passes (query never ran).
  A `context.Background()` regression would run the query and return no error.
  TDD: probe-confirmed the cancellation path, then wrote the assertion.

---

## Key Decisions

- **Don't delete CheckResourceAccess in this PR.** It's dead (0 prod callers)
  but has a test + interface/mock entries; deleting is a wider cleanup. It uses
  `context.Background()` (documented) so the live `usage.go` path gets the
  request-ctx win without churning the dead method's signature.
- **Mock ignores ctx in matching.** Preserves every existing
  `.On("CheckResourceOwnership", …)` setup (which omits ctx).
