# Worklog: Epic 51 S51.1 + S51.3 + S51.4 — Tenant Isolation Implementation (Tranche 1)

**Date:** 2026-06-20
**Session:** First implementation tranche of Epic 51 (gVisor RuntimeClass, tenant pod labels, hardening tests).
**Status:** PR #310 opened for review.

---

## Objective

Implement the primary container-escape control (gVisor RuntimeClass), tenant pod labels for quota enforcement, and hardening regression tests under gVisor. S51.2 (admission webhook quotas) deferred to a follow-up PR.

---

## Work Completed

### S51.1 — gVisor RuntimeClass

- **RuntimeClass template** (`charts/llmsafespaces/templates/runtime-class.yaml`): Creates `gvisor` RuntimeClass (handler: `runsc`), gated on `.Values.gvisor.enabled`.
- **Controller flag** (`--default-runtime-class`): Added to `controller/main.go`, wired through `controller.go` to `WorkspaceReconciler.DefaultRuntimeClass`.
- **Pod builder**: `buildPod` sets `RuntimeClassName` from `DefaultRuntimeClass`, with per-workspace `spec.runtimeClass` override taking precedence.
- **WorkspaceSpec**: Added `RuntimeClass *string` field for per-workspace opt-out (e.g. `"runc"` for ptrace-incompatible workloads). Documented that webhook enforcement is deferred to S51.2.
- **CRD YAML**: Added `runtimeClass` to both `charts/llmsafespaces/crds/workspace.yaml` and `pkg/crds/workspace_crd.yaml`.
- **Helm values**: Added `gvisor:` section (`enabled: false`, `defaultRuntimeClass: "gvisor"`) with operator prerequisites docs.
- **Controller deployment**: Wires `--default-runtime-class=gvisor` when `gvisor.enabled`.

### S51.3 — Tenant Pod Label

- **Label constant + helper**: `LabelTenant = "llmsafespaces.dev/tenant"` + `tenantID(owner)` helper (OrgID || UserID) in `constants.go`.
- **Pod labels**: `buildPod` adds the tenant label to every workspace pod.
- **CR labels**: `buildWorkspaceCRD` adds the tenant label to the Workspace CR.
- **Security fix**: User-supplied labels are merged first, then system labels overwrite — prevents spoofing the tenant identity via `req.Labels`.

### S51.4 — Hardening Under gVisor

- `TestS51_4_HardeningSurvivesUnderGVisor`: Asserts AutomountServiceAccountToken, EnableServiceLinks, and SeccompProfile all survive when RuntimeClassName=gvisor.
- 9 controller-side tests + 3 API-side tests covering all paths.

## Review Feedback (PR #310)

AI review found three issues (all fixed in follow-up commit):

1. **Security: Tenant label override** — User-supplied labels could override the system tenant label on the Workspace CR. Fixed by reordering: user labels merged first, system labels second (system wins).
2. **Missing API-side tests** — Added `epic51_tenant_label_test.go` with 3 tests including spoof prevention regression test.
3. **Missing worklog** — This entry.
4. **RuntimeClass enforcement** — Doc comment amended to acknowledge webhook enforcement deferred to S51.2.

---

## Key Decisions

1. **RuntimeClassName via controller flag** — Follows existing pattern (CLI flags → reconciler struct), consistent with `--inference-relay-url`, `--api-service-url`.
2. **`*string` for spec.runtimeClass** — Nullable pointer distinguishes "not set" (use default) from "explicitly empty" (force runc).
3. **System labels overwrite user labels** — Security-critical ordering; prevents tenant identity spoofing.
