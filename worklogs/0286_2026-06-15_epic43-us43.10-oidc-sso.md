## Objective

Implement OIDC single sign-on for orgs per D17: OIDC Authorization Code Flow, auto-provisioning, group claim → role mapping, domain-to-org matching.

---

## Work Completed

### Migration 035
- `org_sso_configs` table: discovery URL, client ID, encrypted client secret, group claim config, auto-provision toggle, enabled flag.

### Backend
- `OrgSSOConfig` type + `SetOrgSSOConfigRequest` / `OrgSSOConfigResponse` DTOs.
- SSOConfigStore: `GetSSOConfig`, `UpsertSSOConfig`, `DeleteSSOConfig`, `GetSSOConfigByOrgSlug`.
- `SSOHandler`: GET/PUT/DELETE `/orgs/:id/sso` (admin-only, never returns secret).
- `OIDCCallbackHandler`: GET `/auth/oidc/:orgSlug/login` (redirect to IdP), GET `/auth/oidc/:orgSlug/callback` (token exchange, userinfo, auto-provision, group→role mapping).
- Uses `github.com/coreos/go-oidc/v3/oidc` + `golang.org/x/oauth2` for the OIDC flow.
- State cookie for CSRF protection; PKCE via the oidc library.
- Auto-provisioning: creates user + membership on first login when enabled.
- Group claim mapping: checks user's groups for admin/member claim values.
- `oidcStoreAdapter` in app.go composes PgOrgStore + database.Service.

### Frontend
- `OrgSSOTab`: discovery URL, client ID/secret, group claims, auto-provision/enable toggles.
- Nav item in OrgAdminLayout (admin-only).
- API client: `getSSO`, `updateSSO`, `deleteSSO`.

### Key Decisions
- Client secret encryption placeholder (stores raw bytes; production wiring with OrgKeyService is a follow-up). The API never returns the secret.
- OIDC callback redirects to login page with email parameter for session completion; full JWT issuance from OIDC is a follow-up.
