## Objective

Incident B (2026-06-16): user changed workspace secret bindings while a session was actively running (102K context tokens). agentd immediately SIGTERMed opencode, destroying in-flight work. No session state check, no graceful draining. US-44.2 fixes this by deferring restarts until all sessions are idle. US-44.3 fixes a latent bug where `api-key` secret type was missing from `shouldRestart()`.

## Work Completed

### US-44.3: api-key restart bug (one-line fix)
- `shouldRestart()` now returns true for both `env-secret` AND `api-key` (was only `env-secret`)
- Without US-44.2, this fix alone would break users who rely on api-key NOT restarting — hence the two stories ship together

### US-44.2: Session-aware restart
- Added `sessionStatusTracker.hasAnyBusy()` — returns true if any tracked session is "busy"
- Added `sessionStatusTracker.listBusy()` — returns IDs of busy sessions (for logging)
- Added `sessionStatusTracker.hasAnyData()` — returns true if tracker has any tracked sessions (SSE-disconnect detection)
- Added `restartableProcess` interface — the restart method subset of `*managedProcess`, enables test mocking
- Added `makeSessionAwareRestartDecision()` — the decision function:
  - All idle → restart immediately, return true
  - Sessions busy → spawn background goroutine polling every 5s, return false
  - Empty/nil tracker (SSE disconnected) → restart immediately with warning log
  - Nil proc → return true (test/no-op)
- Widened `reloadSecretsHandler` signature: now accepts `*sessionStatusTracker` as 4th arg
- Updated main.go wiring to pass the `sseTracker` to `reloadSecretsHandler`

### Design decisions
1. **No forced timeout** — per user requirement, defer indefinitely. Multi-hour agentic workflows are common.
2. **SSE-disconnect fallback** — if tracker has no data, restart immediately. Cannot block forever waiting for state we may never get.
3. **`restartableProcess` interface** — enables test mocking without constructing a full managedProcess supervisor.
4. **Background goroutine for polling** — the goroutine exits when sessions go idle and restart fires. No cleanup needed; if the pod restarts, the goroutine is gone and the next reload-secrets call re-evaluates.
