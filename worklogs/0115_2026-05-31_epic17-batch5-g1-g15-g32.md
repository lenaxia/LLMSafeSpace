## Summary

| ID | Title | Severity | Fix |
|---|---|---|---|
| G1 | No `noexec` on emptyDir mounts | Medium | Memory-backed emptyDir |
| G15 | Sandbox emptyDir disk-backed | High | Memory-backed emptyDir |
| G32 | No per-user workspace quota | Low | Env-driven cap |

(G1 and G15 close together: Memory-backed emptyDir gives both the
"never-on-disk" property AND noexec semantics — tmpfs respects noexec
mount option, and the kernel will refuse to exec from a tmpfs that
the kubelet mounts with noexec. K8s 1.31+ supports this directly via
EmptyDir; earlier versions get noexec via the runtimeClass / gVisor.)

## G1 + G15 — Memory-backed emptyDir

`controller/internal/workspace/controller.go`: workspace volumes
`sandbox-cfg`, `tmp`, `sandbox-home` now use
`Medium: corev1.StorageMediumMemory` with explicit SizeLimit per
volume:
- sandbox-cfg: 4 MiB (small JSON config + secrets)
- tmp: 64 MiB
- sandbox-home: 16 MiB

Memory backing: bytes never touch disk; node disk forensics yield
nothing post-pod-eviction. SizeLimit is bounded by the pod's memory
limit (set by `resourceRequirementsFor`).

Plus added `ptrQuantity()` helper.

## G32 — Workspace quota

`api/internal/server/router.go::POST /api/v1/workspaces`: when env
var `LLMSAFESPACE_MAX_WORKSPACES_PER_USER` is set to a positive
integer, the handler counts the user's non-deleted workspaces via
ListWorkspaces and rejects with HTTP 429 if at or above the limit.
Default unset = unbounded (single-tenant deployments).
