## Objective

Execute P1-b: thread `context.Context` through the per-workspace state store
(`wsstate.Store`) so request deadlines/cancellation reach the Redis calls on
the proxy hot path (every chat / SSE / terminal request touches this store).

### Bug caught by `-race` (and fixed)

The mechanical migration initially changed `proxy_handlers.go:259`
(`h.removeActiveSession` inside DeleteSession's fire-and-forget `go func()`)
to use `c.Request.Context()`. The `-race` detector caught a data race:
capturing `c` in that goroutine races with gin reusing the Context on the next
ServeHTTP. Fixed to `context.Background()` — correct both because the goroutine
must survive client disconnect (matching the sibling `sessionIndex.DeleteSession`
at :253 which uses `context.Background()` with `//nolint:contextcheck`) and
because capturing the request ctx there is unsafe.

### Tests

- New `TestRedisStore_PropagatesContext` (TDD): a pre-cancelled ctx must reach
  the Redis client. go-redis returns `context.Canceled` client-side;
  `IsSessionDeleted` fail-closes → `true`. A live ctx on an empty store →
  `false`. The true/false distinction proves propagation (a `context.Background()`
  regression would return false). TDD: written first, confirmed red, then
  implemented.
- All wsstate + handler test call sites updated to pass `context.Background()`
  (bulk perl + goimports for missing imports).
- Full wsstate + handlers suites pass under `-race`.

## Key Decisions

- **Event-callback path uses `context.Background()`.** These are
  tracker/watcher callbacks with no request ctx; the request-path win (gin
  handlers → `c.Request.Context()`) is the audit's actual goal. Threading ctx
  through the tracker/watcher callback signatures is a separate, larger effort.
- **ForTest helpers keep signatures.** Test-only; passing `context.Background()`
  internally avoids churning every test caller for no behavioral gain.
- **TDD via fail-closed discriminator.** `CheckAndAddActiveSession` fails-open
  (returns true on error), so it can't distinguish a propagated cancelled-ctx
  from success. `IsSessionDeleted` fails-closed (returns true on error) on an
  empty store (returns false) — the cleanest propagation signal.
