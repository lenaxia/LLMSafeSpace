## Key Decisions

- **Test real functions, not story-invented names.** When the story's test
  inventory names functions that don't exist (PreferNamespaced, ArchFiltered,
  GetExistingByName, MaterializesWorkspaceSecret), test the actual code paths
  and document the divergence. This is Rule 7 + Rule 11 applied.
- **Defer phase-handler edge cases.** `controller_test.go` already covers the
  Pending/Creating/Active happy paths and several transitions via `Reconcile()`.
  The remaining genuine gaps (WFC-unbound→Creating, pendingTimedOut→recovery,
  suspend/Terminating finalizer ordering, the envtest suite) are larger and
  belong in a follow-up increment, not rushed into this one. This increment
  delivers three complete, correct, lint-clean files rather than nine partial ones.
- **No `phase_pending_test.go` yet.** The story lists 5 Pending tests, but 3
  Pending happy paths are already tested in controller_test.go and the
  isPVCStale/stale-PVC path in stale_pvc_test.go. Adding marginal duplicates has
  negative value; the real new edge cases (WFC, timeout) are deferred with this
  documented.
