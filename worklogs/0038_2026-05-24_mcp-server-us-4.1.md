### MCP Server (US-4.1) — Complete

**Architecture decision:** Workspace-centric tools (sandbox layer hidden). Aligns with frontend (AD5) and provides consistent mental model across all interfaces.

**Files created:**
- `pkg/mcp/client.go` — APIClient interface + HTTPClient (resolves workspace→sandbox internally)
- `pkg/mcp/server.go` — MCP server with 6 tools via mcp-go v0.54.0
- `pkg/mcp/server_test.go` — 18 unit tests
- `pkg/mcp/client_test.go` — 17 HTTP client tests (httptest)
- `pkg/mcp/integration_test.go` — Full workflow e2e via in-process MCP client
- `cmd/mcp/main.go` — Entrypoint (stdio default, --sse flag)

**Tools:**
| Tool | API Call |
|------|---------|
| `workspace_create` | POST /api/v1/workspaces |
| `workspace_activate` | POST /api/v1/workspaces/:id/activate |
| `workspace_stop` | POST /api/v1/workspaces/:id/suspend |
| `session_create` | POST /api/v1/sandboxes/:id/sessions (via resolveSandbox) |
| `session_message` | POST .../prompt → SSE /events → session.idle |
| `session_history` | GET .../sessions/:sid/message |

**Hardening:**
- Input validation: K8s name regex on all IDs (blocks path traversal)
- Message size limit: 1MB
- Response body limit: 10MB (prevents OOM)
- SSE accumulation cap: 50MB total, 1MB per line
- Error truncation: 512 chars max (prevents info leakage)
- Per-request timeout: 30s default
- Graceful SSE shutdown on SIGINT
- Startup warning on missing API key

**Test coverage:** 89.5% (unit + integration + e2e)

## Key Decisions

1. **Workspace-centric MCP tools** — sandbox is internal implementation detail everywhere (MCP, frontend, REST). Simpler mental model.
2. **Package at `pkg/mcp/`** not `api/internal/mcp/` — `cmd/mcp` (top-level binary) can't import `api/internal/`. Moved to `pkg/` for visibility.
3. **SSE timeout → fallback to history poll** — uses parent context (not timed-out SSE context) so fallback actually works.
4. **No retry logic in MCP server** — per MCP spec, callers decide retry policy.
