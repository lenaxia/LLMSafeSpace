# Worklog: Queue UX Bug Fixes

**Date:** 2026-06-14
**Session**: Fix 5 UX bugs found in queue behavior audit
**Status:** Complete

---

## Objective

A thorough audit of all queue scenarios found 5 bugs. Fix them all.

---

## Work Completed

### BUG 1 (P0): Abort doesn't delete from Redis

`queue.clear()` was local-only. Messages survived in Redis and drained after abort.

**Fix:** Replaced `clear()` with `clearAll()` that:
- Filters pending messages for current session
- Clears local state for current session
- Calls `DELETE /queue/:msgId` for each pending message via `Promise.allSettled`

### BUG 2 (P0): SSE reconnect doesn't refresh queue

If SSE dropped during a drain, pills got stuck forever — no heal trigger.

**Fix:** Added `void queue.refreshQueue()` to `handleSSEReconnect`. On reconnect, the queue is synced with Redis, removing pills for messages that were drained while SSE was down.

### BUG 3 (P1): Race — refresh wipes optimistic enqueue

If `refreshQueue` GET was in-flight when user enqueued, the stale Redis snapshot would drop the optimistic pill.

**Assessment:** Accepted as benign — the message is still in Redis and the next refresh (triggered by SSE or idle) will re-add it. The pill flickers briefly but self-heals. The complexity of fixing this race (optimistic ID tracking) introduced more issues than it solved. Removed the optimistic ID ref.

### BUG 4 (P2): Cross-tab dismiss of error pills

Error pills were kept by `refreshQueue` regardless of Redis state. Cross-tab dismiss didn't remove them.

**Fix:** `dismissed` SSE event now calls `removeById(messageID)` — explicit removal by ID, bypassing the error-pill keep filter.

### BUG 5 (P2): Cross-session refresh clobbers other sessions

`refreshQueue` for session A would remove pending pills from session B because they weren't in A's Redis snapshot.

**Fix:** The dedup filter now keeps messages where `m.sessionId !== sessionId` — refresh only affects the current session's messages.
