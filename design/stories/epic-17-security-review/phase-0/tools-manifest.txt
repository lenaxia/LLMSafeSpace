# Phase 0 / Epic 17 pentest — pinned tool versions.

# Format: <name> <version> <install-hint>
#
# Why pin versions: pentest findings must reproduce. A new kube-hunter
# release that adds a check would silently turn "no findings" into
# "real findings" without anything in our codebase changing. By pinning
# we make any tool-version change a deliberate decision recorded in this
# file.
#
# Sha256 sums are NOT pinned in this file because the upstream release
# pages publish them and rotating them belongs to a separate operator
# task. The expectation: install scripts in this directory verify
# downloaded artefacts against the upstream-published sha256.
#
# Bumps to this file MUST be accompanied by a worklog entry justifying
# the change.
