## Summary

`pkg/agent/opencode/client.go:60–71` contains `PushCredentials`, which iterates over a slice of providers calling `setAuth` (PUT `/auth/:providerID` with Basic Auth) for each one. It returns immediately on any error. On transient 5xx or network errors, this leaves the workspace with partially-written credentials in `auth.json`. Adding per-provider retry with exponential backoff to `setAuth` makes each credential push resilient to transient failures without re-pushing already-succeeded providers.

## Current Code

The actual code:

```go
func (c *Client) PushCredentials(ctx context.Context, providers []secrets.LLMProviderData) error {
    if len(providers) == 0 {
        return nil
    }

    for _, p := range providers {
        if err := c.setAuth(ctx, p); err != nil {
            return err
        }
    }
    return nil
}
```

And the per-provider helper (`client.go:146–179`):

```go
func (c *Client) setAuth(ctx context.Context, p secrets.LLMProviderData) error {
    payload := authPayload{
        Type: "api",
        Key:  p.APIKey,
    }
    if p.BaseURL != "" {
        payload.Metadata = map[string]string{"baseURL": p.BaseURL}
    }

    body, err := json.Marshal(payload)
    if err != nil {
        return fmt.Errorf("marshal auth payload for %s: %w", p.Provider, err)
    }

    url := c.baseURL + "/auth/" + p.Provider
    req, err := http.NewRequestWithContext(ctx, http.MethodPut, url, bytes.NewReader(body))
    if err != nil {
        return fmt.Errorf("create PUT request for %s: %w", p.Provider, err)
    }
    req.Header.Set("Content-Type", "application/json")
    req.SetBasicAuth(agentd.AuthUsername, c.password)

    resp, err := c.httpClient.Do(req)
    if err != nil {
        return fmt.Errorf("PUT /auth/%s: %w", p.Provider, err)
    }
    defer resp.Body.Close()
    _, _ = io.Copy(io.Discard, resp.Body)

    if resp.StatusCode >= 400 {
        return fmt.Errorf("PUT /auth/%s returned %d", p.Provider, resp.StatusCode)
    }
    return nil
}
```

**Problem:** A single 500 Internal Server Error or a transient network blip (DNS timeout, connection reset) on provider B causes the loop to stop. Provider A is already written to `auth.json` but provider B is not. The caller has no way to retry provider B without re-pushing provider A.

---

## Design Decisions

### Per-provider retry, not loop-level retry

Retry wraps each individual `setAuth` call, **not** the entire `PushCredentials` loop. If provider B's `setAuth` fails after 3 retries, already-written provider A stays in `auth.json`. The loop stops and returns the error. This is acceptable because:

- The caller (`reloadSecretsHandler` in `secrets.go:401`) logs a warning but does not fail the HTTP response — the reload still returns 200 with partial results.
- In-memory provider state is unaffected until `DisposeInstance` is called separately. The on-disk `auth.json` is correct for all providers that succeeded.
- Re-pushing already-succeeded providers is wasteful and adds latency.

### Partial failure semantics

If provider B fails after 3 retries, `PushCredentials` returns the error and stops the loop. Providers before B are already persisted in `auth.json`; providers after B are not attempted. This state is **self-healing**: the `PUT /auth/:providerID` endpoint is idempotent, so re-calling `/v1/reload-secrets` will re-push all providers (including already-succeeded ones) without adverse effects. Operators recovering from a partial failure should simply re-trigger the reload; no manual rollback or cleanup is required.

> **Note:** For true atomicity, `PushCredentials` would need a rollback mechanism (DELETE on succeeded providers), which is out of scope. The idempotent re-push path is the documented recovery mechanism.

### Latency impact

`PushCredentials` is called from `/v1/reload-secrets` via `StageCredentials` at `secrets.go:401`. This call occurs **after** `reloadMu.Unlock()` at `secrets.go:385` — the retry does **not** hold the process-wide reload mutex. Concurrent reloads are therefore not blocked during the retry window; only the HTTP response to the triggering request is delayed.

**True worst-case latency per provider:**

The `httpClient.Timeout` in `client.go` is **10 seconds** per attempt. With 3 attempts and delays of 1s and 2s between them:

```
attempt 1: up to 10s
delay 1:            + 1s
attempt 2:          + up to 10s
delay 2:            + 2s
attempt 3:          + up to 10s
                   ──────────────
worst case:          33s per provider
```

For a 2-provider reload: up to 66 seconds worst case. To reduce this, the implementation adds a **per-attempt context timeout** of 5 seconds (see Step 3 below), capping worst-case at:

```
attempt 1: up to 5s
delay 1:           + 1s
attempt 2:         + up to 5s
delay 2:           + 2s
attempt 3:         + up to 5s
                  ──────────────
worst case:         18s per provider  (vs. 33s without per-attempt cap)
```

### Jitter on backoff delays

To prevent thundering herd when multiple workspace pods experience simultaneous provider failures, each delay includes up to 500ms of random jitter:

```go
delay += time.Duration(rand.Intn(500)) * time.Millisecond
```

### Logging

The `pkg/agent/opencode` package has **zero logging today** — no `log` package import exists in `client.go`. The entire codebase uses `go.uber.org/zap`. This story adds a `logger *zap.Logger` field to `Client` and updates `NewClient` to accept an optional logger. All retry log lines use structured zap fields — **no `log.Printf`**.

### Observability

This story recommends adding a Prometheus counter `agent_setauth_retry_total{provider, attempt}` to make retry rates observable in production. This allows operators to detect providers that are consistently failing on first attempt and to distinguish transient blips from persistent degradation. The counter instrumentation is tracked as a follow-on task.

## Requirements

| Requirement | Detail |
|---|---|
| Max attempts | 3 per provider |
| Initial delay | 1 second (+ up to 500ms jitter) |
| Backoff multiplier | 2× (i.e., delays: ~1–1.5s, ~2–2.5s) |
| Per-attempt timeout | 5 seconds (context.WithTimeout inside retry loop) |
| Retryable errors | HTTP 5xx, network errors (DNS, timeout, connection refused/reset) |
| Non-retryable errors | HTTP 4xx (client errors — retrying won't help) |
| Logging | `c.logger.Warn(...)` with zap fields: provider, attempt number, delay, error |
| Context | Respect context cancellation between retries |
| Body replay | Request body is re-marshaled on each attempt (payload is small) |
| Retry level | Per-provider `setAuth` call, NOT wrapping the `PushCredentials` loop |
| Logger | `logger *zap.Logger` field on `Client`; `NewClient` accepts optional logger; default `zap.NewNop()` |

### Step 3 — Add a retry helper

Add an unexported `retryWithBackoff` function in `pkg/agent/opencode/client.go`. The helper is a method on `*Client` so it has access to `c.logger`:

```go
// retryWithBackoff calls fn up to maxAttempts times. Between each failed
// attempt it sleeps for initialDelay * 2^(attempt-1) plus up to 500ms of
// random jitter (to spread load when multiple providers fail simultaneously).
//
// fn receives the 1-based attempt number. If fn returns a *nonRetryableError
// the loop stops immediately and returns that error without waiting. Context
// cancellation is checked before each sleep.
func (c *Client) retryWithBackoff(ctx context.Context, maxAttempts int, initialDelay time.Duration, fn func(attempt int) error) error {
    var lastErr error
    for attempt := 1; attempt <= maxAttempts; attempt++ {
        lastErr = fn(attempt)
        if lastErr == nil {
            return nil
        }

        var nre *nonRetryableError
        if errors.As(lastErr, &nre) {
            return lastErr
        }

        if attempt == maxAttempts {
            break
        }

        delay := initialDelay*time.Duration(1<<(attempt-1)) +
            time.Duration(rand.Intn(500))*time.Millisecond

        c.logger.Warn("setAuth: attempt failed, retrying",
            zap.String("provider", ""),  // filled in by caller via closure
            zap.Int("attempt", attempt),
            zap.Duration("retrying_in", delay),
            zap.Error(lastErr),
        )

        // WARNING: The `zap.String("provider", "")` above is a placeholder.
        // Because retryWithBackoff's fn signature is `func(attempt int) error`,
        // the provider name is NOT accessible inside retryWithBackoff. Move the
        // retry warning log INTO the setAuth closure (where p.Provider is in
        // scope), not inside retryWithBackoff. retryWithBackoff should only log
        // generic retry info, or accept an optional `description string` parameter.
        //
        // Corrected approach — log inside the setAuth closure:
        //
        //   func (c *Client) setAuth(ctx context.Context, p secrets.LLMProviderData) error {
        //       return c.retryWithBackoff(ctx, 3, 1*time.Second, func(attempt int) error {
        //           err := c.doSetAuth(attemptCtx, p, attempt)
        //           if err != nil {
        //               var nre *nonRetryableError
        //               if !errors.As(err, &nre) && attempt < 3 {
        //                   c.logger.Warn("setAuth: attempt failed, retrying",
        //                       zap.String("provider", p.Provider),
        //                       zap.Int("attempt", attempt),
        //                       zap.Error(err),
        //                   )
        //               }
        //           }
        //           return err
        //       })
        //   }
        //
        // retryWithBackoff itself logs only generic info (attempt count, delay)
        // without a provider string, or accepts a `description string` parameter.

        select {
        case <-ctx.Done():
            return fmt.Errorf("setAuth: context cancelled during retry: %w", ctx.Err())
        case <-time.After(delay):
        }
    }
    return fmt.Errorf("setAuth: failed after %d attempts: %w", maxAttempts, lastErr)
}
```

### Step 4 — Rewrite `setAuth` to use `retryWithBackoff`

Replace the current `setAuth` implementation:

```go
func (c *Client) setAuth(ctx context.Context, p secrets.LLMProviderData) error {
    return c.retryWithBackoff(ctx, 3, 1*time.Second, func(attempt int) error {
        // Cap each individual HTTP attempt at 5s. The httpClient.Timeout is
        // 10s, which gives a worst-case of 33s for 3 attempts. Capping here
        // at 5s reduces that to ~18s (5s + 1s + 5s + 2s + 5s = 18s worst
        // case per provider) while still giving transient-slow endpoints
        // a fair window to respond.
        attemptCtx, cancel := context.WithTimeout(ctx, 5*time.Second)
        defer cancel()

        payload := authPayload{
            Type: "api",
            Key:  p.APIKey,
        }
        if p.BaseURL != "" {
            payload.Metadata = map[string]string{"baseURL": p.BaseURL}
        }

        body, err := json.Marshal(payload)
        if err != nil {
            // Marshal failures are deterministic — no point retrying.
            return &nonRetryableError{Provider: p.Provider, StatusCode: 0, Attempt: attempt}
        }

        url := c.baseURL + "/auth/" + p.Provider
        req, err := http.NewRequestWithContext(attemptCtx, http.MethodPut, url, bytes.NewReader(body))
        if err != nil {
            return fmt.Errorf("create PUT request for %s: %w", p.Provider, err)
        }
        req.Header.Set("Content-Type", "application/json")
        req.SetBasicAuth(agentd.AuthUsername, c.password)

        resp, err := c.httpClient.Do(req)
        if err != nil {
            return fmt.Errorf("PUT /auth/%s (attempt %d): network error: %w", p.Provider, attempt, err)
        }
        defer resp.Body.Close() //nolint:errcheck // best-effort drain
        _, _ = io.Copy(io.Discard, resp.Body)

        if resp.StatusCode >= 400 && resp.StatusCode < 500 {
            return &nonRetryableError{Provider: p.Provider, StatusCode: resp.StatusCode, Attempt: attempt}
        }
        if resp.StatusCode >= 500 {
            return fmt.Errorf("PUT /auth/%s (attempt %d): server error: HTTP %d", p.Provider, attempt, resp.StatusCode)
        }
        return nil
    })
}
```

Key changes from current implementation:
- Retry wraps each `setAuth` call, not the `PushCredentials` loop.
- Body is re-marshaled on each attempt (providers are small JSON payloads).
- 4xx returns `nonRetryableError` which stops retry immediately.
- 5xx and network errors are retryable.
- Per-attempt context caps each HTTP call at 5s rather than the full 10s client timeout.
- `PushCredentials` is unchanged — it still iterates providers and returns on first error.
