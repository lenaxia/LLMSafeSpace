# US-38.2: Decompose ProxyHandler

**Domain:** API Handlers
**Depends On:** None (US-38.9 moves extracted types to services/, but this story creates the interfaces)
**Priority:** High (foundational — largest architectural problem in the codebase)
**Estimated Effort:** Large (4-5 days)

---

## Summary

Decompose the 2323-line `ProxyHandler` god object (spanning five files: `proxy.go` (1623 lines), `proxy_input.go` (180 lines), `proxy_chat_enrichment.go` (62 lines), `session_parents.go` (162 lines), and `stream_user_events.go` (296 lines)) into focused services with single responsibilities. The refactored `ProxyHandler` becomes a thin dispatcher under 400 lines across all handler files.

### Known bugs in current code

1. **Double-release** (proxy.go:514-519): `checkAndAddActiveSession` failure path calls `releaseConnection` explicitly, then `defer` fires again. Entry deleted at count=0, permanently losing the slot.

2. **TOCTOU in terminal** (terminal.go:373-386): `acquireConnection` does atomic load of `globalConns`, then mutex-locked `globalConns.Add(1)`. Under concurrency, count exceeds `maxGlobalConns`.

3. **Unsynchronized writes** (proxy.go:156-161, 1072-1073, 1621-1622): `EnableSessionParentResolution`, `SetSessionIndex`, `SetAgentStateChecker` write without sync, goroutines read.

4. **Swallowed errors** (proxy.go:1503, 1506, 1542): `UpsertTitle`, `UpsertParent`, `UpsertContextUsed` use `_ =`.

5. **Dead code** (proxy.go:559-561, 777-846): `stripPatch` permanently disabled.

#### 2. ConnectionLimiter

**Location:** `api/internal/services/connection/limiter.go`
**Interface location:** `api/internal/interfaces/interfaces.go` (add `ConnectionLimiterService`)

```go
package connection

type Limiter struct {
    wsConns      map[string]int
    wsConnsMu    sync.Mutex
    maxPerWS     int
    globalConns  atomic.Int64
    maxGlobal    int
}

func NewLimiter(maxPerWorkspace, maxGlobal int) *Limiter

// Acquire reserves a connection slot for a workspace.
// Returns error if workspace limit is reached.
// Thread-safe: mutex-protected check-and-increment in one critical section.
func (l *Limiter) Acquire(workspaceID string) error

// Release returns a connection slot for a workspace.
// Idempotent: clamps to 0, never goes negative.
func (l *Limiter) Release(workspaceID string)

// AcquireGlobal reserves a global connection slot (for terminal WebSocket).
// Uses atomic CAS to avoid TOCTOU.
func (l *Limiter) AcquireGlobal() error

// ReleaseGlobal returns a global connection slot.
func (l *Limiter) ReleaseGlobal()

// AcquireBoth atomically reserves both a global slot and a per-workspace slot.
// Used by terminal connections that need both limits enforced simultaneously.
// Rolls back global acquire if workspace acquire fails.
func (l *Limiter) AcquireBoth(workspaceID string) error
```

**Bug fixes included:**
- `Acquire` is atomic: mutex lock + check + increment in one critical section (no separate check-then-add)
- `Release` is idempotent: clamps to 0, never goes negative
- `AcquireGlobal` uses `atomic.CompareAndSwap` loop — no TOCTOU
- `ReleaseGlobal` uses `atomic.AddInt64(-1)` with floor guard
- `AcquireBoth` provides atomic global + per-workspace reservation for terminal connections, preventing partial-acquire states where global slot is held but workspace slot is denied

#### 6. EventRouter

**Location:** `api/internal/services/event/router.go`
**Interface location:** `api/internal/interfaces/interfaces.go` (add `EventRouterService`)

> **Prerequisite step:** Move `WorkspaceSSEEvent` to `pkg/types/events.go` before defining the interface. The event package imports interfaces (for `LoggerInterface`), and interfaces cannot import the event package — this would be a circular import. `pkg/types` has no such constraint. Update all existing references to `WorkspaceSSEEvent` to use `types.WorkspaceSSEEvent`.

```go
package event

// WorkspaceSSEEvent is defined in pkg/types/events.go and imported here.
// It is NOT redefined in the event package.

type subscriber struct {
    ch          chan types.WorkspaceSSEEvent
    missedEvent atomic.Bool
    closed      atomic.Bool
    onDrop      func(eventType string)
}

type shard struct {
    mu       sync.Mutex
    userSubs map[string][]*subscriber
    wsSubs   map[string][]*subscriber
    replay   map[string]*replayBuffer
    wsOwner  map[string]string
}

type Router struct {
    shards [numShards]shard
    logger pkginterfaces.LoggerInterface
}

func NewRouter(logger pkginterfaces.LoggerInterface) *Router

// Publish sends an event to all subscribers of a workspace.
// Assigns event ID and appends to replay buffer.
func (r *Router) Publish(ctx context.Context, workspaceID string, event types.WorkspaceSSEEvent)

// PublishUser sends an event to all subscribers of a user.
// Assigns event ID and appends to replay buffer.
func (r *Router) PublishUser(ctx context.Context, userID string, event types.WorkspaceSSEEvent)

// SubscribeWorkspace returns a receive channel for workspace events and an unsubscribe func.
// If subscriber limit is reached, returns error.
func (r *Router) SubscribeWorkspace(workspaceID string) (<-chan types.WorkspaceSSEEvent, func(), error)

// SubscribeUser returns a receive channel for user events and an unsubscribe func.
// Includes replay buffer for late subscribers.
func (r *Router) SubscribeUser(userID string) (<-chan types.WorkspaceSSEEvent, func(), error)

// Replay returns buffered events for a user since lastID.
func (r *Router) Replay(userID string, lastID uint64) ([]types.WorkspaceSSEEvent, bool)
```

**Interface in `interfaces.go`** (channel-based, uses only exported types from `pkg/types`):

```go
// EventRouterService uses only exported types from pkg/types so this interface
// can be defined without importing the event package (avoiding circular imports).
type EventRouterService interface {
    Publish(ctx context.Context, workspaceID string, event types.WorkspaceSSEEvent)
    PublishUser(ctx context.Context, userID string, event types.WorkspaceSSEEvent)
    SubscribeWorkspace(workspaceID string) (<-chan types.WorkspaceSSEEvent, func(), error)
    SubscribeUser(userID string) (<-chan types.WorkspaceSSEEvent, func(), error)
    Replay(userID string, lastID uint64) ([]types.WorkspaceSSEEvent, bool)
}
```

**This replaces both `WorkspaceEventBroker` and `UserEventBroker`** with a single unified implementation that:

- Uses `types.WorkspaceSSEEvent` as the concrete event type — no phantom `Event`/`UserEvent` types, no `interface{}`
- Uses sharded `sync.Mutex` (one per shard) that protects both subscriptions AND replay buffer, following the existing `UserEventBroker` pattern — this prevents replay buffer races because the same lock serializes both subscription mutations and replay buffer writes within each shard
- Has subscriber limits (like `UserEventBroker`)
- Has replay buffer for all event types (like `UserEventBroker`)
- Reuses the existing `subscriber` struct with `missedEvent`/resync behavior
- Returns `(<-chan types.WorkspaceSSEEvent, func(), error)` from Subscribe methods so callers receive events via a plain channel — no unexported types cross package boundaries

## Other Options Considered

### Option A: Incremental extraction (one service at a time) — SELECTED

Extract services one at a time, each as a self-contained PR. Lower risk, each step is fully testable. ProxyHandler shrinks incrementally.

**Pros:** Each step is reviewable, testable, and revertable independently. No big-bang risk.
**Cons:** More PRs, more reviews, longer total elapsed time.

### Option B: Big-bang rewrite

Rewrite ProxyHandler from scratch using the new services.

**Pros:** Faster total time, single review.
**Cons:** 2323 lines of concurrent code (spanning five files) with subtle invariants. High risk of regressions. Hard to review. Discarded.

### Option C: Leave as-is, add documentation

Add comments, break into sections with `// --- Section: Password Caching ---`.

**Pros:** Zero risk.
**Cons:** Does not fix the architectural problem. Handler still grows. Testing still requires wiring all 27 fields. Discarded.

### Option D: Split into multiple handlers (not services)

Create `PasswordHandler`, `SessionHandler`, `Event handler` etc.

**Pros:** Splits the code.
**Cons:** Handlers should be thin dispatchers, not logic containers. The logic belongs in services. Would create handlers that make K8s API calls, DB writes, and HTTP dispatch — same problem, different struct names. Discarded.

## Weaknesses of Selected Design

### 1. More interfaces to mock in tests

**Weakness:** Each extracted service needs a mock. Currently tests mock the monolithic ProxyHandler's dependencies.
**Mitigation:** Small interfaces (2-4 methods) are easy to mock. Use `testify/mock` generation. The interfaces are already defined in `interfaces.go`.

### 2. More constructor parameters in app.go

**Weakness:** `app.go` wiring becomes longer with more services to initialize.
**Mitigation:** Use constructor structs (e.g., `DispatcherOptions`) to keep function signatures reasonable. Group related services.

### 3. Potential for circular dependencies

**Weakness:** If `EventRouter` needs `PodDispatcher` (or vice versa), we get import cycles.
**Mitigation:** Services are stateless and directional: PodDispatcher → (no dependencies), EventRouter → (no dependencies on other new services), ConnectionLimiter → (no dependencies), SessionManager → DatabaseService (already exists). No cycles possible.

### 4. SSE event format must be preserved exactly

**Weakness:** Moving event routing into `EventRouter` changes the code path for SSE events. Any format change breaks frontend.
**Mitigation:** Extract `emitNormalizedInputEvent` and `onRawEvent` verbatim — copy the function bodies, don't rewrite them. Integration tests compare event payloads before and after.

### 5. SessionManager has mixed responsibilities

**Weakness:** SessionManager combines session tracking (in-memory state) with DB persistence (title, parent, context-used) and workspace config caching. These could be separate services.
**Mitigation:** All three concerns are session-scoped and always used together. Splitting would create a trivially thin persistence service and a trivially thin config cache. The combined interface extends the existing `SessionIndexService`, keeping the API surface small.

## Failure Modes

### FM-1: SSE events dropped during migration

**Cause:** EventRouter's Publish method has a subtle difference from the inline code.
**Detection:** Integration test: subscribe to workspace, send 100 events, count received.
**Recovery:** Revert to inline code; fix EventRouter.

### FM-2: Password cache miss causes latency spike

**Cause:** Cache TTL too short or invalidation too aggressive.
**Detection:** Prometheus metric on password resolution latency. Alert on p99 > 1s.
**Recovery:** Increase TTL. The cache is advisory — misses just cause a K8s API call.

### FM-3: Connection limiter deadlock

**Cause:** `Acquire` holds mutex while waiting for something.
**Detection:** Goroutine dump shows goroutines blocked on `wsConnsMu`.
**Recovery:** The limiter does not wait — it immediately returns error on limit reached. Deadlock is impossible by design.

### FM-4: Subscription leak

**Cause:** Subscriber forgets to call unsubscribe function.
**Detection:** `wsSubs` map grows without bound. Prometheus metric on subscriber count.
**Recovery:** Stale subscriber detection: if channel is full for >30s, force-unsubscribe.

### FM-5: SessionManager DB connection pool exhaustion

**Cause:** Too many concurrent PersistTitle calls.
**Detection:** DB connection pool metrics. Alert on pool utilization >80%.
**Recovery:** These are already bounded by the connection limiter upstream.

   ```go
   func (l *Limiter) AcquireGlobal() error {
       for {
           current := l.globalConns.Load()
           if current >= int64(l.maxGlobal) {
               return ErrGlobalLimitReached
           }
           if l.globalConns.CompareAndSwap(current, current+1) {
               return nil
           }
       }
   }
   ```
5. Add `AcquireBoth` for terminal connections needing atomic global + per-workspace:
   ```go
   func (l *Limiter) AcquireBoth(workspaceID string) error {
       if err := l.AcquireGlobal(); err != nil {
           return err
       }
       if err := l.Acquire(workspaceID); err != nil {
           l.ReleaseGlobal()
           return err
       }
       return nil
   }
   ```
