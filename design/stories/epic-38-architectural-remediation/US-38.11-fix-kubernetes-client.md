## Summary

The `pkg/kubernetes/` package (3 files: `client.go`, `client_crds.go`, `informers.go`) has six correctness and reliability issues: per-call typed client allocation, swallowed errors, hardcoded `context.TODO()`, duplicate informer creation, unsafe leader election, and a second `RunOrDie` in the controller. This story fixes all six.

### Blast radius

- `LlmsafespaceV1()` has **34 production call sites** across `api/internal/handlers/` and `api/internal/services/workspace/`, plus **2 test call sites** in `mocks/kubernetes/`. Changing its return signature touches every one.
- The 14 CRUD methods in `client_crds.go` each need `ctx context.Context` added. Every caller of each method must be updated. This is the bulk of the work — estimated **60+ call sites** across production and test code.
- The interface in `pkg/interfaces/kubernetes.go` is implemented by `pkg/kubernetes/client.go` and `mocks/kubernetes/mocks.go`. Both must be updated simultaneously.
- Changing `RuntimeEnvironments(namespace string)` to `RuntimeEnvironments()` (Fix 4) affects: `informers.go` (1 call site), `client_crds.go` (1 definition), `pkg/interfaces/kubernetes.go` (1 interface definition), and any mock implementations in `mocks/kubernetes/mocks.go`.

### Recommended split into sub-stories

Given the scope, consider splitting into **2 PRs**:

1. **PR 1 (Fix 1 + Fix 2):** Lazy-init `LLMSafespaceV1Client` with error return. This changes the `LlmsafespaceV1()` signature and all 36 call sites. ~1 day.
2. **PR 2 (Fix 3):** Replace `context.TODO()` with context parameters in all 14 CRUD methods. This changes all CRUD method signatures and their callers. ~1–2 days.

Fixes 4–6 are independent and can go in either PR or their own.

## Fix 1 — Lazy-Init Typed Client (`client.go:206–213`)

### Problem

`LlmsafespaceV1()` creates a new `*LLMSafespaceV1Client` (which internally creates a new `rest.Interface`) on every call. This allocates memory, opens connections, and discards the client after one use. For a hot path (CRD operations), this is wasteful.

### Current Code

```go
func (c *Client) LlmsafespaceV1() interfaces.LLMSafespaceV1Interface {
    client, err := newLLMSafespaceV1Client(c.restConfig)
    if err != nil {
        c.logger.Error("failed to construct LlmsafespaceV1 REST client", err)
        return nil
    }
    return client
}
```

### Fix

Cache the `*LLMSafespaceV1Client` (the typed client), not the raw `rest.Interface`. The return type changes to `(interfaces.LLMSafespaceV1Interface, error)`:

```go
import "sync"

type Client struct {
    // ... existing fields ...
    v1Once   sync.Once
    v1Client *LLMSafespaceV1Client
    v1Err    error
}

// sync.Once caches both success and failure. If init fails (which should
// not happen given restConfig was validated at startup in New()), a process
// restart is required to retry — Once will not re-attempt the failed init.
func (c *Client) LlmsafespaceV1() (interfaces.LLMSafespaceV1Interface, error) {
    c.v1Once.Do(func() {
        c.v1Client, c.v1Err = newLLMSafespaceV1Client(c.restConfig)
    })
    if c.v1Err != nil {
        return nil, c.v1Err
    }
    return c.v1Client, nil
}
```

**Important:** This changes the return signature from `interfaces.LLMSafespaceV1Interface` to `(interfaces.LLMSafespaceV1Interface, error)`. All **34+ callers** must be updated (see Fix 2).

## Fix 3 — Replace `context.TODO()` with Context Parameters (`client_crds.go`)

### Problem

`client_crds.go` uses `context.TODO()` in **14 locations** across all CRUD methods (7 RuntimeEnvironment + 7 Workspace). These calls perform Kubernetes API server requests that can block indefinitely. Without a proper context:
- No timeout enforcement.
- No cancellation on shutdown.
- No trace propagation for observability.

### Complete list of 14 methods

**RuntimeEnvironment (7 methods):**

| Method | Line | Operation |
|---|---|---|
| `runtimeEnvironments.Create` | 106 | Post |
| `runtimeEnvironments.Update` | 118 | Put |
| `runtimeEnvironments.UpdateStatus` | 131 | Put (status sub-resource) |
| `runtimeEnvironments.Delete` | 142 | Delete |
| `runtimeEnvironments.Get` | 153 | Get |
| `runtimeEnvironments.List` | 164 | Get (list) |
| `runtimeEnvironments.Watch` | 175 | Watch |

**Workspace (7 methods):**

| Method | Line | Operation |
|---|---|---|
| `workspaces.Create` | 189 | Post |
| `workspaces.Update` | 201 | Put |
| `workspaces.UpdateStatus` | 214 | Put (status sub-resource) |
| `workspaces.Delete` | 225 | Delete |
| `workspaces.Get` | 236 | Get |
| `workspaces.List` | 247 | Get (list) |
| `workspaces.Watch` | 258 | Watch |

### Fix

For every method listed above, add `ctx context.Context` as the first parameter. Replace `context.TODO()` with `ctx`.

**Example transformation (RuntimeEnvironment.Create):**

Before:
```go
func (r *runtimeEnvironments) Create(runtimeEnv *v1.RuntimeEnvironment) (*v1.RuntimeEnvironment, error) {
    result := &v1.RuntimeEnvironment{}
    err := r.client.Post().
        Namespace(r.ns).
        Resource("runtimeenvironments").
        Body(runtimeEnv).
        Do(context.TODO()).
        Into(result)
    return result, err
}
```

After:
```go
func (r *runtimeEnvironments) Create(ctx context.Context, runtimeEnv *v1.RuntimeEnvironment) (*v1.RuntimeEnvironment, error) {
    result := &v1.RuntimeEnvironment{}
    err := r.client.Post().
        Namespace(r.ns).
        Resource("runtimeenvironments").
        Body(runtimeEnv).
        Do(ctx).
        Into(result)
    return result, err
}
```

Apply the same transformation to all 14 methods.

### Impact on `informers.go` — CRITICAL BUILD BREAK without this fix

After Fix 3 adds `ctx context.Context` to `List` and `Watch`, the `InformerFactory`'s `ListFunc`/`WatchFunc` closures in `informers.go` call `.List(options)` and `.Watch(options)` without a context. These closures **cannot compile** after Fix 3 because the signatures no longer match.

The `cache.ListWatch` API requires:
```go
ListFunc:  func(options metav1.ListOptions) (runtime.Object, error)
WatchFunc: func(options metav1.ListOptions) (watch.Interface, error)
```

These function types do **not** accept a context parameter — they are a fixed API from `k8s.io/client-go/tools/cache`. They cannot be changed to accept context.

**Fix:** Store a `context.Context` on `InformerFactory` and use it inside the closures:

```go
type InformerFactory struct {
    client        interfaces.LLMSafespaceV1Interface
    defaultResync time.Duration
    namespace     string
    ctx           context.Context  // ADD: long-lived context for informer List/Watch calls
    // ... caching fields from Fix 5 ...
}

func NewInformerFactory(client interfaces.LLMSafespaceV1Interface, defaultResync time.Duration, namespace string) *InformerFactory {
    return &InformerFactory{
        client:        client,
        defaultResync: defaultResync,
        namespace:     namespace,
        ctx:           context.Background(), // long-lived background context for informers
    }
}
```

> **Struct definition conflict note:** The `InformerFactory` struct shown above (in Fix 3) is abbreviated — it shows only the `client`, `defaultResync`, `namespace`, and `ctx` fields to illustrate context propagation. Fix 5 (informer caching section) shows the **complete** struct with the additional `mu`, `started`, `runtimeEnvInformer`, and `workspaceInformer` fields. Apply the COMPLETE struct from Fix 5 only — do not apply Fix 3's abbreviated struct definition. Fix 3's struct is illustrative only.

Then inside closures, use `f.ctx`:

```go
ListFunc: func(options metav1.ListOptions) (runtime.Object, error) {
    return f.client.RuntimeEnvironments().List(f.ctx, options)
},
WatchFunc: func(options metav1.ListOptions) (watch.Interface, error) {
    return f.client.RuntimeEnvironments().Watch(f.ctx, options)
},
```

> **Design note:** The `ListFunc`/`WatchFunc` closures use `f.ctx` (a stored `context.Background()`) because the `cache.ListWatch` API does not support context parameters. This is the **only** acceptable remaining use of `context.Background()` in `pkg/kubernetes/` after Fix 3. All other `context.Background()` calls must be eliminated.

## Fix 4 — Fix Namespace for Cluster-Scoped Resources (`client_crds.go:87–93`)

### Problem

`RuntimeEnvironment` is a cluster-scoped CRD (no namespace in its spec), but the code passes a namespace to the client builder via `r.ns`. This either silently fails or causes incorrect behavior with the Kubernetes API server.

Additionally, the `namespace string` parameter on `RuntimeEnvironments(namespace string)` is meaningless for a cluster-scoped resource. After removing `.Namespace(r.ns)` from the 7 RuntimeEnvironment methods, `namespace` serves no purpose and should be removed from the signature entirely to prevent callers from passing a namespace that is silently ignored.

### Current Code

```go
func (c *LLMSafespaceV1Client) RuntimeEnvironments(namespace string) interfaces.RuntimeEnvironmentInterface {
    return &runtimeEnvironments{client: c.restClient, ns: namespace}
}
```

And all 7 RuntimeEnvironment methods call `.Namespace(r.ns)`.

### Fix

**Step 1:** Remove the `namespace` parameter from `RuntimeEnvironments`:

```go
// RuntimeEnvironments returns an interface for RuntimeEnvironment operations.
// RuntimeEnvironment is a cluster-scoped CRD — namespace is not applicable
// and is not sent in any REST request.
func (c *LLMSafespaceV1Client) RuntimeEnvironments() interfaces.RuntimeEnvironmentInterface {
    return &runtimeEnvironments{client: c.restClient}
}
```

Update `runtimeEnvironments` struct to remove `ns string` field.

**Step 2:** Remove `.Namespace(r.ns)` from all 7 RuntimeEnvironment methods:

```go
func (r *runtimeEnvironments) Create(ctx context.Context, runtimeEnv *v1.RuntimeEnvironment) (*v1.RuntimeEnvironment, error) {
    result := &v1.RuntimeEnvironment{}
    err := r.client.Post().
        // No .Namespace() — RuntimeEnvironment is cluster-scoped
        Resource("runtimeenvironments").
        Body(runtimeEnv).
        Do(ctx).
        Into(result)
    return result, err
}
```

**Step 3:** Update the interface in `pkg/interfaces/kubernetes.go`:

```go
type LLMSafespaceV1Interface interface {
    RuntimeEnvironments() RuntimeEnvironmentInterface  // no namespace parameter
    Workspaces(namespace string) WorkspaceInterface    // namespace kept — Workspace is namespaced
}
```

**Affected call sites for `RuntimeEnvironments(...)` → `RuntimeEnvironments()`:**
- `client_crds.go`: 1 definition (updated above)
- `pkg/interfaces/kubernetes.go`: 1 interface definition (updated above)
- `informers.go`: 2 lines — `f.client.RuntimeEnvironments(f.namespace)` appears in both `ListFunc` (line 36) and `WatchFunc` (line 39) inside `RuntimeEnvironmentInformer()`; both become `f.client.RuntimeEnvironments()`

## Fix 5 — Cache Informer Instances (`informers.go:64–71`)

### Problem

`StartInformers` creates new informer instances every time it's called. This is the **custom `InformerFactory`** in `informers.go` (not the Kubernetes shared informer factory). Each call creates new `cache.NewSharedIndexInformer` instances. If called more than once, duplicate informers run concurrently, causing duplicate event handling and resource leaks.

### Current Code

```go
func (f *InformerFactory) StartInformers(stopCh <-chan struct{}) {
    informers := []cache.SharedIndexInformer{
        f.RuntimeEnvironmentInformer(),
        f.WorkspaceInformer(),
    }
    for _, informer := range informers {
        go informer.Run(stopCh)
    }
}
```

### Fix

Cache informer instances and guard against double-start. Note that `StartInformers` calls the accessor methods (`f.RuntimeEnvironmentInformer()`, `f.WorkspaceInformer()`) to **ensure lazy initialization before calling `Run`** — it does not access `f.runtimeEnvInformer` directly. This preserves the lazy-init semantics:

```go
import "sync"

type InformerFactory struct {
    client        interfaces.LLMSafespaceV1Interface
    defaultResync time.Duration
    namespace     string
    ctx           context.Context // long-lived context for ListFunc/WatchFunc closures (see Fix 3 note)

    mu                 sync.Mutex
    started            bool
    runtimeEnvInformer cache.SharedIndexInformer
    workspaceInformer  cache.SharedIndexInformer
}

func NewInformerFactory(client interfaces.LLMSafespaceV1Interface, defaultResync time.Duration, namespace string) *InformerFactory {
    return &InformerFactory{
        client:        client,
        defaultResync: defaultResync,
        namespace:     namespace,
        ctx:           context.Background(),
    }
}

func (f *InformerFactory) RuntimeEnvironmentInformer() cache.SharedIndexInformer {
    f.mu.Lock()
    defer f.mu.Unlock()
    if f.runtimeEnvInformer == nil {
        f.runtimeEnvInformer = cache.NewSharedIndexInformer(
            &cache.ListWatch{
                // f.ctx is a stored context.Background() because the cache.ListWatch
                // API requires func(metav1.ListOptions) signatures that do not accept
                // context. This is the only acceptable remaining use of context.Background()
                // in pkg/kubernetes/ after Fix 3.
                ListFunc: func(options metav1.ListOptions) (runtime.Object, error) {
                    return f.client.RuntimeEnvironments().List(f.ctx, options)
                },
                WatchFunc: func(options metav1.ListOptions) (watch.Interface, error) {
                    return f.client.RuntimeEnvironments().Watch(f.ctx, options)
                },
            },
            &v1.RuntimeEnvironment{},
            f.defaultResync,
            cache.Indexers{cache.NamespaceIndex: cache.MetaNamespaceIndexFunc},
        )
    }
    return f.runtimeEnvInformer
}

func (f *InformerFactory) WorkspaceInformer() cache.SharedIndexInformer {
    f.mu.Lock()
    defer f.mu.Unlock()
    if f.workspaceInformer == nil {
        f.workspaceInformer = cache.NewSharedIndexInformer(
            &cache.ListWatch{
                ListFunc: func(options metav1.ListOptions) (runtime.Object, error) {
                    return f.client.Workspaces(f.namespace).List(f.ctx, options)
                },
                WatchFunc: func(options metav1.ListOptions) (watch.Interface, error) {
                    return f.client.Workspaces(f.namespace).Watch(f.ctx, options)
                },
            },
            &v1.Workspace{},
            f.defaultResync,
            cache.Indexers{cache.NamespaceIndex: cache.MetaNamespaceIndexFunc},
        )
    }
    return f.workspaceInformer
}

func (f *InformerFactory) StartInformers(stopCh <-chan struct{}) {
    f.mu.Lock()
    defer f.mu.Unlock()

    if f.started {
        return
    }
    f.started = true

    // Call accessor methods to ensure lazy initialization before Run.
    // Do NOT access f.runtimeEnvInformer / f.workspaceInformer directly here —
    // the accessors hold f.mu and initialize if nil. Since we already hold f.mu,
    // initialize inline to avoid deadlock:
    if f.runtimeEnvInformer == nil {
        // Unlock temporarily is not possible here — call the init path directly
        // (the nil check in the accessor is the same logic):
        f.runtimeEnvInformer = f.newRuntimeEnvInformerLocked()
    }
    if f.workspaceInformer == nil {
        f.workspaceInformer = f.newWorkspaceInformerLocked()
    }

    go f.runtimeEnvInformer.Run(stopCh)
    go f.workspaceInformer.Run(stopCh)
}
```

> **Implementation note:** Because `StartInformers` already holds `f.mu`, it cannot call `RuntimeEnvironmentInformer()` / `WorkspaceInformer()` (which also try to acquire `f.mu`) without deadlocking. The fix extracts the informer construction into `newRuntimeEnvInformerLocked()` / `newWorkspaceInformerLocked()` helper functions (called under `f.mu`), shared by both the public accessor methods and `StartInformers`. Alternatively, split the lock acquisition from the construction.

The bodies of these helpers are:

```go
// newRuntimeEnvInformerLocked initializes the RuntimeEnvironment informer.
// MUST be called with f.mu held.
func (f *InformerFactory) newRuntimeEnvInformerLocked() cache.SharedIndexInformer {
    return cache.NewSharedIndexInformer(
        &cache.ListWatch{
            ListFunc: func(opts metav1.ListOptions) (runtime.Object, error) {
                return f.client.RuntimeEnvironments().List(f.ctx, opts)
            },
            WatchFunc: func(opts metav1.ListOptions) (watch.Interface, error) {
                return f.client.RuntimeEnvironments().Watch(f.ctx, opts)
            },
        },
        &v1.RuntimeEnvironment{},
        f.defaultResync,
        cache.Indexers{},
    )
}

// newWorkspaceInformerLocked initializes the Workspace informer.
// MUST be called with f.mu held.
func (f *InformerFactory) newWorkspaceInformerLocked() cache.SharedIndexInformer {
    return cache.NewSharedIndexInformer(
        &cache.ListWatch{
            ListFunc: func(opts metav1.ListOptions) (runtime.Object, error) {
                return f.client.Workspaces(f.namespace).List(f.ctx, opts)
            },
            WatchFunc: func(opts metav1.ListOptions) (watch.Interface, error) {
                return f.client.Workspaces(f.namespace).Watch(f.ctx, opts)
            },
        },
        &v1.Workspace{},
        f.defaultResync,
        cache.Indexers{},
    )
}
```

The public accessor methods (`RuntimeEnvironmentInformer`, `WorkspaceInformer`) delegate to these helpers after acquiring `f.mu` and checking for nil. `StartInformers` also calls them directly (already holding `f.mu`) to initialize before calling `Run`.

Key properties:
- `sync.Mutex` prevents concurrent calls from creating duplicate informers.
- `started` flag makes subsequent `StartInformers` calls a no-op.
- Informer instances are cached (created once by the accessor or `StartInformers`).

### sync.Once vs sync.Mutex

This fix uses `sync.Mutex` with an explicit `nil` check rather than `sync.Once` because multiple independent informer instances are managed. If you choose `sync.Once` for individual informers, note: **`sync.Once` caches both success and failure.** If informer construction ever fails (which should not happen given a validated `restConfig`), a process restart is required to retry — the Once will not re-attempt the failed initialization. This behavior is not a practical concern here, but should be documented to prevent future confusion.

## Fix 6 — Replace `RunOrDie` with `NewLeaderElector` (`client.go:157`)

### Problem

`leaderelection.RunOrDie` calls `log.Fatal` on failure, which calls `os.Exit(1)` and bypasses all defer statements, graceful shutdown, and cleanup. This can leave the process in an inconsistent state.

**Note:** There is a second `RunOrDie` at `controller/internal/common/leader_election.go:70` in the controller binary. That call is being addressed separately in US-38.7 (which removes the controller's leader election entirely). This story only fixes the one in `pkg/kubernetes/client.go:157`.

### Current Code

```go
func (c *Client) runLeaderElection() {
    lock := &resourcelock.LeaseLock{...}
    leaderelection.RunOrDie(context.Background(), leaderelection.LeaderElectionConfig{
        Lock:            lock,
        ReleaseOnCancel: true,
        LeaseDuration:   c.config.LeaderElection.LeaseDuration,
        RenewDeadline:   c.config.LeaderElection.RenewDeadline,
        RetryPeriod:     c.config.LeaderElection.RetryPeriod,
        Callbacks: leaderelection.LeaderCallbacks{
            OnStartedLeading: func(ctx context.Context) { c.logger.Info("Started leading") },
            OnStoppedLeading: func() { c.logger.Info("Stopped leading") },
            OnNewLeader: func(identity string) { c.logger.Info("New leader elected", "leader", identity) },
        },
    })
}
```

### Fix

```go
func (c *Client) runLeaderElection(ctx context.Context) error {
    lock := &resourcelock.LeaseLock{...}

    elector, err := leaderelection.NewLeaderElector(leaderelection.LeaderElectionConfig{
        Lock:            lock,
        ReleaseOnCancel: true,
        LeaseDuration:   c.config.LeaderElection.LeaseDuration,
        RenewDeadline:   c.config.LeaderElection.RenewDeadline,
        RetryPeriod:     c.config.LeaderElection.RetryPeriod,
        Callbacks: leaderelection.LeaderCallbacks{
            OnStartedLeading: func(ctx context.Context) { c.logger.Info("Started leading") },
            OnStoppedLeading: func() { c.logger.Info("Stopped leading") },
            OnNewLeader: func(identity string) { c.logger.Info("New leader elected", "leader", identity) },
        },
    })
    if err != nil {
        return fmt.Errorf("create leader elector: %w", err)
    }

    go elector.Run(ctx)
    return nil
}
```

Key changes:
- `NewLeaderElector` returns an error instead of calling `log.Fatal`.
- `elector.Run(ctx)` is called in a goroutine so the caller can continue.
- Method now accepts a `ctx context.Context` parameter (replacing `context.Background()`).
- Returns `error` so callers can handle initialization failure.
