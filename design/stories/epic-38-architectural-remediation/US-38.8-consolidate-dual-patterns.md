## Summary

The API handler layer contains two active "dual patterns" ‚Äî places where the codebase uses an old approach in some files and a new, better approach in others. This story migrates all old-pattern call sites to the new pattern and removes the old pattern entirely. Two additional dual patterns (leader election, condition helpers) are already resolved by US-38.7 and are listed here only for completeness.

## Weaknesses / Pre-Migration Notes

Before beginning migration, be aware of the following subtleties confirmed by reading the actual source:

- **`extractAuth` does NOT write HTTP responses.** Read `api/internal/handlers/secrets.go:855‚Äì867`. The function signature is `func extractAuth(c *gin.Context) (userID, sessionID string)`. It performs comma-ok type assertions on `c.Get("userID")` and `c.Get("sessionID")`, and returns empty strings on failure. It does **not** call `c.JSON(...)` or any other response-writing function. Every caller in `secrets.go` (e.g. `CreateSecret:147‚Äì150`, `ListSecrets:170‚Äì172`) explicitly writes its own `c.JSON(http.StatusUnauthorized, ...)` after checking for an empty `userID`. The migration template must preserve this behavior ‚Äî see Step 3 below.

- **`user_provider_credentials.go:75‚Äì76` requires both return values.** Line 75 is `c.GetString("userID")` and line 76 is `c.GetString("sessionID")` ‚Äî both on the `Create` handler. When migrating this site, use both return values of `extractAuth`: `userID, sessionID := extractAuth(c)`. The `sessionID` is passed to `h.keys.GetDEK` at line 95.

- **`prepareSecretsLegacy` does not simply nil-check and return.** Read `pkg/secrets/injection.go:223‚Äì270`. The function verifies workspace ownership, fetches bindings, gets the DEK, decrypts each secret, and returns the JSON payload. The nil check that gates the legacy/new path split is in the outer `PrepareSecretsForInjection` at line 43. This is important context for the Pattern 4 removal scope.

## Pattern 1 ‚Äî Auth Extraction

### Problem

Some handler methods extract the authenticated user ID with a bare `c.GetString("userID")` call, which silently returns an empty string if the key is missing or if a non-string was stored under that key. Other handlers use the safer `extractAuth()` helper that performs a comma-ok type assertion on `c.Get(...)` and returns both the userID and sessionID on success, or empty strings on failure.

### Current State

**Old pattern** (bare `GetString`, no comma-ok type assertion) ‚Äî **10 `c.GetString("userID")` call sites + 1 `c.GetString("sessionID")` call site (line 76, migrated together with line 75 in Step 3):**
- `api/internal/handlers/settings.go:85`
- `api/internal/handlers/settings.go:120`
- `api/internal/handlers/terminal.go:139`
- `api/internal/handlers/user_provider_credentials.go:75` *(also reads `c.GetString("sessionID")` at line 76 ‚Äî migrate both)*
- `api/internal/handlers/user_provider_credentials.go:172`
- `api/internal/handlers/user_provider_credentials.go:198`
- `api/internal/handlers/user_provider_credentials.go:222`
- `api/internal/handlers/user_provider_credentials.go:249`
- `api/internal/handlers/user_provider_credentials.go:285`
- `api/internal/handlers/user_provider_credentials.go:326`

**New pattern** (`extractAuth()` with comma-ok, empty-string check with explicit 401):
- `api/internal/handlers/secrets.go:855‚Äì867` ‚Äî reference implementation
- `api/internal/handlers/secrets.go:145‚Äì150` ‚Äî reference call site (`CreateSecret`)

### Reference Implementation

Read `api/internal/handlers/secrets.go:855‚Äì867`. The actual implementation is:

```go
// extractAuth gets userID and sessionID (jti) from the Gin context.
// Both values are type-asserted with the comma-ok form so a malformed
// context (e.g. middleware put a non-string under the key) produces an
// empty result rather than a goroutine panic that takes down the
// request. Empty userID is treated as unauthenticated by every caller.
func extractAuth(c *gin.Context) (userID, sessionID string) {
    if uid, exists := c.Get("userID"); exists {
        if s, ok := uid.(string); ok {
            userID = s
        }
    }
    if sid, exists := c.Get("sessionID"); exists {
        if s, ok := sid.(string); ok {
            sessionID = s
        }
    }
    return userID, sessionID
}
```

**Critical:** `extractAuth` returns `(userID, sessionID string)` and does **not** write any HTTP response. It only returns empty strings when auth is missing or malformed. The caller is entirely responsible for writing the 401 response. This is demonstrated by every call site in `secrets.go`, for example `CreateSecret` at lines 146‚Äì150:

```go
userID, sessionID := extractAuth(c)
if userID == "" {
    c.JSON(http.StatusUnauthorized, gin.H{"error": "authentication required"})
    return
}
```

### Migration Plan

#### Step 1 ‚Äî Verify `extractAuth` is accessible from all three files

`extractAuth` is an unexported package-level function defined in `secrets.go` within the `handlers` package. It is already accessible to `settings.go`, `terminal.go`, and `user_provider_credentials.go` without any changes needed.

```bash
grep -n 'func extractAuth' api/internal/handlers/secrets.go
```

Expected: `855:func extractAuth(c *gin.Context) (userID, sessionID string)`

#### Step 2 ‚Äî Understand the correct migration template

There are two migration templates depending on whether the call site already has a guard.

**Template A ‚Äî call site HAS an existing empty-string guard (4 sites):**
- `terminal.go:139`
- `settings.go:85`
- `settings.go:120`
- `user_provider_credentials.go:75` (also has `sessionID` ‚Äî see Step 3)

**Before (old pattern):**
```go

if userID == "" {
    c.JSON(http.StatusUnauthorized, gin.H{"error": "authentication required"})
    return
}
```

**After (Template A ‚Äî only `userID` needed):**

```go
    c.JSON(http.StatusUnauthorized, gin.H{"error": "authentication required"})
    return
}
```

**After (Template A ‚Äî both `userID` and `sessionID` needed):**
```go
userID, sessionID := extractAuth(c)
if userID == "" {
    c.JSON(http.StatusUnauthorized, gin.H{"error": "authentication required"})
    return
}
```

**Template B ‚Äî call site has NO existing empty-string guard (6 sites):**
- `user_provider_credentials.go:172`
- `user_provider_credentials.go:198`
- `user_provider_credentials.go:222`
- `user_provider_credentials.go:249`
- `user_provider_credentials.go:285`
- `user_provider_credentials.go:326`

These routes are protected by auth middleware, but consistency requires an explicit guard.

**Before (old pattern, no guard):**
```go
userID := c.GetString("userID")
```

**After (Template B ‚Äî add belt-and-suspenders guard):**
```go
userID, _ := extractAuth(c)
if userID == "" {
    c.JSON(http.StatusUnauthorized, gin.H{"error": "authentication required"})
    return
}
```

> **WARNING:** Do NOT use `if userID == "" { return }` without the `c.JSON(...)` call. `extractAuth` does not write any HTTP response. Returning without writing a response produces HTTP 200 with an empty body for unauthenticated requests ‚Äî a silent security regression.
>
> Always preserve or add the `c.JSON(http.StatusUnauthorized, ...)` response.

#### Step 3 ‚Äî Special case: `user_provider_credentials.go:75‚Äì76`

The `Create` handler reads both `userID` (line 75) and `sessionID` (line 76) as separate `GetString` calls. The `sessionID` is used at line 95 (`h.keys.GetDEK(c.Request.Context(), sessionID)`). Migrate both at once:

**Before:**
```go
userID := c.GetString("userID")
sessionID := c.GetString("sessionID")
if userID == "" || sessionID == "" {
    c.JSON(http.StatusUnauthorized, gin.H{"error": "authentication required"})
    return
}
```

**After:**
```go
userID, sessionID := extractAuth(c)
if userID == "" {
    c.JSON(http.StatusUnauthorized, gin.H{"error": "authentication required"})
    return
}
```

Note: `extractAuth` returns an empty `sessionID` if it is missing. The existing guard `|| sessionID == ""` becomes implicit: if the session ID is missing the `GetDEK` call at line 95 will fail with a meaningful error. If you prefer to keep explicit session validation, add `|| sessionID == ""` to the guard.

> **Behavior note:** The current guard is `if userID == "" || sessionID == ""` which returns 401
> for a missing sessionID. The template above removes the sessionID clause, changing a
> missing-sessionID response from HTTP 401 to HTTP 503 (from the `GetDEK` call failing). If
> preserving the 401 behavior for a missing sessionID is important, keep `|| sessionID == ""`
> in the guard. Document your choice explicitly in the commit message.

#### Step 4 ‚Äî Migrate all call sites

For each of the 10 `c.GetString("userID")` call sites listed above, plus the `c.GetString("sessionID")` at line 76:

1. Read surrounding context (¬±15 lines) to understand the error-return path and whether `sessionID` is needed downstream.
2. Identify whether the site falls under **Template A** (has existing guard) or **Template B** (no guard).
3. Replace the bare `c.GetString` call(s) with `extractAuth(c)` using the appropriate template.
4. Confirm the `c.JSON(http.StatusUnauthorized, ...)` response is preserved or added as required.
5. Run the file's tests after each change.

#### Step 5 ‚Äî Verify no old-pattern call sites remain

```bash
grep -rn 'c\.GetString("userID")' --include='*.go' api/internal/handlers/
grep -rn 'c\.GetString("sessionID")' --include='*.go' api/internal/handlers/
```

Both should return zero results.

#### Step 6 ‚Äî Run tests

```bash

go test ./api/... -v
```

## Pattern 2 ‚Äî Error Classification

### Problem

Two helper functions in the handlers package classify PostgreSQL errors by string-matching the error message. `isDuplicateErr` at `admin_provider_credentials.go:19‚Äì21` checks for the SQLSTATE code `"23505"` (unique violation). `isNotFound` at `admin_provider_credentials.go:23‚Äì26` checks for `"no rows"` (pgx no-rows sentinel message). Both are fragile: they will silently mis-classify any future error whose message happens to contain those substrings, and they break if the driver changes its error message wording.

Other handlers use the Go-idiomatic `errors.Is` / `errors.As` pattern with typed sentinel errors (see `handleSecretError` in `secrets.go:874‚Äì895`).

### Current State

**Old pattern (string matching) ‚Äî functions and all call sites:**

`isDuplicateErr` definition (string-match on `"23505"`):
- `api/internal/handlers/admin_provider_credentials.go:19‚Äì21`

`isNotFound` definition (string-match on `"no rows"`):
- `api/internal/handlers/admin_provider_credentials.go:23‚Äì26`

`isDuplicateErr` call sites:
- `api/internal/handlers/admin_provider_credentials.go:137` (`Create` handler)
- `api/internal/handlers/admin_provider_credentials.go:305` (`Update` handler)
- `api/internal/handlers/user_provider_credentials.go:139` (`Create` handler)

`isNotFound` call sites:
- `api/internal/handlers/admin_provider_credentials.go:338` (`Delete` handler)

**New pattern** (`errors.As` with typed sentinel, `errors.Is` at call sites):
- `api/internal/handlers/secrets.go:874‚Äì895` ‚Äî reference implementation (`handleSecretError`)

### Migration Plan

#### Step 1 ‚Äî Confirm the Postgres driver

Read `go.mod`. The project uses `github.com/jackc/pgx/v5 v5.9.0` as a **direct** dependency. `github.com/lib/pq` is present only as an **indirect** dependency pulled in by a transitive dep and must **not** be used for error classification.

The correct pgx v5 error type is `*pgconn.PgError` from `github.com/jackc/pgx/v5/pgconn`. The `Code` field holds the SQLSTATE string.

#### Step 2 ‚Äî Create a classification helper

Create `api/internal/handlers/errors.go` (or add to an existing shared file in the package):

```go
package handlers

import (
    "errors"

    "github.com/jackc/pgx/v5/pgconn"
)

var (
    ErrDuplicateCredential = errors.New("duplicate credential")
    ErrCredentialNotFound  = errors.New("credential not found")
)

// ClassifyPostgresError maps pgx driver errors to typed sentinels.
// SQLSTATE 23505 = unique_violation ‚Üí ErrDuplicateCredential
// SQLSTATE 02000 / P0002 = no_data / no_data_found ‚Üí ErrCredentialNotFound
// All other errors are returned unchanged.
func ClassifyPostgresError(err error) error {
    var pgErr *pgconn.PgError
    if errors.As(err, &pgErr) {
        switch pgErr.Code {
        case "23505":
            return ErrDuplicateCredential
        case "02000", "P0002":
            return ErrCredentialNotFound
        }
    }
    return err
}
```

> **Driver note:** Import path is `github.com/jackc/pgx/v5/pgconn`. Do **not** use the standalone
> `github.com/jackc/pgconn` module (pgx/v4 era) ‚Äî it is not in `go.mod` and `*PgError` will not
> match pgx/v5 pool errors. Also do **not** use `github.com/lib/pq` (`*pq.Error`) ‚Äî that driver
> is not used directly by this project and the error type will never match.

#### Step 3 ‚Äî Migrate `isDuplicateErr` call sites

**Before (admin_provider_credentials.go:137):**
```go
if err := h.store.CreateAdminCredential(c.Request.Context(), row); err != nil {
    if isDuplicateErr(err) {
        c.JSON(http.StatusConflict, gin.H{"error": "credential for this provider already exists"})
        return
    }
    c.JSON(http.StatusInternalServerError, gin.H{"error": "failed to store credential"})
    return
}
```

**After:**
```go
if err := h.store.CreateAdminCredential(c.Request.Context(), row); err != nil {
    err = ClassifyPostgresError(err)
    if errors.Is(err, ErrDuplicateCredential) {
        c.JSON(http.StatusConflict, gin.H{"error": "credential for this provider already exists"})
        return
    }
    c.JSON(http.StatusInternalServerError, gin.H{"error": "failed to store credential"})
    return
}
```

Apply the same transformation to:
- `admin_provider_credentials.go:305` (`Update` handler ‚Äî `isDuplicateErr` after `UpdateAdminCredential`)
- `user_provider_credentials.go:139` (`Create` handler ‚Äî `isDuplicateErr` after `CreateUserCredential`)

#### Step 4 ‚Äî Migrate `isNotFound` call site

**Before (admin_provider_credentials.go:338):**
```go
if err := h.store.DeleteAdminCredential(c.Request.Context(), id); err != nil {
    if isNotFound(err) {
        c.JSON(http.StatusNotFound, gin.H{"error": "credential not found"})
        return
    }
    c.JSON(http.StatusInternalServerError, gin.H{"error": "failed to delete credential"})
    return
}
```

**After:**
```go
if err := h.store.DeleteAdminCredential(c.Request.Context(), id); err != nil {
    err = ClassifyPostgresError(err)
    if errors.Is(err, ErrCredentialNotFound) {
        c.JSON(http.StatusNotFound, gin.H{"error": "credential not found"})
        return
    }
    c.JSON(http.StatusInternalServerError, gin.H{"error": "failed to delete credential"})
    return
}
```

#### Step 5 ‚Äî Remove the old helper functions

Delete the `isDuplicateErr` function (lines 18‚Äì21) and the `isNotFound` function (lines 23‚Äì26) from `admin_provider_credentials.go`. Remove the `"strings"` import from both `admin_provider_credentials.go` and `user_provider_credentials.go` if it is no longer used.

#### Step 6 ‚Äî Grep for remaining string-matching error patterns

```bash
grep -rn 'strings\.Contains.*\.Error()' --include='*.go' api/internal/
grep -rn 'Contains.*23505\|Contains.*23503\|Contains.*no rows' --include='*.go' .
```

Migrate any additional occurrences found.

#### Step 7 ‚Äî Run tests

```bash
go test ./api/internal/handlers/... -v
go test ./api/... -v
```

**Commit message:** `refactor(handlers): migrate error classification to typed sentinels`

## Pattern 3 ‚Äî Event Brokers (consolidate to UserEventBroker)

### Problem

Two event broker implementations exist: `WorkspaceEventBroker` and `UserEventBroker`. The workspace broker has approximately **45 test references** across the test suite, and its functionality overlaps with the user broker. Consolidation requires a comprehensive test migration plan.

### Current State

- `api/internal/handlers/event_broker.go` ‚Äî `WorkspaceEventBroker`
- `api/internal/handlers/event_broker_user.go` ‚Äî `UserEventBroker`

### Behavioral Changes

Before migrating, confirm and document the following behavioral differences between the two brokers:

| Property | WorkspaceEventBroker | UserEventBroker |
|---|---|---|
| Channel buffer size | **16** | **128** |
| Max subscribers | **0 (no limit)** | **20** |

**These are not cosmetic differences.** Any production code that relies on unbounded workspace subscriptions (e.g. long-lived SSE streams, monitoring dashboards) must be reviewed against the 20-subscriber cap before the migration is complete. If the cap is too low for any known workload, increase `UserEventBroker`'s limit before cutting over rather than discovering it post-deployment.

### Migration Plan

#### Step 1 ‚Äî Identify all WorkspaceEventBroker consumers

```bash
grep -rn 'WorkspaceEventBroker\|workspaceEventBroker\|NewWorkspaceEventBroker' --include='*.go' .
```

#### Step 2 ‚Äî Read both broker implementations

```bash
cat api/internal/handlers/event_broker.go
cat api/internal/handlers/event_broker_user.go
```

Understand:
- What events each broker emits
- The subscriber interface
- How consumers obtain an instance
- The exact buffer sizes and subscriber limits in each implementation

#### Step 3 ‚Äî Analyze whether UserEventBroker can serve WorkspaceEventBroker's consumers

Compare the two APIs. If `UserEventBroker` can be extended with a workspace-scoped method (e.g., `SubscribeWorkspace(workspaceID string, ch chan Event)`), do that.

#### Step 4 ‚Äî Catalog all ~45 test references

```bash
grep -rn 'WorkspaceEventBroker\|workspaceEventBroker\|NewWorkspaceEventBroker' --include='*_test.go' .
```

For each test reference, classify into one of these categories:

| Category | Description | Migration Action |
|---|---|---|
| Constructor test | Tests `NewWorkspaceEventBroker()` | Replace with `NewUserEventBroker()` or remove if redundant |
| Publish test | Tests `WorkspaceEventBroker.Publish()` | Map to equivalent `UserEventBroker` publish method |
| Subscribe test | Tests `WorkspaceEventBroker.Subscribe()` | Map to workspace-scoped subscribe on `UserEventBroker` |
| Integration test | Uses broker as a dependency in a larger test | Swap the dependency, verify test expectations |
| Mock/stub | Defines a mock `WorkspaceEventBroker` | Replace with mock `UserEventBroker` |

#### Step 5 ‚Äî Migrate each consumer (production code)

For each file that uses `WorkspaceEventBroker` in production code:

1. Read the consumer code to understand how it uses the broker.
2. Replace with `UserEventBroker` usage (or the extended method).
3. Update the constructor / wiring in `api/internal/app/app.go` if needed.

#### Step 6 ‚Äî Migrate each test reference

For each of the ~45 test references:

1. Update the test to use `UserEventBroker` (or its workspace-scoped method).
2. Update mock implementations to match the new interface.
3. Adjust assertions as needed (event payloads may differ slightly).
4. Run the individual test file after migration to catch regressions early.

**Batch strategy:** Migrate tests in groups by file to keep commits reviewable:
- `event_broker_test.go` ‚Äî broker unit tests (~15 references)
- `*_handler_test.go` files ‚Äî handler integration tests (~20 references)
- Other test files ‚Äî remaining references (~10 references)

#### Step 7 ‚Äî Remove WorkspaceEventBroker

```bash
rm api/internal/handlers/event_broker.go
```

#### Step 8 ‚Äî Run full test suite

```bash
go test ./api/... -v
```

## Pattern 4 ‚Äî Secret Injection Path (remove legacy silent-skip)

### Problem

`pkg/secrets/injection.go` has two code paths for secret injection. `PrepareSecretsForInjection` (line 42) checks `if s.deriveAdminKey == nil` and, when true, calls the legacy path `prepareSecretsLegacy` (lines 223‚Äì270). The legacy path omits all admin credentials entirely (it only processes `user_secrets`) and silently skips any secret whose DEK retrieval fails (`continue` on decrypt error at line 254). This masks configuration errors ‚Äî a deployment that forgets to wire `deriveAdminKey` silently degrades to user-secrets-only injection with no operator signal.

### Current State

- `pkg/secrets/injection.go:43‚Äì45` ‚Äî nil check that gates the legacy/new path split
- `pkg/secrets/injection.go:223‚Äì270` ‚Äî `prepareSecretsLegacy` function (48 lines)

The legacy function itself:
1. Verifies workspace ownership
2. Fetches bindings from `user_secrets` only (no `workspace_credential_bindings`)
3. Gets the user DEK (returns an error if unavailable ‚Äî does not nil-check)
4. Decrypts each secret, silently skipping (`continue`) on decrypt failure
5. Returns the JSON payload

The new path (lines 42‚Äì138) performs the same steps but also:
- Handles admin credentials via the server KEK
- Merges by provider priority
- Audits decrypt failures instead of silently skipping
- Requires the store to implement `CredentialStore`

### Migration Plan

#### Step 1 ‚Äî Read the full injection file

```bash
cat pkg/secrets/injection.go
```

Understand both code paths before making any changes.

#### Step 2 ‚Äî Audit test files for NewSecretService usages

**This step is required before removal.** Run:

```bash
grep -rn 'NewSecretService' --include='*_test.go' .
```

For every test that constructs a `SecretService`, check whether:
- It calls `SetAdminKeyDeriver` (wires `deriveAdminKey` ‚Üí new path will be used), **or**
- It does **not** call `SetAdminKeyDeriver` (leaves `deriveAdminKey` nil ‚Üí currently routes to legacy path)

Any test in the second category that calls `PrepareSecretsForInjection` will break when the legacy path is removed. Update those tests to either:
- Provide a `SetAdminKeyDeriver` (preferred ‚Äî tests the production path), or
- Mock the store as a `CredentialStore` so the new path can run.

Do **not** proceed to Step 3 until every such test is categorized and a migration plan for each is written.

#### Step 3 ‚Äî Verify deriveAdminKey is always wired in production

```bash
grep -rn 'deriveAdminKey\|SetAdminKeyDeriver' --include='*.go' .
```

Confirm that the production wiring in `api/internal/app/app.go` (or equivalent) always calls `SetAdminKeyDeriver` before any workspace activation can trigger `PrepareSecretsForInjection`. If any production code path leaves `deriveAdminKey` nil, it must be fixed before the legacy path is removed.

#### Step 4 ‚Äî Remove the legacy function and nil check

1. Delete the `if s.deriveAdminKey == nil` branch (lines 43‚Äì45) from `PrepareSecretsForInjection`, making the `CredentialStore` cast mandatory.
2. If the `CredentialStore` cast fails (store wrapper without the interface), the explicit error at line 57 will surface it immediately rather than silently downgrading.
3. Delete the entire `prepareSecretsLegacy` function (lines 223‚Äì270).

> **No-bindings case:** The new path already handles zero bindings correctly ‚Äî the for-range loop
> over bindings produces no output and the function returns an empty JSON array. Verify this with
> the `TestPrepareSecrets_NoBindings` unit test. No additional guard code is needed.

#### Step 5 ‚Äî Run tests

```bash
go test ./pkg/secrets/... -v
go test ./api/... -v
```

If tests fail because they relied on the nil-`deriveAdminKey` path, update them per the plan from Step 2.

**Commit message:** `refactor(injection): remove legacy prepareSecretsLegacy path, enforce audit-tracked injection`
