## Design

### 1. `decryptBinding` — add `case "org":`

`SecretService` currently has no reference to `OrgKeyService`. Add it:

```go
// pkg/secrets/secret_service.go (or wherever SecretService is defined)
type SecretService struct {
    // ... existing fields ...
    orgKeySvc *OrgKeyService  // nil if org support not enabled (pre-US-11.2 deployments)
}

func (s *SecretService) SetOrgKeyService(svc *OrgKeyService) {
    s.orgKeySvc = svc
}
```

`decryptBinding` change:

```go
func (s *SecretService) decryptBinding(ctx context.Context, b CredentialBinding, sessionID string, serverKEK []byte) (LLMProviderData, error) {
    var key []byte
    switch b.OwnerType {
    case "user":
        dek, err := s.keys.GetDEK(ctx, sessionID)
        if err != nil {
            return LLMProviderData{}, fmt.Errorf("get user DEK: %w", err)
        }
        key = dek
    case "admin":
        if serverKEK == nil {
            return LLMProviderData{}, fmt.Errorf("server KEK unavailable")
        }
        key = serverKEK
    case "org":
        if s.orgKeySvc == nil {
            return LLMProviderData{}, fmt.Errorf("org credential injection not enabled (OrgKeyService not wired)")
        }
        orgDEK, err := s.orgKeySvc.GetOrgDEK(ctx, b.OwnerID)
        if err != nil {
            // ErrOrgDEKUnavailable is the common case: no org admin is currently logged in.
            // The binding is skipped (falls through to lower-priority credential or no credential).
            return LLMProviderData{}, fmt.Errorf("get org DEK for %s: %w", b.OwnerID, err)
        }
        key = orgDEK
    default:
        return LLMProviderData{}, fmt.Errorf("unsupported owner_type %q", b.OwnerType)
    }
    // ... rest unchanged ...
}
```

**Behaviour when org DEK is not in cache:**
The `decryptBinding` error is caught in `PrepareSecretsForInjection:83-89` — the binding is audit-logged and skipped. The workspace falls back to a lower-priority credential (admin platform credential or no LLM access). This is the correct and documented behaviour: zero-knowledge design means the platform cannot decrypt org credentials without an active admin session.

**Wiring in `app.go`:**
```go
secretService.SetOrgKeyService(orgKeyService)
```

### 2. `SeedWorkspaceCredentials` — fix org placeholder

Current code (`pg_credential_store.go:91`):
```sql
OR (caa.target_type = 'org' AND caa.target_id = $2)   -- $2 is userID (placeholder)
```

This is wrong: `target_id` for org rules is an `orgID`, not a `userID`. Using `userID` as `target_id` means the query never matches any real org auto-apply rule (since `orgID` ≠ `userID`), which is accidentally safe (no wrong bindings) but still incorrect.

Fix: `SeedWorkspaceCredentials` must be extended to accept the workspace's `orgID` (nullable):

```go
// Updated signature
func (s *PgSecretStore) SeedWorkspaceCredentials(ctx context.Context, workspaceID, userID string, orgID *string) error
```

Implementation:

```sql
-- Admin auto-apply rules (target_type='all' or 'user' matching userID)
INSERT INTO workspace_credential_bindings (credential_id, workspace_id, source_type, within_priority)
SELECT caa.credential_id, $1, 'auto', caa.within_priority
FROM credential_auto_apply caa
WHERE caa.target_type = 'all'
   OR (caa.target_type = 'user' AND caa.target_id = $2)
ON CONFLICT (credential_id, workspace_id) DO NOTHING

-- Org auto-apply rules (only if workspace has an org)
-- $3 is orgID (NULL for personal workspaces — this block skipped when $3 IS NULL)
INSERT INTO workspace_credential_bindings (credential_id, workspace_id, source_type, within_priority)
SELECT caa.credential_id, $1, 'auto', caa.within_priority
FROM credential_auto_apply caa
JOIN provider_credentials pc ON pc.id = caa.credential_id AND pc.owner_type = 'org' AND pc.owner_id = $3
WHERE $3 IS NOT NULL
  AND caa.target_type = 'org' AND caa.target_id = $3
ON CONFLICT (credential_id, workspace_id) DO NOTHING

-- Also bind all org-owned credentials for this org
INSERT INTO workspace_credential_bindings (credential_id, workspace_id, source_type, within_priority)
SELECT pc.id, $1, 'auto', 5
FROM provider_credentials pc
WHERE $3 IS NOT NULL AND pc.owner_type = 'org' AND pc.owner_id = $3
ON CONFLICT (credential_id, workspace_id) DO NOTHING
```

(Three separate `Exec` calls in sequence, or reformulated as a single query with conditional logic.)

**Caller update:** `workspace_service.go:224` must pass `meta.OrgID` (added in US-11.8):
```go
s.credProvisioner.SeedWorkspaceCredentials(ctx, meta.ID, userID, meta.OrgID)
```

**Interface updates — DECISION: diverge `CredentialProvisioner` from `CredentialStore`.**

The chosen approach is to update `CredentialProvisioner` (workspace-facing) but leave `CredentialStore` (secrets-facing) unchanged. Rationale: `CredentialStore` is a broad interface used by many callers; changing it would require updating the `AsyncAuditLogger`, the test doubles for `CredentialStore`, and any future caller. `CredentialProvisioner` is a narrow single-method interface used only by `workspace_service.go`, making it the correct and minimal change surface.

Concrete update required at **all four locations**:

1. **`CredentialProvisioner` interface** in `workspace_service.go` — update signature:
   ```go
   type CredentialProvisioner interface {
       SeedWorkspaceCredentials(ctx context.Context, workspaceID, userID string, orgID *string) error
   }
   ```

2. **`PgSecretStore.SeedWorkspaceCredentials`** — update method body and signature to accept `orgID *string`.
   `PgSecretStore` implements `CredentialProvisioner` (called via `wsSvc.SetCredentialProvisioner(pgStore)`). After this change, `PgSecretStore` satisfies `CredentialProvisioner` with the new signature.

   `CredentialStore` interface in `pkg/secrets/credential_store.go` does NOT include `SeedWorkspaceCredentials` — verify this before implementation. If it does, it must also be updated. Grep for `SeedWorkspaceCredentials` in `credential_store.go` to confirm before writing any code.

3. **`AsyncAuditLogger`** in `pkg/secrets/pg_secret_store.go` — if `AsyncAuditLogger` delegates `SeedWorkspaceCredentials` to its inner store, update the delegation method signature. If `AsyncAuditLogger` does not implement `CredentialProvisioner` (check callers), no change needed.

4. **Test doubles** — update every `fakeCredentialProvisioner` or mock that implements `CredentialProvisioner`:
   - `workspace_service_test.go`: `fakeCredentialProvisioner.SeedWorkspaceCredentials`
   - Any other test file — run `grep -r "SeedWorkspaceCredentials" --include="*_test.go"` before writing code to enumerate all test doubles

**Callers to update**: only `workspace_service.go:224` — one call site.

## Acceptance Criteria

- `decryptBinding` with `owner_type='org'` and org DEK in cache: returns decrypted `LLMProviderData`
- `decryptBinding` with `owner_type='org'` and org DEK NOT in cache: returns `ErrOrgDEKUnavailable`-wrapped error; binding is skipped (not a 500)
- `decryptBinding` with `orgKeySvc == nil`: returns explicit error (not silent)
- `SeedWorkspaceCredentials` for personal workspace (`orgID=nil`): org rules not applied; behaviour identical to before
- `SeedWorkspaceCredentials` for org workspace: org auto-apply rules applied with correct `orgID`; old `userID`-as-placeholder query gone
- All four interface locations updated (compilation succeeds): `CredentialProvisioner`, `CredentialStore`, `AsyncAuditLogger`, test fakes
- `make build` passes with zero errors after this story
