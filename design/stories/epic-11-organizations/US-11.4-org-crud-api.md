# US-11.4: Org CRUD API

**Epic:** 11 (Organizations)
**Depends On:** US-11.1 (schema), US-11.2 (OrgKeyService)
**Estimated Effort:** ~5 hours

---

## Goal

Implement org creation, listing, get, update, and soft-delete. Creating an org makes the caller its founding admin — `InitializeOrgKeys` is called inline, and the org DEK is cached for the session.

## Types

```go
// pkg/types/types.go additions

type OrgRole string
const (
    OrgRoleAdmin  OrgRole = "admin"
    OrgRoleMember OrgRole = "member"
)

type Organization struct {
    ID        string    `json:"id"`
    Name      string    `json:"name"`
    Slug      string    `json:"slug"`
    CreatedBy string    `json:"createdBy"`
    CreatedAt time.Time `json:"createdAt"`
    UpdatedAt time.Time `json:"updatedAt"`
}

type OrgMember struct {
    OrgID          string    `json:"orgId"`
    UserID         string    `json:"userId"`
    Username       string    `json:"username"`
    Email          string    `json:"email"`
    Role           OrgRole   `json:"role"`
    PendingKeyWrap bool      `json:"pendingKeyWrap"`
    CreatedAt      time.Time `json:"createdAt"`
}

type CreateOrgRequest struct {
    Name     string `json:"name"     binding:"required,min=2,max=100"`
    // Slug must match ^[a-z0-9-]+$. The validator tag "slug" is registered at app
    // startup via validator.RegisterValidation("slug", validateSlug) where validateSlug
    // checks regexp.MustCompile(`^[a-z0-9-]+$`). Do NOT use "alphanum_dash" — that tag
    // does not exist in go-playground/validator and will cause a runtime panic.
    Slug     string `json:"slug"     binding:"required,min=2,max=50,slug"`
    // Password is required to initialize the org DEK (wrap with founding admin's KEK).
    // Never stored. Required only at creation time.
    Password string `json:"password" binding:"required"`
}

type UpdateOrgRequest struct {
    Name string `json:"name" binding:"omitempty,min=2,max=100"`
    Slug string `json:"slug" binding:"omitempty,min=2,max=50,slug"`
}

// OrgResponse is returned by Create, Get, Update, and List endpoints.
// It extends Organization with the calling user's membership context.
type OrgResponse struct {
    Organization
    // UserRole is the calling user's role in this org ("admin" or "member").
    // Included so the frontend can determine which actions to show without a
    // separate /members call.
    UserRole       OrgRole `json:"userRole"`
    // UserPendingKeyWrap is true when the calling user is an admin of this org but
    // has not yet completed the key handshake (POST /accept-key). The frontend uses
    // this to show the "Complete key setup" banner (US-11.11).
    UserPendingKeyWrap bool `json:"userPendingKeyWrap"`
    MemberCount    int     `json:"memberCount"`
}
```

## Handler: `OrgsHandler`

```go
// api/internal/handlers/orgs.go

type OrgsHandler struct {
    orgStore    OrgStore
    orgKeySvc   *secrets.OrgKeyService
    dekCache    secrets.DEKCache         // to cache org DEK after creation
    // workspaceStore provides ListOrgWorkspaces for GET /orgs/:id/workspaces.
    // OrgStore implements this method — same object as orgStore.
    // Declared separately for clarity.
}
```

Note: `workspaceStore` is the same `OrgStore` instance passed as `orgStore` — `OrgStore.ListOrgWorkspaces` is defined on `PgOrgStore`. The handler uses `orgStore` for both membership operations and workspace listing. No second dependency needed.

### `POST /api/v1/orgs` — Create

1. Bind `CreateOrgRequest`; validate name + slug
2. Check slug uniqueness via `GetOrgBySlug` → 409 if taken
3. Generate `org.ID = uuid.New()`
4. Execute a single DB transaction containing:
   a. `INSERT INTO organizations ...`
   b. `INSERT INTO org_memberships (org_id, user_id, role='admin', pending_key_wrap=false)`
   c. `adminSalt = SELECT salt FROM user_keys WHERE user_id = adminUserID`
   d. Derive `adminKEK = DeriveKEK(password, adminSalt, "llmsafespace-org-kek")`
   e. `orgDEK = GenerateDEK()`
   f. `wrappedDEK = WrapDEK(adminKEK, orgDEK)`
   g. `INSERT INTO org_key_members (org_id, user_id, wrapped_dek, key_version=1)`
   — Steps (a)–(g) are all within one `pgx.Tx`. If any step fails, the entire transaction rolls back: no partial org state.
5. `cache.CacheDEK(ctx, secrets.OrgCacheKey(org.ID), orgDEK, sessionTTL)` — outside transaction; non-fatal on failure
6. Zero `orgDEK`, `adminKEK`; return 201 with `OrgResponse`

**Atomicity design:** All three DB writes must be atomic. The chosen approach is a single `PgOrgStore.CreateOrgWithAdmin` transactional method that encapsulates all three steps:

```go
// CreateOrgWithAdmin creates the org, founding membership, and wrapped org DEK
// in a single database transaction. Returns the created Organization.
// adminWrappedDEK is pre-computed by the handler (to keep crypto out of the store layer).
CreateOrgWithAdmin(ctx context.Context, org *types.Organization, adminUserID string, adminWrappedDEK []byte) (*types.Organization, error)
```

The handler flow becomes:
1. Validate slug uniqueness (outside tx — cheap check, acceptable TOCTOU given DB unique index)
2. `orgDEK, err := GenerateDEK()` — two-return form, handle error
3. `adminSalt, err := orgKeyStore.GetUserSalt(ctx, adminUserID)`
4. `adminKEK, err := secrets.DeriveKEK([]byte(req.Password), adminSalt, "llmsafespace-org-kek")`; `defer zeroBytes(adminKEK)`
5. `wrappedDEK, err := secrets.WrapDEK(adminKEK, orgDEK)`
6. `org, err := orgStore.CreateOrgWithAdmin(ctx, newOrg, adminUserID, wrappedDEK)` — single transaction
7. `cache.CacheDEK(ctx, secrets.OrgCacheKey(org.ID), orgDEK, sessionTTL)` — non-fatal
8. Zero `orgDEK`, `adminKEK`; return 201

This eliminates the need for `InitializeOrgKeys` to accept a `pgx.Tx`. `OrgKeyService.InitializeOrgKeys` remains a standalone method used only for the key handshake (US-11.5 `accept-key`). Org creation uses `CreateOrgWithAdmin` directly.

**Password logging risk:** The `req.Password` field must never appear in structured log fields. The Gin request logger logs the request body only at DEBUG level and is disabled in production (confirmed from existing logging middleware). No additional guard is needed, but `CreateOrgRequest` must have `Password` tagged with a custom `log:"-"` or the logging middleware must redact it. Add `Password string \`json:"password" log:"-"\`` — the `log:"-"` tag follows the existing `pkg/redact` convention for sensitive fields.

### `GET /api/v1/orgs` — List

Returns all orgs where `userID` is a member (any role), via `ListOrgsForUser`. This query JOINs `organizations` and filters `organizations.deleted_at IS NULL` — soft-deleted orgs are not returned.

### `GET /api/v1/orgs/:id` — Get

`IsOrgMember` check → 403 if not member. `IsOrgMember` JOINs `organizations WHERE deleted_at IS NULL` — soft-deleted orgs return 404 (treated as not-found, same as `GetOrg` which filters `deleted_at IS NULL`).

### `PUT /api/v1/orgs/:id` — Update

`IsOrgAdmin` check → 403. Updates name and/or slug. Slug uniqueness re-checked against active orgs only.

### `DELETE /api/v1/orgs/:id` — Soft delete

`IsOrgAdmin` check → 403. `OrgHasActiveWorkspaces` → 409 if any active workspaces remain.

Executes a transaction:
```sql
UPDATE workspaces SET org_id = NULL WHERE org_id = $1;
UPDATE organizations SET deleted_at = now() WHERE id = $1;
```

Both in a single transaction. This prevents the race where a workspace is created between the two statements. After this transaction, `workspaces.org_id` is NULL for all formerly-org workspaces — `verifyOwner`'s org membership path will not fire for them (no org_id → no org check).

Note: `org_memberships` and `org_key_members` rows are NOT deleted by soft-delete (no FK cascade on soft-delete). They remain but are invisible to all queries because `IsOrgMember`/`IsOrgAdmin` filter `organizations.deleted_at IS NULL`. If the org is later hard-deleted (administrative cleanup), the FK cascade removes memberships and key members.

## `OrgMemberGuard` and `OrgAdminGuard` middlewares

```go
// api/internal/server/org_guard.go (new)

// OrgMemberGuard returns a Gin middleware that verifies the caller is a member of
// the org identified by the ":id" path parameter. Writes 403 if not.
func OrgMemberGuard(orgStore OrgStore) gin.HandlerFunc

// OrgAdminGuard returns a Gin middleware that verifies the caller is an admin
// (role='admin', pending_key_wrap=false) of the org. Writes 403 if not.
func OrgAdminGuard(orgStore OrgStore) gin.HandlerFunc
```

These are used on route groups in `router.go`:

```go
orgGroup := v1.Group("/orgs")
orgGroup.Use(authMiddleware)

orgGroup.POST("", orgsHandler.Create)
orgGroup.GET("", orgsHandler.List)

orgIDGroup := orgGroup.Group("/:id")
orgIDGroup.GET("", OrgMemberGuard(orgStore), orgsHandler.Get)
orgIDGroup.PUT("", OrgAdminGuard(orgStore), orgsHandler.Update)
orgIDGroup.DELETE("", OrgAdminGuard(orgStore), orgsHandler.Delete)
```

### `RouterConfig` additions

Following the existing pattern (e.g. `AdminProviderCredentialsHandler` at `router.go:208` and `app.go:369`), add to `RouterConfig`:

```go
// api/internal/server/router.go
type RouterConfig struct {
    // ... existing fields ...
    OrgsHandler            *handlers.OrgsHandler            // nil disables all /api/v1/orgs routes
    OrgCredentialsHandler  *handlers.OrgCredentialsHandler  // nil disables org credential routes
}
```

In `NewRouter` / `registerRoutes`, add a guarded call:
```go
if cfg.OrgsHandler != nil {
    registerOrgRoutes(v1, cfg.OrgsHandler, orgStore, authMiddleware)
}
```

In `app.go`, construct and wire:
```go
// Inside the secrets/org block, after pgOrgKeyStore is constructed:
orgStore := database.NewPgOrgStore(dbPool)
orgKeyService := secrets.NewOrgKeyService(pgOrgKeyStore, dekCache)
orgAwareKS := secrets.NewOrgAwareKeyService(keyService, orgKeyService)
authSvc.SetKeyService(orgAwareKS)

orgsHandler := handlers.NewOrgsHandler(orgStore, orgKeyService, dekCache)
orgCredsHandler := handlers.NewOrgCredentialsHandler(pgStore, orgKeyService, orgStore)
rotateKeyHandler.SetOrgKeyService(orgKeyService)  // US-11.10

// In RouterConfig construction:
server.RouterConfig{
    // ... existing ...
    OrgsHandler:           orgsHandler,
    OrgCredentialsHandler: orgCredsHandler,
}
```

**`pgOrgKeyStore` construction:** `PgOrgKeyStore` lives in `pkg/secrets/org_key_store.go` and takes a `pgxpool.Pool`. It should be constructed using the same `secretsPool` that `PgSecretStore` uses, inside the `{}` secrets block in `app.go` where `secretsPool` is in scope. `orgStore` (for membership queries) uses `dbPool` (the main app pool).

**`rotateKeyHandler` vs `orgAwareKS`:** `rotateKeyHandler` holds a reference to the raw `*secrets.KeyService` (not `OrgAwareKeyService`) because its `ChangePassword` call needs the concrete `*KeyService.ChangePassword` method, which is not on `KeyServiceInterface`. `orgAwareKS` is only passed to `authSvc.SetKeyService`. There is no conflict.

## Acceptance Criteria

- `POST /api/v1/orgs`: creates org, founding admin added with `role='admin'`, `org_key_members` row created — all in a single DB transaction; partial state impossible
- `POST /api/v1/orgs` with duplicate slug: 409
- `POST /api/v1/orgs`: `password` field never appears in response body or server logs
- `GET /api/v1/orgs`: returns all user orgs where `deleted_at IS NULL`; non-member orgs not included
- `GET /api/v1/orgs/:id` as non-member: 403
- `GET /api/v1/orgs/:id` for soft-deleted org: 404
- `PUT /api/v1/orgs/:id` as admin: updates name/slug
- `PUT /api/v1/orgs/:id` as member: 403
- `DELETE /api/v1/orgs/:id` with active workspaces: 409
- `DELETE /api/v1/orgs/:id` with no workspaces: atomically nulls `workspaces.org_id` + sets `deleted_at`; subsequent `GET /orgs/:id` returns 404; `GET /workspaces` no longer shows org attribution for those workspaces
- `IsOrgMember` and `IsOrgAdmin` return false for members of soft-deleted orgs
- `OrgMemberGuard` and `OrgAdminGuard` return 403 for unauthorized callers and for soft-deleted orgs
