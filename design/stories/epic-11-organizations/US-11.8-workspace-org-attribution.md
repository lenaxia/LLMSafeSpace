# US-11.8: Workspace Org Attribution

**Epic:** 11 (Organizations)
**Depends On:** US-11.1 (schema — `workspaces.org_id`), US-11.4 (OrgStore — membership checks)
**Estimated Effort:** ~5 hours

---

## Goal

Thread `orgID` through the workspace lifecycle: creation, listing, ownership verification, CRD, and DB queries. Personal workspaces (no `orgID`) are entirely unaffected.

---

### `verifyOwner` and the full access matrix

`verifyOwner` is called in **12 places** in `workspace_service.go`. The correct access policy for each operation is:

| Operation | Call site | Creator | Org Admin | Org Member | Use |
|---|---|---|---|---|---|
| `GetWorkspace` | implicit in handler | ✅ | ✅ | ✅ | `verifyOwner` |
| `GetWorkspaceStatus` | implicit | ✅ | ✅ | ✅ | `verifyOwner` |
| `ListWorkspaceSessions` | ~910 | ✅ | ✅ | ✅ | `verifyOwner` |
| `EnsureSession` | ~740 | ✅ | ✅ | ✅ | `verifyOwner` |
| `ActivateWorkspace` | ~856 | ✅ | ✅ | ✅ | `verifyOwner` |
| `MarkSessionSeen` | ~920 | ✅ | ✅ | ✅ | `verifyOwner` |
| `RenameSession` | ~939 | ✅ | ✅ | ✅ | `verifyOwner` |
| `RenameWorkspace` | ~931 | ✅ | ✅ | ❌ | `verifyOrgAdmin` |
| `SuspendWorkspace` | ~414 | ✅ | ✅ | ❌ | `verifyOrgAdmin` |
| `RestartWorkspace` | ~475 | ✅ | ✅ | ❌ | `verifyOrgAdmin` |
| `TerminateWorkspace` | ~522 | ✅ | ✅ | ❌ | `verifyOrgAdmin` |
| `DeleteWorkspace` | ~390 | ✅ | ✅ | ❌ | `verifyOrgAdmin` |

**Rationale for member access:** org members need `EnsureSession`, `ActivateWorkspace`, `ListWorkspaceSessions`, `MarkSessionSeen`, `RenameSession` to actually USE an org workspace. Without these, an org member who is not the creator can see the workspace but cannot interact with it at all.

**Rationale for admin-only operations:** `RenameWorkspace`, `Suspend`, `Restart`, `Terminate`, `Delete` are destructive or structurally-changing operations. Allowing any org member to suspend or rename org workspaces would create operational chaos in a multi-member environment.

```go
func (s *Service) verifyOwner(ctx context.Context, userID, workspaceID string) error {
    meta, err := s.dbService.GetWorkspaceMetadata(ctx, workspaceID)
    if err != nil || meta == nil {
        return apierrors.NewNotFoundError("workspace")
    }

    // Direct owner — always allowed
    if meta.UserID == userID {
        return nil
    }

    // Org membership — allowed for any active member (IsOrgMember filters deleted_at IS NULL)
    if meta.OrgID != nil && s.orgStore != nil {
        isMember, err := s.orgStore.IsOrgMember(ctx, *meta.OrgID, userID)
        if err == nil && isMember {
            return nil
        }
    }

    return apierrors.NewForbiddenError("workspace access denied")
}
```

```go
func (s *Service) verifyOrgAdmin(ctx context.Context, userID, workspaceID string) error {
    meta, err := s.dbService.GetWorkspaceMetadata(ctx, workspaceID)
    if err != nil {
        return apierrors.NewNotFoundError("workspace")
    }
    if meta == nil {
        return apierrors.NewNotFoundError("workspace")
    }
    if meta.UserID == userID {
        return nil // creator always has full control
    }
    if meta.OrgID != nil && s.orgStore != nil {
        isAdmin, err := s.orgStore.IsOrgAdmin(ctx, *meta.OrgID, userID)
        if err != nil {
            // DB error: return 500, not 403. Silently treating a DB error as "not admin"
            // would lock out legitimate org admins on transient failures.
            return fmt.Errorf("check org admin: %w", err)
        }
        if isAdmin {
            return nil
        }
    }
    return apierrors.NewForbiddenError("workspace admin access denied")
}
```

Apply the same error-propagation fix to `verifyOwner`'s `IsOrgMember` call (line ~157 in the current spec):

```go
    if meta.OrgID != nil && s.orgStore != nil {
        isMember, err := s.orgStore.IsOrgMember(ctx, *meta.OrgID, userID)
        if err != nil {
            return fmt.Errorf("check org membership: %w", err)
        }
        if isMember {
            return nil
        }
    }
```

Replace `verifyOwner` with `verifyOrgAdmin` for: `DeleteWorkspace` (~390), `SuspendWorkspace` (~414), `RestartWorkspace` (~475), `TerminateWorkspace` (~522), `RenameWorkspace` (~931).

All other call sites keep `verifyOwner` (org members allowed).

### DB: `ListWorkspaces`

Two changes are required: the data query and the pagination COUNT query must both be updated in lockstep. A mismatch produces incorrect `Total` in `PaginationMetadata`, causing "load more" to stop early.

```sql
-- Data query (extended)
SELECT w.*, s.agent_needs_refresh, s.credentials_pending_since
FROM workspaces w
LEFT JOIN workspace_agent_state s ON s.workspace_id = w.id
LEFT JOIN org_memberships om ON om.org_id = w.org_id AND om.user_id = $1
WHERE w.deleted_at IS NULL
  AND (
    w.user_id = $1
    OR (w.org_id IS NOT NULL AND om.org_id IS NOT NULL)
  )
ORDER BY w.created_at DESC
LIMIT $2 OFFSET $3

-- COUNT query (must use IDENTICAL predicate)
SELECT COUNT(*) FROM workspaces w
LEFT JOIN org_memberships om ON om.org_id = w.org_id AND om.user_id = $1
WHERE w.deleted_at IS NULL
  AND (
    w.user_id = $1
    OR (w.org_id IS NOT NULL AND om.org_id IS NOT NULL)
  )
```

Using a LEFT JOIN rather than a subquery `IN (SELECT ...)` gives the planner more flexibility and ensures `idx_org_memberships_user` is used efficiently.

### K8s phase fetch for org workspaces

The workspace CRD carries a label `user-id=<creatorUserID>`. When org members list workspaces created by other users, the existing `fetchUserWorkspacePhases` K8s list (which uses `LabelSelector: "user-id=<listingUserID>"`) will NOT return those workspaces. The live phase will be missing.

**Fix:** For org workspaces, add an `"org-id": orgID` label to the CRD at creation time:

```go
// buildWorkspaceCRD: add org-id label when orgID is set
labels := map[string]string{
    "user-id": userID,
}
if req.OrgID != nil {
    labels["org-id"] = *req.OrgID
}
```

`fetchUserWorkspacePhases` is extended to also fetch CRDs by org-id for each org the user is a member of. Since this involves multiple K8s list calls (one per org), the implementation uses a goroutine per org with a bounded concurrency pool, or a single fan-out:

```go
// For each orgID in the user's org memberships:
orgWorkspaces, _ := k8sClient.LlmsafespaceV1().Workspaces(ns).List(
    metav1.ListOptions{LabelSelector: "org-id=" + orgID})
```

This is at most one K8s list call per org the user is in. For users in many orgs this could be expensive — but org membership is expected to be small (single digits). The total K8s API load is bounded and acceptable. If this becomes a bottleneck, a future story can add a `LIST` by org label to a shared informer cache.

**Fallback for workspaces without the org-id label** (workspaces created before this story): `fetchUserWorkspacePhases` falls back to DB `Phase` field for any workspace where no K8s CRD is found. The DB `Phase` is less fresh but avoids showing "Unknown" for every pre-existing org workspace.

## Acceptance Criteria

- `POST /workspaces` with `orgId` where user is a member: workspace created with `org_id` set in DB and CRD; CRD has `"org-id"` label
- `POST /workspaces` with `orgId` where user is NOT a member: 403
- `POST /workspaces` without `orgId`: personal workspace created, `org_id=NULL`, behaviour identical to today
- `GET /workspaces` returns both personal workspaces and org workspaces for all user's orgs; `PaginationMetadata.Total` reflects both counts
- `GET /workspaces` pagination COUNT query uses identical predicate to data query (no `Total` mismatch)
- `GET /orgs/:id/workspaces` returns all workspaces for that org (any member)
- `GET /orgs/:id/workspaces` as non-member: 403
- `GET /workspaces/:id` as org member (not creator): 200 — `verifyOwner` allows org members
- `EnsureSession` as org member (not creator): 200 — `verifyOwner` allows org members
- `ActivateWorkspace` as org member (not creator): 200 — `verifyOwner` allows org members
- `DELETE /workspaces/:id` as org member (not creator, not org admin): 403 — `verifyOrgAdmin` blocks
- `DELETE /workspaces/:id` as org admin (not creator): 200 — `verifyOrgAdmin` allows
- `SuspendWorkspace`, `RestartWorkspace`, `TerminateWorkspace`, `RenameWorkspace` as org member: 403
- Org workspace CRD has both `user-id` and `org-id` labels set
- `fetchUserWorkspacePhases` retrieves live phase for workspaces created by other org members (via `org-id` K8s label selector)
- `WorkspaceOwner.OrgID` in CRD spec correctly set; deepcopy does not panic
- `SeedWorkspaceCredentials` called with correct `orgID` (non-nil) for org workspaces
