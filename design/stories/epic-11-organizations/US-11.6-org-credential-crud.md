# US-11.6: Org Credential CRUD

**Epic:** 11 (Organizations)
**Depends On:** US-11.5 (membership + OrgKeyService wired), US-11.4 (OrgAdminGuard)
**Estimated Effort:** ~4 hours

## Goal

Allow org admins to create, list, update, and delete org-level LLM provider credentials. These are `provider_credentials` rows with `owner_type='org'` and `owner_id=<orgID>`, encrypted with the org DEK. Also implement org auto-apply rules (which org credentials are automatically bound to all org workspaces).

## Design

Org credentials are structurally identical to admin credentials (`api/internal/handlers/admin_provider_credentials.go`) and user credentials (`api/internal/handlers/user_provider_credentials.go`), with these differences:

1. **Encryption key:** org DEK from `OrgKeyService.GetOrgDEK(ctx, orgID)` (requires admin login)
2. **Owner:** `owner_type='org'`, `owner_id=<orgID>` in `provider_credentials`
3. **Access control:** `OrgAdminGuard` for ALL operations (create, list, get, update, delete)
   - Rationale: org credentials contain sensitive API keys. Even the existence of a credential for a given provider is sensitive information (it reveals which LLM providers the org uses). Members can see org workspaces use LLM providers generally â€” but enumerating specific provider credentials is admin-only. This is consistent with the README API surface which lists `GET /orgs/:id/credentials` as "admin only".
4. **Auto-apply:** org credential auto-apply uses `target_type='org'`, `target_id=<orgID>` in `credential_auto_apply`

### `OrgCredentialStore` interface

`OrgCredentialsHandler` needs specific DB operations on `provider_credentials` that are scoped to an org. Rather than expanding the full `CredentialStore`, define a focused interface:

```go
// pkg/secrets/org_credential_store.go (new file, or add to pg_credential_store.go)

// OrgCredentialStore is the DB interface for org-scoped credential operations.
// Implemented by PgSecretStore alongside CredentialStore.
type OrgCredentialStore interface {
    // CreateOrgCredential inserts a provider_credentials row with owner_type='org'.
    CreateOrgCredential(ctx context.Context, orgID, name, provider string, ciphertext []byte, modelAllowlist []string) (string, error) // returns credential ID

    // ListOrgCredentials returns all credentials for an org. Never decrypts.
    ListOrgCredentials(ctx context.Context, orgID string) ([]*OrgCredentialMetadata, error)

    // GetOrgCredential returns one credential by ID, verifying owner_type='org' and owner_id=orgID.
    GetOrgCredential(ctx context.Context, orgID, credID string) (*OrgCredentialRow, error) // includes ciphertext for update

    // UpdateOrgCredential updates name and/or re-encrypted ciphertext for a credential.
    UpdateOrgCredential(ctx context.Context, orgID, credID string, name *string, ciphertext []byte, modelAllowlist []string, keyVersion int) error

    // DeleteOrgCredential deletes a credential; FK cascades handle bindings and auto-apply.
    DeleteOrgCredential(ctx context.Context, orgID, credID string) error

    // BindCredentialToAllOrgWorkspaces inserts workspace_credential_bindings for every
    // workspace with org_id=orgID. Called after CreateOrgCredential so existing org
    // workspaces immediately get the new credential bound. Idempotent (ON CONFLICT DO NOTHING).
    BindCredentialToAllOrgWorkspaces(ctx context.Context, credentialID, orgID string) error

    // OrgAutoApply CRUD (mirrors admin auto-apply)
    CreateOrgAutoApply(ctx context.Context, credentialID, orgID string, withinPriority int) error
    ListOrgAutoApply(ctx context.Context, orgID string) ([]*secrets.AutoApplyRule, error)
    DeleteOrgAutoApply(ctx context.Context, credentialID, orgID string) error
}

// OrgCredentialMetadata is the list-view of an org credential (no ciphertext).
type OrgCredentialMetadata struct {
    ID             string
    OrgID          string
    Name           string
    Provider       string
    ModelAllowlist []string
    CreatedAt      time.Time
    UpdatedAt      time.Time
}

// OrgCredentialRow is the full row including ciphertext (for update operations).
type OrgCredentialRow struct {
    OrgCredentialMetadata
    Ciphertext []byte
    KeyVersion int
}
```

**`PgSecretStore` implements both `CredentialStore` (existing) and `OrgCredentialStore` (new).** No interface conflict â€” these are separate named interfaces with non-overlapping method sets.

### `OrgCredentialsHandler` (revised)

```go
type OrgCredentialsHandler struct {
    credStore   OrgCredentialStore   // narrow interface â€” not full CredentialStore
    orgKeySvc   *secrets.OrgKeyService
    autoApply   *secrets.PgAutoApplyStore // for auto-apply route registration (reuse admin pattern)
}
```

### `POST /api/v1/orgs/:id/credentials`

1. `OrgAdminGuard`
2. Bind `CreateOrgCredentialRequest{Name, Provider, APIKey, BaseURL, ModelAllowlist}`
3. `orgDEK, err = orgKeySvc.GetOrgDEK(ctx, orgID)` â†’ `ErrOrgDEKUnavailable` â†’ 409
4. Encrypt `LLMProviderData{Provider, APIKey, BaseURL, ...}` with org DEK â†’ `ciphertext`
5. `credID, err = credStore.CreateOrgCredential(ctx, orgID, name, provider, ciphertext, modelAllowlist)`
6. `credStore.BindCredentialToAllOrgWorkspaces(ctx, credID, orgID)` â€” bind to all existing org workspaces
7. Return 201

**Why 409 on unavailable DEK:** The admin is logged in (OrgAdminGuard passed), but the org DEK may not be cached if the admin's key handshake was just completed but they haven't re-logged in. Message: `"org DEK not available â€” please log out and back in to refresh your org key"`.

**Why bind to existing workspaces:** Without step 6, an org with 3 existing workspaces would not get the new credential until those workspaces are deleted and recreated. `BindCredentialToAllOrgWorkspaces` is the org equivalent of `BindCredentialToAllUserWorkspaces` (live in `pg_credential_store.go:116`). Idempotent via `ON CONFLICT DO NOTHING`.

### `GET /api/v1/orgs/:id/credentials`

1. `OrgAdminGuard` (admin only â€” see access control rationale above)
2. `creds, err = credStore.ListOrgCredentials(ctx, orgID)`
3. Return `OrgCredentialMetadata[]` â€” names, providers, model_allowlist, timestamps; never ciphertext

### `PUT /api/v1/orgs/:id/credentials/:credID`

1. `OrgAdminGuard`
2. `orgDEK` from cache â†’ 409 if unavailable
3. `existing, err = credStore.GetOrgCredential(ctx, orgID, credID)` â€” fetches ciphertext
4. If `APIKey` is being updated: decrypt existing ciphertext with orgDEK, apply field changes, re-encrypt with orgDEK; increment `keyVersion`
5. `credStore.UpdateOrgCredential(ctx, orgID, credID, name, newCiphertext, modelAllowlist, keyVersion)`

### `DELETE /api/v1/orgs/:id/credentials/:credID`

1. `OrgAdminGuard`
2. `credStore.DeleteOrgCredential(ctx, orgID, credID)` â€” FK cascade removes `workspace_credential_bindings` and `credential_auto_apply` rows

### Auto-apply routes

```
POST   /api/v1/orgs/:id/credentials/:credID/auto-apply
GET    /api/v1/orgs/:id/credentials/:credID/auto-apply
DELETE /api/v1/orgs/:id/credentials/:credID/auto-apply
```

Call `credStore.CreateOrgAutoApply`, `ListOrgAutoApply`, `DeleteOrgAutoApply` respectively. The `target_type` is fixed as `"org"` and `target_id` is the org's ID â€” seeding for future workspaces. Existing org workspace binding is handled by the credential create step above.

## Acceptance Criteria

- `POST /orgs/:id/credentials` as admin: credential created; immediately bound to all existing org workspaces
- `POST /orgs/:id/credentials` with org DEK not in cache: 409 with guidance message
- `POST /orgs/:id/credentials` as member: 403
- `GET /orgs/:id/credentials` as admin: 200, names + providers only, no ciphertext
- `GET /orgs/:id/credentials` as member (non-admin): 403
- `GET /orgs/:id/credentials` as non-member: 403
- `PUT /orgs/:id/credentials/:id` updates ciphertext correctly; old API key cannot be decrypted with org DEK after update
- `DELETE /orgs/:id/credentials/:id`: credential and all bindings removed
- Auto-apply rule `target_type='org'`, `target_id=<orgID>` created correctly
- `provider` uniqueness constraint: two org credentials for the same provider â†’ 409
