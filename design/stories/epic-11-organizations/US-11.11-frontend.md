## API Client (`frontend/src/api/orgs.ts`)

```typescript
// Organization types (mirror pkg/types/types.go)
export interface Organization { id: string; name: string; slug: string; createdBy: string; createdAt: string; updatedAt: string; }
export interface OrgMember { orgId: string; userId: string; username: string; email: string; role: "admin" | "member"; pendingKeyWrap: boolean; createdAt: string; }
export interface OrgCredential { id: string; name: string; provider: string; modelAllowlist: string[]; createdAt: string; updatedAt: string; }
export interface CreateOrgRequest { name: string; slug: string; password: string; }
export interface AddMemberRequest { userId: string; role: "admin" | "member"; }
export interface AcceptKeyRequest { password: string; }
export interface CreateOrgCredentialRequest { name: string; provider: string; apiKey: string; baseURL?: string; modelAllowlist?: string[]; }

export const orgsApi = {
  list:          ()                         => api.get<Organization[]>("/orgs"),
  create:        (req: CreateOrgRequest)    => api.post<Organization>("/orgs", req),
  get:           (id: string)               => api.get<Organization>(`/orgs/${id}`),
  update:        (id: string, req: Partial<{name: string; slug: string}>) => api.put<Organization>(`/orgs/${id}`, req),
  delete:        (id: string)               => api.delete<void>(`/orgs/${id}`),

  listMembers:   (id: string)               => api.get<OrgMember[]>(`/orgs/${id}/members`),
  addMember:     (id: string, req: AddMemberRequest) => api.post<OrgMember>(`/orgs/${id}/members`, req),
  removeMember:  (id: string, userId: string) => api.delete<void>(`/orgs/${id}/members/${userId}`),
  acceptKey:     (id: string, req: AcceptKeyRequest) => api.post<{message: string}>(`/orgs/${id}/accept-key`, req),
  rotateKey:     (id: string, req: {password: string}) => api.post<{message: string}>(`/orgs/${id}/rotate-key`, req),

  listCredentials:   (id: string)           => api.get<OrgCredential[]>(`/orgs/${id}/credentials`),
  createCredential:  (id: string, req: CreateOrgCredentialRequest) => api.post<OrgCredential>(`/orgs/${id}/credentials`, req),
  deleteCredential:  (id: string, credId: string) => api.delete<void>(`/orgs/${id}/credentials/${credId}`),

  listWorkspaces:    (id: string)           => api.get<WorkspaceListItem[]>(`/orgs/${id}/workspaces`),
};
```

**Security note:** `password` fields in `CreateOrgRequest` and `AcceptKeyRequest` are never cached, logged, or stored in component state beyond the form submit handler. Use `useRef` or clear immediately after submission.

## Key UI Components

### `CreateOrgModal`

- Fields: Name, Slug (auto-generated from name, editable), Password (required â€” explain why: "used to encrypt your organisation's credentials")
- Password field has `type="password"` and `autoComplete="current-password"`
- On submit: calls `orgsApi.create()`; on success: navigates to new org page
- Error handling: 409 slug conflict shown inline

### `OrgPage` (`/orgs/:id`)

Tabbed layout with three tabs:
1. **Members** â€” list of members with role badges; "Invite Member" button (admin only); pending admins shown with warning badge; "Remove" button per member (admin only)
2. **Credentials** â€” list of org LLM credentials; "Add Credential" button (admin only); follows same card pattern as `UserProviderCredentialsTab`
3. **Workspaces** â€” list of org workspaces; links to workspace chat pages; "Create Workspace" button

### Accept Key Banner

When a user logs in and has a `pendingKeyWrap=true` membership, show a persistent banner:
> "You've been added as an admin of **Org Name**. [Complete key setup] to enable access to org credentials."

Clicking the link opens a modal asking for their password, calls `acceptKey`.

**Detection â€” how the frontend knows:** `GET /api/v1/orgs` returns `OrgResponse[]` (defined in US-11.4 types), which includes `userRole` and `userPendingKeyWrap` for the calling user in each org. The frontend checks `orgsApi.list()` on app load/login and filters for `org.userPendingKeyWrap === true`. This is a single API call â€” no N+1 per-org membership polling needed.

The `orgsApi` TypeScript type must be updated to include these fields:
```typescript
export interface OrgResponse {
  id: string;
  name: string;
  slug: string;
  createdBy: string;
  createdAt: string;
  updatedAt: string;
  userRole: "admin" | "member";
  userPendingKeyWrap: boolean;
  memberCount: number;
}
```

The `ListOrgsForUser` DB query in `PgOrgStore` must be updated to JOIN `org_memberships` on the calling user and include `role` and `pending_key_wrap` in the result set.

### Workspace Create â€” Org Selector

If user is member of orgs: shown as optional select with placeholder "Personal workspace (no organisation)".
