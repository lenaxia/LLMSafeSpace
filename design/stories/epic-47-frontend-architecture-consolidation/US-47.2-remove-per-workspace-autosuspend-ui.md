# US-47.2 — Remove per-workspace autoSuspend UI (silent failure)

| Field | Value |
|-------|-------|
| Epic | 47 — Frontend Architecture Consolidation |
| Priority | High (live user-facing silent failure) |
| Depends On | None |
| Blocks | None |
| Estimated | 30 minutes |
| Evidence | `design/0039` Q1 |

## Summary

`WorkspaceSettingsDrawer.tsx` collects an Auto-Suspend toggle and idle-timeout input from the user, shows a warning ("Disabling auto-suspend will keep this workspace running indefinitely, consuming compute minutes"), and calls `onSave({ autoSuspendEnabled, autoSuspendIdleMinutes })`. The caller wires `onSave` to **nothing**: `Sidebar.tsx:421` — `onSave={async () => {}}`. The settings are silently discarded.

**Product decision (user-confirmed):** no per-workspace autoSuspend. Account-level (Tier 3) autoSuspend is tracked in US-47.4. The per-workspace UI must be removed.

## Scope

### Remove from `WorkspaceSettingsDrawer.tsx`
- The `WorkspaceSettings` interface (`autoSuspendEnabled`, `autoSuspendIdleMinutes` fields)
- The `autoSuspend` / `idleMinutes` state (`useState` at `:31-32`)
- The Auto-Suspend section of the form (`:111-130`)
- The `autoSuspend` / `idleMinutes` fields from the `onSave` payload (`:54-57`)
- The `onSave` prop itself (it becomes unused after removing the autoSuspend section — the only other thing the drawer saves is secret bindings, which it persists directly via `api.put` at `:75`)

### Update `Sidebar.tsx`
- Remove `onSave={async () => {}}` from the `WorkspaceSettingsDrawer` usage (`:421`)
- Remove the `onSave` prop from the drawer's interface

### Preserve
- The secret-bindings section of the drawer (it persists correctly via `api.put`)
- The `NumberInput` and `Toggle` UI components (reusable, not dead)

## Acceptance Criteria

1. The drawer no longer shows Auto-Suspend controls.
2. Secret bindings still persist correctly.
3. No TypeScript errors from the removed interface fields.
4. `npm run typecheck && npm run test && npm run lint && npm run build` pass.
5. No test asserts the presence of the autoSuspend UI (if any do, update them).

## Notes

This is a pure removal — no backend changes needed. The backend already does NOT accept per-workspace autoSuspend from the frontend (it reads only the global instance setting at workspace creation, `workspace_service.go:867-877`).

US-47.4 adds account-level (Tier 3) autoSuspend as a **new** feature with UI in the user Preferences tab, not the workspace drawer.
