## Summary

Three independent performance/idiom fixes in the chat state layer. Each is small but addresses a measurable cost.

## Fix 1: Remove per-message useNow interval (`design/0038` §3)

`MessageBubble.tsx:54` calls `useNow()` (60s `setInterval` per instance). The format function (`formatTimestamp` at `:40-49`) only produces clock-time or date — never relative. N messages = N intervals for output that never changes.

**Change:** Remove `useNow()` from `MessageBubble`. Pass `formatTimestamp(message.createdAt)` directly (no `now` parameter needed — the function doesn't use it for relative formatting).

## Fix 2: Fix queryCache.getAll() hot path (`design/0038` §4)

`SessionActivityProvider.tsx:56-165` subscribes to the query cache and runs `seedBusy()` + `reconcileUnread()` on every `["sessions"]` update. Both call `queryCache.getAll()` (walk the entire cache) even though the changed query is available on `event.query`.

Amplifier: `ChatPage.tsx:550` invalidates `["sessions"]` on every `session.status` SSE event.

**Change:** In the subscribe callback, use `event.query.state.data` (already filtered by key prefix at `:157`) instead of `getAll()`. Restructure `seedBusy` and `reconcileUnread` to accept the changed query's data + wsId rather than scanning all queries.

Preserve the initial "seed all on mount" path (needs `getAll()` once, on mount only).

## Fix 3: Replace ref+counter workaround (`design/0038` §5)

`ChatPage.tsx:209-223` uses `contextBySessionRef` (Map) + `contextVersion` (counter) + `void contextVersion` (lint silencer) to force reactivity. The Map only serves the current session — a scalar `useState<number | undefined>` is both simpler and correct.

**Change:**
```ts
// before
const contextBySessionRef = useRef<Map<string, number>>(new Map());
const [contextVersion, setContextVersion] = useState(0);
// ... void contextVersion; read ref

// after
const [realtimeContextUsed, setRealtimeContextUsed] = useState<number | undefined>(undefined);
// on session.next.step.ended: setRealtimeContextUsed(promptTokens)
// on session change: setRealtimeContextUsed(undefined)
```

Update the `compactionDetected` effect to depend on `realtimeContextUsed` directly.

## Acceptance Criteria

1. `MessageBubble` no longer calls `useNow()`; timestamps still render correctly.
2. `SessionActivityProvider` subscriber uses `event.query` instead of `getAll()` on incremental updates; initial mount still seeds all.
3. `ChatPage` uses scalar `useState` for realtime context; `contextBySessionRef`, `contextVersion`, and `void contextVersion` are removed.
4. All existing tests pass (the 1000+ line `SessionActivityProvider.test.tsx` is the primary coverage).
5. `npm run typecheck && npm run test && npm run lint && npm run build` pass.
