# US-47.11 — Route code-splitting (lazy-load org-admin + settings)

| Field | Value |
|-------|-------|
| Epic | 47 — Frontend Architecture Consolidation |
| Priority | Low |
| Depends On | None (do alongside US-47.10 for clean lazy boundaries) |
| Blocks | None |
| Estimated | 1 hour |
| Evidence | `design/0039` Q5 |

## Summary

`router.tsx` eagerly imports every page — `ChatPage`, `SettingsPage`, all five `Org*Tab` components, `LoginPage`/`RegisterPage`. The entire org-admin section (which most users never visit) ships in the initial bundle.

## Change

```tsx
// before
import { ChatPage } from "./pages/ChatPage";
import { SettingsPage } from "./pages/SettingsPage";
import { OrgAdminLayout } from "./components/org-admin/OrgAdminLayout";

// after
import { lazy } from "react";
const SettingsPage = lazy(() => import("./pages/SettingsPage").then(m => ({ default: m.SettingsPage })));
const OrgAdminLayout = lazy(() => import("./components/org-admin/OrgAdminLayout").then(m => ({ default: m.OrgAdminLayout })));
```

Wrap lazy routes in `<Suspense fallback={<Spinner size="lg" />}>`. The `RequireAuth`/`GuestOnly` wrappers already handle a loading state via `<Spinner>`.

**Keep eager:** `LoginPage`, `RegisterPage` (small, needed for first paint). `ChatPage` is the primary surface — measure bundle impact before deciding whether to lazy-load it.

**Lazy-load:** `SettingsPage`, `OrgAdminLayout` + all `Org*Tab` children, `NotFoundPage`.

## Acceptance Criteria

1. Org-admin + settings are separate chunks in the build output.
2. Initial bundle size decreases measurably (`vite build` output).
3. Navigating to `/settings` or `/orgs/:id` shows a brief `<Spinner>` then loads.
4. `npm run build` passes.
