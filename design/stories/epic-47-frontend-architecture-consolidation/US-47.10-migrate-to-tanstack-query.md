## Summary

16 settings/org-admin components hand-roll `useState(true)` loading + `useState('')` error + `useEffect` fetch + `try/catch/finally` (33 `setLoading` call sites). The chat/sidebar half of the app uses TanStack Query idiomatically. Two paradigms for server state. Every migration follows the same mechanical shape.

## Components to migrate

**Settings tabs:** `AdminProviderCredentialsTab`, `UserProviderCredentialsTab`, `ApiKeysTab`, `SecretsTab`, `UserSettingsTab`, `OrgSettingsTab`, `RelayTab`, `RelaySetupWizard`, `RelayStatusDashboard`

**Org-admin tabs:** `OrgAdminLayout`, `OrgMembersTab`, `OrgWorkspacesTab`, `OrgAuditTab`, `OrgCredentialsTab`, `OrgBillingTab`

## Migration pattern (per component)

```tsx
// before
const [data, setData] = useState<T[]>([]);
const [loading, setLoading] = useState(true);
const [error, setError] = useState("");

const refresh = useCallback(async () => {
  setLoading(true); setError("");
  try { setData(await api.list()); }
  catch (e) { setError(e instanceof Error ? e.message : "Failed"); }
  finally { setLoading(false); }
}, [deps]);
useEffect(() => { refresh(); }, [refresh]);

// after
const { data, isLoading, error, refetch } = useQuery({
  queryKey: ["entity", scopeId],
  queryFn: () => api.list(),
});
```

Mutations follow the same pattern: replace manual `setSaving(true)` + try/catch with `useMutation` + `onSuccess: invalidate`.

## Benefits (free with migration)

- **Caching** — tab remount no longer re-fetches
- **Deduplication** — two components with the same key share one request
- **Background refetch** — data stays fresh
- **Invalidation** — after a mutation, sibling tabs refresh via `invalidateQueries`
- **Consistent error/loading state** — one pattern, one UX

## Acceptance Criteria

1. No `useState(true)` loading + `useEffect` fetch pattern remains in settings/org-admin components.
2. All data fetching goes through `useQuery` / `useMutation`.
3. No behaviour regression — each tab still loads, displays, mutates, and errors correctly.
4. `npm run typecheck && npm run test && npm run lint && npm run build` pass.
5. No new test failures (existing component tests mock `*Api` objects — they continue to work with Query).

## Notes

- Migrate per-tab in separate PRs (each is independently shippable, reviewable, and revertable).
- Org-admin tabs first (most uniform pattern, ship in the lazy-loaded bundle from US-47.11).
- `useUserSettings.ts` uses `useSyncExternalStore` — that one is justified (cross-component shared mutable state with persistence). Leave it as-is.
