# US-47.8 — Gate wsLog + remove stray console.log

## Summary

`wsLog` (`lib/wsLog.ts:36`) unconditionally calls `console.log` — 21 production call sites. Every SSE reconnect storm fills the user's browser console. Separately, `useSessionTitle.ts:39` has a stray `console.log("[SessionTitle] streaming ended...")`.

## Change

### Gate wsLog
Replace the unconditional `console.log` with a build-time or runtime gate:

```ts
// Option A: build-time (zero runtime cost in prod)
const enabled = import.meta.env.DEV;

// Option B: runtime toggle (support-team friendly)
const enabled = (() => {
  try { return localStorage.getItem("lsp.debug.ws") === "1"; } catch { return false; }
})();

export function wsLog(...): void {
  if (!enabled) return;
  // existing console.log body
}
```

**Product decision:** is always-on logging intentional for support diagnostics? If yes, use Option B (default off, toggle on for support sessions). If no, use Option A.

### Remove stray console.log
Delete `console.log("[SessionTitle] streaming ended, refetching title for", sessionId)` at `useSessionTitle.ts:39`.

## Acceptance Criteria

1. No `console.log` output from `wsLog` in production build (or only when explicitly enabled).
2. `useSessionTitle.ts:39` stray log removed.
3. All tests pass.
