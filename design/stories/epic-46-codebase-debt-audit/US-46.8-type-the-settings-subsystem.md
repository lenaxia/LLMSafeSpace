# US-46.8: Type the Settings Subsystem

**Epic:** 46 — Codebase Debt Audit & Abstraction Foundation
**Effort:** Medium (2d)
**ROI:** Medium-High
**Depends on:** —
**Status:** Proposed

---

## Problem

`pkg/settings/instance_service.go` stores all settings as `data map[string]any` (line 31). Consumers retrieve values via untyped getters:

```go
instanceSettings.GetString(ctx, "workspace.defaultStorageSize")
instanceSettings.GetInt(ctx, "rateLimiting.defaultLimit")
```

This violates Rule 1 (Type Safety First): *"Maps are acceptable only when parsing external JSON/YAML with unknown structure — and even then, convert to a typed struct immediately."* The setting subsystem does not convert.

Concrete harm:
- A typo in the key string (`"workspace.defualtStorageSize"`) compiles and silently returns the default at runtime.
- The admin UX (`PUT /admin/settings/{key}`) accepts any key, including unknown ones, with no compile-time or schema-time check.
- The setting key→type mapping lives only in the README-LLM.md Storage Settings table and the minds of the maintainers.

## Scope

**In:**
- Define a typed settings registry in `pkg/settings/registry.go`:
  ```go
  type Key struct {
      name    string
      section string
      default any
      validate func(any) error
  }
  var (
      KeyWorkspaceDefaultStorageSize = Key{"workspace.defaultStorageSize", "workspace", "15Gi", validateStorageSize}
      KeyWorkspaceDefaultStorageClass = Key{"workspace.defaultStorageClass", "workspace", "", nil}
      KeyRateLimitingDefaultLimit = Key{"rateLimiting.defaultLimit", "rateLimiting", 100, validatePositiveInt}
      // ... enumerate all current settings
  )
  ```
- Add typed getters to `InstanceService`:
  ```go
  func (s *InstanceService) WorkspaceDefaultStorageSize(ctx context.Context) (resource.Quantity, error)
  func (s *InstanceService) RateLimitingDefaultLimit(ctx context.Context) (int, error)
  ```
- Migrate consumers (`api/internal/services/workspace/workspace_service.go`, `api/internal/middleware/rate_limit.go`, `api/internal/middleware/metering.go`) to call the typed getters.
- Validate on write: `PUT /admin/settings/{key}` rejects unknown keys or invalid values with a typed `DomainError` (from US-46.4).
- Keep the underlying storage as `map[string]any` for backwards compatibility during migration (the typed getters translate); a later story can replace the storage shape.

**Out:**
- User settings (`pkg/settings/user_service.go`) — separate subsystem; out of scope here.
- Migrating every consumer in one PR (prioritise the 4 highest-value consumers; long tail adopts organically).
- Removing the untyped `GetString`/`GetInt` methods (deprecate them with a comment; remove in a later epic).
- Frontend admin UX changes (the typed registry is server-side; frontend keeps sending string keys).

## Acceptance Criteria

- [ ] `pkg/settings/registry.go` defines a `Key` type with name, section, default, validator.
- [ ] Every setting currently listed in README-LLM.md Storage Settings + Configuration Reference has a typed `Key` constant.
- [ ] `InstanceService` exposes typed getters for the 4 highest-value settings (defaultStorageSize, defaultStorageClass, rateLimitingDefaultLimit, rateLimitingBurstSize).
- [ ] The 4 migrated consumers call typed getters, not `GetString`.
- [ ] `PUT /admin/settings/{key}` rejects unknown keys with a `DomainError` (Category = Validation).
- [ ] `grep -rn 'GetString(ctx, "' --include="*.go" | grep -v _test.go` count decreases by ≥4.
- [ ] All existing tests pass.
- [ ] Worklog entry.
