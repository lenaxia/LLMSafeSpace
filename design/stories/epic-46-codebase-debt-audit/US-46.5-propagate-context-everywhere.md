# US-46.5: Propagate Context Everywhere

**Epic:** 46 — Codebase Debt Audit & Abstraction Foundation
**Effort:** Medium (2d)
**ROI:** High
**Depends on:** —
**Status:** Proposed

---

## Problem

`context.TODO()` literally means *"I don't know what context belongs here."* Using it in production code violates Rule 3 (explicit over implicit) and breaks deadline propagation — a request handler with a 30s deadline will silently launch a background operation that outlives the user's request.

**187 occurrences in production code:**
```bash
$ grep -rn "context.TODO()\|context.Background()" --include="*.go" | grep -v _test.go | grep -v zz_generated | wc -l
187
```

Note: `golangci-lint`'s `contextcheck` is already enabled (per `.golangci.yml` comment), so why are these not flagged? Because most are inside `func main()` or startup code where `context.Background()` is legitimate. The audit needs to distinguish:

| Legitimate | Illegitimate |
|------------|--------------|
| `func main()` boot sequence | Request handler creating a background ctx for a sub-call |
| Background goroutines spawned at startup (e.g. `metering.Service.run`) | Middleware that drops `c.Request.Context()` for a K8s call |
| Tests | Service methods that take a ctx param then ignore it for downstream calls |

## Scope

**In:**
- Enumerate every `context.TODO()` and `context.Background()` in production code.
- Classify each: legitimate (background goroutine, boot sequence) vs illegitimate (drops request deadline).
- For illegitimate cases, propagate the parent context:
  - In Gin handlers: replace `context.Background()` with `c.Request.Context()`.
  - In service methods that already accept `ctx context.Context`: replace internal `context.Background()` calls with the passed `ctx`.
  - For background goroutines that need their own ctx: derive from a startup ctx with explicit `context.WithCancel` so shutdown is observable.
- Add a custom lint rule (or grep-based pre-commit check) that flags new `context.TODO()` in non-`main()` production code.

**Out:**
- Removing `context.Background()` from `func main()` and `func init()` (legitimate).
- Removing it from tests (legitimate).
- Adding deadlines where none existed before (separate concern; this story only propagates ctx, doesn't add timeouts).
- Refactoring function signatures that don't currently take a ctx (those are flagged for follow-up but not changed here — adding ctx params is a breaking interface change).

## Acceptance Criteria

- [ ] Audit document (in worklog) listing every `context.TODO()` and `context.Background()` site, classified legitimate/illegitimate.
- [ ] Every illegitimate site fixed to propagate the parent context.
