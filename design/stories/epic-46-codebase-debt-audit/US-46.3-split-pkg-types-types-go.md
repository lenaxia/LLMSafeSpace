# US-46.3: Split `pkg/types/types.go`

**Epic:** 46 — Codebase Debt Audit & Abstraction Foundation
**Effort:** Small (1d)
**ROI:** High
**Depends on:** —
**Status:** Proposed

---

## Problem

`pkg/types/types.go` is a **905-line god file containing 76 types** spanning every domain: auth, workspace, session, network, secrets, settings, organisations, billing, pagination. Rule 4 requires small focused units. This file:

- Causes merge conflicts on every cross-domain PR
- Forces every importer to compile the entire type graph even when they need one struct
- Makes it impossible to see "what types belong to domain X" without grepping
- Violates the "every struct has single responsibility" PR rubric criterion

## Scope

**In:**
- Split `pkg/types/types.go` into per-domain files **in the same package** (no import path changes; this is purely a file reorganisation):
  - `pkg/types/auth.go` — `User`, `RegisterRequest`, `LoginRequest`, `AuthResponse`, `CreateAPIKeyRequest`, `APIKey`, `UserUpdates`, `CachedSession`
  - `pkg/types/workspace.go` — `Workspace`, `CreateWorkspaceRequest`, `WorkspaceMetadata`, `WorkspaceUpdates`, `WorkspaceListResult`, `WorkspaceListItem`, `WorkspaceStatusResult`, `ResourceRequirements`, `StorageConfig`, `FilesystemConfig`, `SecurityContext`, `ProfileReference`, `ContainerStatus`, `ContainerStateValue`, `NetworkInfo`, `NetworkAccess`, `EgressRule`, `PortRule`
  - `pkg/types/session.go` — `Session`, `SessionStatusItem`, `Message`, `WSConnection`, `AgentHealthResult`, `WorkspaceConditionResult`, `CredentialStateResult`
  - `pkg/types/pagination.go` — `PaginationMetadata`, `ListOptions`
  - `pkg/types/event.go` — `Event`, `ResourceStatus`, `ExecutionResult`, `FileInfo`
  - `pkg/types/orgs.go` — all `Org*` types (added by Epic 11)
  - `pkg/types/billing.go` — `Usage*`, `Billing*`, `Quota*` types (added by Epic 12)
  - `pkg/types/settings.go` — settings-related DTOs
  - `pkg/types/context.go` — `contextKey` and the existing context helpers
- Delete the original `types.go` after the split (or keep it as a thin re-export shim if any external importers rely on the file existing — check `sdks/`).
- Update `doc.go` to describe the new layout.

**Out:**
- Any type rename, field change, or tag change (pure file move).
- Splitting the package into subpackages (`pkg/types/auth/`, etc.) — that would break the import path for every consumer and is a larger blast radius than this story allows.
- The `pkg/apis/llmsafespace/v1/` CRD types (separate concern, owned by kubebuilder).
- Merging CRD types and DTOs (forbidden per README-LLM.md:352).

## Acceptance Criteria

- [ ] `wc -l pkg/types/*.go` shows every file ≤250 lines (except `doc.go`).
- [ ] `grep -c '^type ' pkg/types/*.go` shows types grouped by domain.
- [ ] No import path changes: `go build ./...` compiles unchanged.
- [ ] `go test -timeout 60s -race ./...` passes unchanged.
- [ ] `contract_test.go` and `types_test.go` still pass.
- [ ] Worklog entry.

## Verification

```bash
wc -l pkg/types/*.go
awk '/^type /{c++} END{print c}' pkg/types/*.go    # sums to 76 across files
go build ./...
go test -timeout 60s -race ./pkg/types/...
```
