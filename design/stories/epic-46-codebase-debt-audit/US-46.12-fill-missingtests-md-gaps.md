# US-46.12: Fill `MISSINGTESTS.md` Gaps

**Epic:** 46 — Codebase Debt Audit & Abstraction Foundation
**Effort:** Medium (2d)
**ROI:** Medium
**Depends on:** US-46.7 (Service interfaces — makes fakes easier)
**Status:** Proposed

---

## Problem

`api/internal/middleware/MISSINGTESTS.md` exists. Its presence is itself a **Rule 0 violation**: *"TDD is MANDATORY. Write tests BEFORE writing functional code. Always."* The file lists 6 categories of missing tests, of which the two highest-signal gaps are:

1. **Auth Middleware RBAC tests** (item 5): *"Tests for role-based access control with complex permission hierarchies"* — auth is security-critical; RBAC holes are how data leaks happen.
2. **Rate Limiting bursting behaviour tests** (item 6): *"Tests for rate limit bursting behavior"* — Epic 38 US-38.1 is rewriting the rate limiter; bursting must be tested before and after that rewrite to catch regressions.

The other 4 categories (middleware chaining, context propagation, error handling edge cases, validation of nested objects) are lower-signal and are explicitly out of scope here.

## Scope

**In:**
- Implement Auth Middleware RBAC tests covering:
  - User with `role=admin` accesses admin endpoint → 200
  - User with `role=member` accesses admin endpoint → 403
  - User with no role accesses protected endpoint → 401
  - Org member vs org admin vs platform admin permission matrix
  - Permission hierarchy escalation attempts (member trying admin action)
- Implement Rate Limiting bursting tests covering:
  - Burst allowance is honoured (≤burst requests in window succeed)
  - Burst exceeded → 429
  - Window reset behaviour (after window, counter resets)
  - IP fallback path when no API key in context (per `rate_limit.go:54-58`)
- Use the fakes from US-46.7 where available; otherwise construct test doubles per the existing `mocks/` conventions.

**Out:**
- The other 4 categories in `MISSINGTESTS.md` (chaining, context propagation, error handling, nested validation) — those are real gaps but lower-signal; defer to a future testing epic.
- E2E tests (those live in `tests/` and are owned by individual feature epics).
- Rewriting the rate limiter itself (Epic 38 US-38.1 owns that; this story only adds tests that will guard the rewrite).
- Deleting `MISSINGTESTS.md` — only delete it once *all* listed categories are implemented. After this story, update the file to mark items 5 and 6 as done and leave the rest.

## Acceptance Criteria

- [ ] Auth Middleware RBAC tests implemented in `api/internal/middleware/auth_test.go` (or `tests/` per convention).
- [ ] Rate Limit bursting tests implemented in `api/internal/middleware/rate_limit_test.go`.
- [ ] Tests pass with `-race -count=1`.
- [ ] Coverage of `auth.go` middleware paths increases measurably (record before/after percentages).
- [ ] `MISSINGTESTS.md` updated: items 5 and 6 marked complete; remaining items explicitly listed as deferred.
- [ ] Worklog entry.
