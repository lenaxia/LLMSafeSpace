# US-46.15: Fix README-LLM.md Stale References

**Epic:** 46 — Codebase Debt Audit & Abstraction Foundation

**ROI:** Low-Medium
**Depends on:** US-46.14 (archive V1 docs — paths will change)
**Status:** Proposed

## Problem

`README-LLM.md` contains factual errors in its design-doc references:

| README-LLM.md line | Claims | Actual file |
|--------------------|--------|-------------|
| :57 | `design/0007_2025-03-05_network.md` | `design/0020_2025-03-05_network.md` |
| :58 | `design/0006_2025-03-05_runtimeenv.md` | `design/0007_2025-03-05_runtimeenv.md` |
| :185 | `design/0006_2025-03-05_runtimeenv.md` (in Rule 8 table) | `design/0007_2025-03-05_runtimeenv.md` |
| :186 | `design/0007_2025-03-05_network.md` (in Rule 8 table) | `design/0020_2025-03-05_network.md` |

A reader following these links gets "file not found", which undermines trust in the rest of the document. The README is the LLM's bible (per the multi-agent workflow delegation template: *"Primary doc: README-LLM.md (your bible)"*); broken links in the bible are corrosive.

## Scope

**In:**
- Fix the four stale references in `README-LLM.md` (lines 57, 58, 185, 186).
- Grep `README-LLM.md` for every `design/NNNN_` reference and verify each resolves.
- After US-46.14 lands, update V1 doc paths to `design/archive/v1/NNNN_...`.
- Verify the Rule 8 table (README-LLM.md:178-186) matches actual filenames.

**Out:**
- Restructuring README-LLM.md.
- Removing the V1 references entirely (they are still useful as historical context; just point them at the archive).
- Fixing the overlap between README.md and README-LLM.md (separate doc epic).
- Version History entries (those are append-only).

## Acceptance Criteria

- [ ] Every `design/NNNN_` reference in `README-LLM.md` resolves to an existing file.
- [ ] The Rule 8 table matches actual filenames.
