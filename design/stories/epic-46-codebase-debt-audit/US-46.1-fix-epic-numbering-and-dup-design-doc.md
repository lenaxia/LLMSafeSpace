# US-46.1: Fix Epic Numbering Collisions + Delete Duplicate Design Doc

**Epic:** 46 — Codebase Debt Audit & Abstraction Foundation
**Effort:** Trivial (0.25d)
**ROI:** Very High
**Depends on:** —
**Status:** Proposed

---

## Problem

The epic and design-doc numbering system has collisions that break navigation:

1. **`design/0018_2025-03-05_controller-wARmpool.md`** and **`design/0018_2025-03-05_controller-warmpool.md`** are byte-for-byte identical (15153 bytes). The casing difference is the only delta.
2. **Epic 38 is used twice:** `epic-38-architectural-remediation` and `epic-38-session-activity-integrity` both exist in `design/stories/`.
3. **Epic 43 is used twice:** `epic-43-organization-management` and `epic-43-relay-admin-ux` both exist.
4. **Epic 39 is missing** (skipped during numbering).

This makes cross-references in worklogs, PRs, and design docs unreliable.

## Scope

**In:**
- Delete `design/0018_2025-03-05_controller-wARmpool.md` (keep lowercase version).
- Rename one of the epic-38 directories so numbers are unique. Recommended: keep `epic-38-architectural-remediation` (active, 13 stories); renumber `epic-38-session-activity-integrity` to **epic-39-session-activity-integrity** (filling the gap).
- Rename one of the epic-43 directories. Recommended: keep `epic-43-organization-management` (older, larger); renumber `epic-43-relay-admin-ux` to **epic-46.5-relay-admin-ux** OR to **epic-47-relay-admin-ux** (user decision).
- Update `design/stories/README.md` to reflect the renumbering.

**Out:**
- Any content changes to the design docs themselves.
- Updating every worklog that references the old numbers (those are append-only per Rule 5; renumbering note goes in the new worklog 0330).

## Acceptance Criteria

- [ ] `ls design/0018*` returns exactly one file.
- [ ] `ls design/stories/ | grep epic- | awk -F- '{print $2}' | sort | uniq -d` returns empty.
- [ ] `design/stories/README.md` updated with new numbers.
- [ ] `make repolint` (or equivalent) passes — sequence checks unchanged because worklogs are not renumbered.
- [ ] Worklog entry referencing this story.
