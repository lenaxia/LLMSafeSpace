## Problem

One confirmed dead-code location ships in the codebase without justification:

### `controller/internal/relay/gcp_driver.go` (28 lines, all `ErrNotImplemented`)

```go
type GCPDriver struct{}
func (d *GCPDriver) Provision(...) { return nil, ErrNotImplemented }
func (d *GCPDriver) Destroy(...)   { return ErrNotImplemented }
func (d *GCPDriver) GetStatus(...) { return nil, ErrNotImplemented }
func (d *GCPDriver) ListInstances(...) { return nil, ErrNotImplemented }
```

The file's own comment says: *"GCP was dropped from the default fleet (Always Free tier eliminated)."* This satisfies an interface for no purpose; it adds maintenance surface without behaviour. Rule 5 (zero tech debt) requires removal.

## What was originally in scope but is now explicitly OUT of scope

### ~~`annotateModels` remap guard — `api/internal/handlers/models.go:454`~~

This story originally proposed removing the `annotateModels` remap guard based on `README-LLM.md:517` classifying it as "dead code (tech debt to remove)". On adversarial review of the code author's documented reasoning vs the README's classification, **the guard stays**.

**Reasoning** (see worklog 0341 for the full argument):

1. **The code's own comment (`models.go:450-453`) makes a stronger argument than the README does.** The author explicitly designed this as defense-in-depth: *"if an opencode version ever keeps the built-in provider in connected[] despite `disabled_providers`, this code correctly remaps it rather than silently routing to a disabled provider."*

2. **`disabled_providers` is an external mechanism** (opencode honours it today). Defense-in-depth exists precisely for external dependencies whose behaviour can change without warning. The README's claim that `disabled_providers` is "the correct solution" is an opinion — single mechanisms fail, which is why we layer them.

3. **The guard is the result of careful evolution, not accidental cruft.** Worklog 0178 shows the remap was specifically *narrowed* (not added) to this exact form to fix a real `ProviderModelNotFoundError` bug. Worklog 0189 re-reasoned the comments after a follow-up audit. Removing it discards the hardening from two prior incident cycles.

4. **The cost of keeping is ~20 LoC** (4-line conditional + 2 tests + 14-line comment). The cost of removing is silent mis-routing of free-tier models if opencode ever ignores `disabled_providers` — upstream we do not control.

**Action item:** A separate follow-up should update `README-LLM.md:517` to remove the "tech debt to remove" classification and instead document the guard as intentional defense-in-depth (consistent with the code's own comment). That edit is out of scope here because it touches the README's narrative, not this story.
