# US-46.9: Type MCP Request Bodies

**Epic:** 46 — Codebase Debt Audit & Abstraction Foundation
**Effort:** Small (1d)
**ROI:** Medium
**Depends on:** —
**Status:** Proposed

---

## Problem

`pkg/mcp/client.go` builds five JSON request bodies using `map[string]interface{}` / `map[string]any`:

```
pkg/mcp/client.go:301:  result := map[string]interface{}{...}
pkg/mcp/client.go:358:  body := map[string]interface{}{"answers": answers}
pkg/mcp/client.go:379:  body := map[string]interface{}{"reply": reply}
pkg/mcp/client.go:390:  providerData := map[string]any{"provider": req.Provider, "apiKey": req.APIKey}
pkg/mcp/client.go:404:  body := map[string]any{...}
```

This violates Rule 1 (Type Safety First). The MCP server is the project's externally-visible integration surface (per README-LLM.md:39 *"any MCP-compatible client can connect"*); untyped request bodies mean a field rename or typo compiles and silently sends malformed requests.

## Scope

**In:**
- For each of the 5 request bodies in `pkg/mcp/client.go`, define a struct with `json:` tags:
  ```go
  type questionReplyRequest struct {
      Answers []string `json:"answers"`
  }
  type permissionReplyRequest struct {
      Reply string `json:"reply"`
  }
  type providerSubmitRequest struct {
      Provider string `json:"provider"`
      APIKey   string `json:"apiKey"`
  }
  // ... etc for each body
  ```
- Replace the `map[string]any{}` literal with a struct literal + `json.Marshal`.
- Update tests to assert on the typed request, not the raw map.
- Where the response is decoded into `map[string]interface{}` (e.g. `pkg/mcp/client.go:301`), define a response struct.

**Out:**
- Changing the wire format of any MCP request or response (pure refactor — same JSON bytes, just constructed from a typed struct).
- The MCP *server* (`pkg/mcp/server.go`) `strArg` helper (line 205) — it correctly accepts `map[string]any` for incoming args because MCP args are inherently dynamic; leave it.
- Adding new MCP tools or methods.

## Acceptance Criteria

- [ ] `grep -n 'map\[string\]interface{}\|map\[string\]any' pkg/mcp/client.go` returns 0 matches (excluding tests).
- [ ] Each of the 5 request bodies has a named struct in `pkg/mcp/` (or `pkg/mcp/types.go`).
- [ ] Wire format unchanged: snapshot tests of the JSON bytes confirm no behaviour change.
- [ ] All existing tests pass.
- [ ] Worklog entry.
