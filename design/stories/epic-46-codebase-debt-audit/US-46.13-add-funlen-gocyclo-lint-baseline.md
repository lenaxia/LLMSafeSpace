# US-46.13: Add `funlen` / `gocyclo` Lint with Baseline

**Epic:** 46 — Codebase Debt Audit & Abstraction Foundation
**Effort:** Small (0.5d)
**ROI:** Medium
**Depends on:** US-46.6 (split main.go — without it, baseline would need to grandfather a 1451-line file)
**Status:** Proposed

---

## Problem

`.golangci.yml` deliberately excludes `funlen` and `gocyclo`:

```yaml
# Deliberately NOT enabled (would add noise without dedicated cleanup):
#   - funlen / gocyclo (subjective, lots of false positives)
```

The comment is correct *today* — enabling them cold would generate hundreds of violations. But the absence of any complexity guard is what allowed `main.go` to grow to 1451 lines and `main()` to grow to 367 lines in the first place. After US-46.6 splits the worst offenders, the linter can be enabled with a **baseline file** that grandfathers existing violations while blocking new ones.

## Scope

**In:**
- Enable `funlen` (suggested threshold: 60 lines, slightly above Rule 4's 50-line guidance to allow a grace band) and `gocyclo` (suggested threshold: 15) in `.golangci.yml`.
- Generate a baseline using `golangci-lint run --out-format=line-number > .golangci-baseline.txt` (or use golangci-lint v2's native baseline feature if available).
- Configure the linter to **fail on new violations**, **warn on baseline violations**.
- Document in `.golangci.yml` how to regenerate the baseline and the policy for shrinking it (target: zero grandfathered violations within 2 quarters).
- Add a CI check verifying no new violations outside the baseline.

**Out:**
- Fixing every existing violation (those are tracked by the baseline; teams chip away at them via normal PRs).
- Enabling other excluded linters (`revive`, `lll`, `tagliatelle`) — out of scope.
- Changing the existing linter configuration beyond adding `funlen` + `gocyclo`.

## Acceptance Criteria

- [ ] `funlen` and `gocyclo` enabled in `.golangci.yml` with thresholds documented.
- [ ] Baseline file committed at `.golangci-baseline.txt` (or equivalent v2 mechanism).
- [ ] `make lint` passes on `main` (the baseline covers existing violations).
- [ ] A test PR that adds a 100-line function is rejected by CI.
- [ ] CI workflow (`ci.yml`) updated to enforce baseline.
- [ ] `.golangci.yml` comment updated to explain the new policy.
- [ ] Worklog entry.
