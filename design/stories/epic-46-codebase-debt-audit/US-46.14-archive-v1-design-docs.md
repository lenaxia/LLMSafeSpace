## Problem

`README-LLM.md:52` states: *"V1 design docs (reference only — superseded by evolution-v2.md where they conflict)"*. Twenty V1 design docs (`design/0001` through `design/0020`) live alongside the active V2 docs (`design/0021`+), polluting directory navigation and making it ambiguous which docs are authoritative.

The risk: an engineer reading `design/0003_controller.md` (V1 Sandbox CRD) implements against V1 assumptions, not realising `design/0021_evolution-v2.md` supersedes it. README-LLM.md calls out the V1 docs as superseded, but a reader landing directly in `design/` does not see that warning.

## Scope

**In:**
- Create `design/archive/v1/` directory.
- `git mv` `design/0001_*.md` through `design/0020_*.md` into `design/archive/v1/`.
- Add a `design/archive/v1/README.md` explaining: these docs are V1, superseded by `design/0021_evolution-v2.md`, kept for historical reference only.
- Update `README-LLM.md:52-58` to reflect the new paths (`design/archive/v1/0001_...` instead of `design/0001_...`).
- Update any cross-references in active docs (`design/0021`+) that point to V1 docs.

**Out:**
- Deleting V1 docs (they have historical value; archive, don't delete).
- Archiving V2 docs (`0021`+ are active).
- Archiving the four "design review" docs (`0033`–`0036`) — those are recent (2026-06-15) and may still be referenced; defer.
- Editing V1 doc contents.
