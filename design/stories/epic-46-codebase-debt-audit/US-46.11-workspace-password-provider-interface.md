# US-46.11: Define `WorkspacePasswordProvider` Interface

**Epic:** 46 — Codebase Debt Audit & Abstraction Foundation
**Effort:** Small (0.5d)
**ROI:** Medium
**Depends on:** —
**Status:** Proposed

---

## Problem

`SecretsHandler` has a dependency injected as a **function type**:

```go
type SecretsHandler struct {
    // ...
    passwordGetter func(ctx context.Context, workspaceID string) (string, error)
}

func (h *SecretsHandler) SetPasswordGetter(getter func(ctx context.Context, workspaceID string) (string, error)) {
    h.passwordGetter = getter
}
```

Function-typed dependencies are:
- **Unnamed** — the signature doesn't express intent; readers must find every call site to understand what it does.
- **Unmockable by name** — tests must construct an inline closure; IDEs and mock generators can't auto-generate a fake.
- **Inconsistent with the rest of the handler** — every other dependency (`PodIPResolver`, `SecretsManifestWriter`, `PasswordVerifier`) is a named interface; only `passwordGetter` is a function type.

This is a Tier A abstraction (≥2 consumers: `SecretsHandler` and `ModelsHandler` both need the workspace password for opencode Basic auth). It complements US-29.1 (`AgentClient`) but is smaller in scope — `AgentClient` covers all opencode HTTP calls; this story only covers password retrieval.

## Scope

**In:**
- Define `WorkspacePasswordProvider` interface in `api/internal/handlers/` (alongside the existing `PodIPResolver`, `PasswordVerifier`):
  ```go
  type WorkspacePasswordProvider interface {
      WorkspacePassword(ctx context.Context, workspaceID string) (string, error)
  }
  ```
- Replace the `passwordGetter func(...)` field on `SecretsHandler` with `passwordProvider WorkspacePasswordProvider`.
- Update the constructor and setter (this story does not remove the setter — that's US-29.8; it only renames the type).
- Find `ModelsHandler` (and any other handler with the same function-typed injection) and migrate it too.
- Update tests to use a named fake `fakeWorkspacePasswordProvider` instead of an inline closure.

**Out:**
- Removing the setter (US-29.8 owns that).
- Extracting the full `AgentClient` (US-29.1 owns that; this story is the narrower prerequisite).
- Changing the password retrieval implementation (the K8s secret lookup stays as-is; only the handler's view of it changes).
- Any new behaviour.

## Acceptance Criteria

- [ ] `WorkspacePasswordProvider` interface defined.
