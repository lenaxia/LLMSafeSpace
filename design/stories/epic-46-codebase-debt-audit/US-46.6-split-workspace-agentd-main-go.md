# US-46.6: Split `cmd/workspace-agentd/main.go`

**Epic:** 46 — Codebase Debt Audit & Abstraction Foundation
**Effort:** Medium (3d)
**ROI:** High
**Depends on:** —
**Status:** Proposed

---

## Problem

`cmd/workspace-agentd/main.go` is **1451 lines with 43 functions**. The `main()` function alone spans lines 770→1137 (~367 lines). It mixes six distinct responsibilities:

1. **HTTP client to opencode** (`OpenCodeClient` — `IsHealthy`, `ConnectedProviders`, `ConfiguredProviderCount`, `ModelContextLimit`, `ListSessions`, `fetchSessionTitle`, `fetchSessionPromptTokens`)
2. **Session status tracking** (`sessionStatusTracker` — `set`, `get`, `hasAnyBusy`, `listBusy`, `hasAnyData`, `prune`, `subscribe`, `connectAndRead`, `processEvent`, `handleSessionStatus`, `handleStepEnded`)
3. **Process supervision** (`managedProcess` — `start`, `supervise`, `applyBackoff`, `restart`, `stop`, `healthProbeAfterRestart`, `defaultOpencodeCmdFactory`)
4. **System metrics** (`getMemoryUsage`, `getDiskUsage`, `getCPUUsage`)
5. **HTTP server wiring** (`buildStatuszHandler`, `requireBearerToken`, all the `defer resp.Body.Close()` handlers)
6. **Boot/shutdown orchestration** (the 367-line `main()` itself)

This violates Rule 4 (functions ≤50 lines, small focused units) and makes every change to one concern touch a file that eight other engineers/agents might be editing.

## Scope

**In:**
- Extract each responsibility into its own file or subpackage under `cmd/workspace-agentd/`:
  - `client.go` (or `internal/agentclient/`) — `OpenCodeClient` and all its methods
  - `session_tracker.go` (or `internal/sessiontracker/`) — `sessionStatusTracker` and its methods
  - `process.go` (or `internal/processsupervisor/`) — `managedProcess` and its methods
  - `metrics.go` (or `internal/sysmetrics/`) — `getMemoryUsage`, `getDiskUsage`, `getCPUUsage`
  - `server.go` — `buildStatuszHandler`, `requireBearerToken`, HTTP handler closures
  - `main.go` — boot sequence + shutdown only, ≤300 lines
- Move associated tests alongside (e.g. `main_test.go` content splits to match the new files).
- Keep the package name `main` (cmd binaries stay in `package main` for `go build` to work) OR move helpers to internal subpackages imported by main.
- No behaviour change — pure refactor. Every test that passed before passes after.

**Out:**
- Redesigning the boot sequence or shutdown order (that's US-46.10).
- Adding new tests beyond what's needed to cover the move.
- Splitting `secrets.go` (594 lines) or `relay_injector.go` (480 lines) — those are already separate files; they may be touched only if a circular import emerges from the split.
- Changing the public HTTP API of agentd (routes, response shapes).

## Acceptance Criteria

- [ ] `wc -l cmd/workspace-agentd/main.go` returns ≤300.
- [ ] No file under `cmd/workspace-agentd/` exceeds 400 lines.
- [ ] `main()` is ≤80 lines (delegates to functions in other files/packages).
- [ ] Each responsibility lives in a clearly-named file (the file name matches the responsibility).
- [ ] `go build ./cmd/workspace-agentd/...` succeeds.
- [ ] `go test -timeout 60s -race ./cmd/workspace-agentd/...` passes.
- [ ] `make lint` passes (US-46.13 will add `funlen`/`gocyclo` to enforce this going forward).
- [ ] Worklog entry with file-by-file breakdown.
