# US-46.4: Introduce `DomainError` Type

**Epic:** 46 — Codebase Debt Audit & Abstraction Foundation
**Effort:** Small (1d)
**ROI:** High
**Depends on:** —
**Status:** Done (2026-06-19, worklog 0365)

---

## Problem

Rule 2 requires *"custom error types for domain-specific errors (see `api/internal/errors/errors.go`)"*. The codebase has **43 sentinel errors** (`var ErrX = errors.New(...)`) but only **4 custom error types** (outside SDK code). Callers cannot use `errors.As` to distinguish domain conditions — they must fall back to string matching (`strings.Contains(err.Error(), "...")`), which is fragile and forbidden by the same rule.

Audit counts:
- `grep -rn "errors.New\|fmt.Errorf" --include="*.go" | grep -v _test.go` → **995 occurrences**
- `grep -rn "type.*Error struct" --include="*.go" | grep -v _test.go` → **7 types, of which 4 are domain-relevant** (`WorkspaceNotFoundError`, `validationError`, `probeError`, `nonRetryableError`)
- `grep -rn "errors.Is\|errors.As" --include="*.go" | grep -v _test.go` → **93 call sites**

The 93 call sites prove consumers want typed errors; the 995 inline errors prove producers don't provide them.

## Scope

**In:**
- Define a `DomainError` type in `api/internal/errors/errors.go`:
  ```go
  type DomainError struct {
      Code     string        // stable, machine-readable, e.g. "workspace_not_found"
      Message  string        // user-safe message (no secrets, no internal paths)
      Cause    error         // wrapped underlying error, if any
      Category ErrorCategory // NotFound, Conflict, Validation, Unauthorized, Forbidden, RateLimited, Internal
  }
  type ErrorCategory string
  const (
      CategoryNotFound     ErrorCategory = "not_found"
      CategoryConflict     ErrorCategory = "conflict"
      CategoryValidation   ErrorCategory = "validation"
      CategoryUnauthorized ErrorCategory = "unauthorized"
      CategoryForbidden    ErrorCategory = "forbidden"
      CategoryRateLimited  ErrorCategory = "rate_limited"
      CategoryInternal     ErrorCategory = "internal"
  )
  func (e *DomainError) Error() string { /* ... */ }
  func (e *DomainError) Unwrap() error { return e.Cause }
  ```
- Provide constructors: `NewNotFound(code, msg string)`, `NewConflict(...)`, `NewValidation(...)`, etc.
- Migrate **≥10 of the 43 sentinel errors** to wrap `DomainError`. Pick the highest-value ones first: `ErrWorkspaceNotFound`, `ErrNoAgentStateRow`, `ErrPartialFailure`, `ErrAutoBindingProtected`, `ErrTooManySubscribers`, plus 5 more from the audit list.
- Migrate the call sites of those 10 errors from `strings.Contains` / `err == ErrX` to `errors.As(err, &domainErr)` + switch on `domainErr.Category`.
- Update the handler error mapping (`respondWithError` in `api/internal/server/router.go`, `handleSecretError` in `api/internal/handlers/secrets.go`) to map `DomainError.Category` → HTTP status code in one place.

**Out:**
- Migrating all 995 inline errors (out of scope — this story establishes the pattern and migrates 10; later stories or PRs adopt it organically).
- Changing HTTP status codes returned to clients (preserve current behaviour — only the internal representation changes).
- Introducing a generic `WrappedError` or `AppError` hierarchy (Rule 4 forbids speculative generality; one typed error + categories is sufficient).

## Acceptance Criteria

- [ ] `DomainError` type defined in `api/internal/errors/errors.go` with 7 categories and constructor helpers.
- [ ] ≥10 sentinel errors migrated to constructors.
- [ ] `grep -rn "errors.As" --include="*.go" | grep -v _test.go` count increases by ≥10.
- [ ] `grep -rn 'strings.Contains.*err.Error' --include="*.go" | grep -v _test.go` count decreases (callers switched to `errors.As`).
- [ ] HTTP status mapping centralised in one place (router error handler).
- [ ] All existing tests pass; no behaviour change.
- [ ] Worklog entry documenting which errors were migrated and why.
