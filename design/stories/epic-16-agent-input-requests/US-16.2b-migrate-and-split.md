# US-16.2b: Migrate Session Routes to Dialect + Split proxy.go

**Epic:** 16 - Agent Input Requests
**Priority:** Medium
**Depends on:** US-16.2a
**Layer:** Backend

## User Story

As a platform developer preparing for a second agent runtime, I need the existing session proxy handlers to use the Dialect interface for path generation, and I need proxy.go split into focused files, so that adding a new agent doesn't require modifying handler logic.

## Acceptance Criteria

- [ ] All existing session handlers (`CreateSession`, `ListSessions`, `SendMessage`, `SendPromptAsync`, `GetHistory`, `GetSession`, `AbortSession`) use `h.dialect.SessionXxxPath()` instead of hardcoded string concatenation
- [ ] `proxy.go` is split into focused files (see structure below)
- [ ] Zero behavioral change ‚Äî all existing tests pass without modification (pure refactor)
- [ ] No new features, no new tests (existing coverage validates the refactor)

### Session Handler Migration

Before:
```go
func (h *ProxyHandler) SendMessage(c *gin.Context) {
    sid := c.Param("sessionId")
    h.proxyToWorkspace(c, "/session/"+sid+"/message", true, sid)
}
```

After:
```go
func (h *ProxyHandler) SendMessage(c *gin.Context) {
    sid := c.Param("sessionId")
    h.proxyToWorkspace(c, h.dialect.SessionMessagePath(sid), true, sid)
}
```

### Validation Approach

1. Run `go test -race ./api/internal/handlers/...` before the refactor (baseline)
2. Apply the refactor (file split + dialect migration)
3. Run the same tests ‚Äî must produce identical results
4. Run `go vet ./api/internal/handlers/...` ‚Äî no issues
5. Verify `git diff --stat` shows only renames/moves + the path string changes
