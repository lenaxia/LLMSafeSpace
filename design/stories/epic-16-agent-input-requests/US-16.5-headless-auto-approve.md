# US-16.5: Headless Permission Auto-Approve

**Epic:** 16 - Agent Input Requests
**Priority:** High
**Depends on:** US-16.2, US-16.3
**Layer:** Backend

## User Story

As a workspace owner using programmatic access (MCP, API automation), I need a workspace-level setting to auto-approve permissions, so that the agent never blocks waiting for human input when I've explicitly opted into unattended operation.

## Acceptance Criteria

- [ ] New workspace CRD field: `spec.autoApprovePermissions` (boolean, default: false)
- [ ] When `autoApprovePermissions=true` and a `permission.asked` event is detected, the backend auto-approves with `"always"`
- [ ] When `autoApprovePermissions=false` (default), permissions are published as `agent.permission` events for the browser to handle
- [ ] The auto-approve calls `POST /permission/:requestID/reply` on the pod with `{"reply":"always"}`
- [ ] The auto-approve happens asynchronously (does not block event processing)
- [ ] Auto-approve failures are logged at Warn level but do not crash
- [ ] Questions are NEVER auto-answered regardless of this setting (they always require human input)
- [ ] The setting can be changed via the workspace update API (existing PATCH endpoint)

### Auto-Approve Logic in `emitNormalizedInputEvent`

```go
func (h *ProxyHandler) emitNormalizedInputEvent(workspaceID, eventType, rawData string) {
    // ... existing parsing ...

    if h.dialect.IsPermissionAsked(eventType) {
        req, err := h.dialect.ParsePermissionRequest(eventType, envelope.Properties)
        if err != nil {
            h.logger.Warn("Failed to parse permission event", "error", err)
            return
        }

        // Check workspace setting
        if h.shouldAutoApprovePermissions(workspaceID) {
            go h.autoApprovePermission(workspaceID, req.ID)
            return
        }

        // Publish for browser/MCP handling
        h.broker.Publish(workspaceID, WorkspaceSSEEvent{
            Type: "agent.permission",
            Data: req,
        })
    }
}

func (h *ProxyHandler) shouldAutoApprovePermissions(workspaceID string) bool {
    workspace, err := h.k8sClient.LlmsafespaceV1().Workspaces(h.namespace).Get(workspaceID, metav1.GetOptions{})
    if err != nil {
        return false // fail closed
    }
    return workspace.Spec.AutoApprovePermissions
}
```

### `autoApprovePermission` Implementation

```go
func (h *ProxyHandler) autoApprovePermission(workspaceID, requestID string) {
    ctx, cancel := context.WithTimeout(context.Background(), 10*time.Second)
    defer cancel()

    workspace, err := h.k8sClient.LlmsafespaceV1().Workspaces(h.namespace).Get(workspaceID, metav1.GetOptions{})
    if err != nil || workspace.Status.PodIP == "" {
        h.logger.Warn("Cannot auto-approve permission: workspace not reachable",
            "workspaceID", workspaceID, "requestID", requestID)
        return
    }

    password, err := h.getPassword(ctx, workspaceID)
    if err != nil {
        h.logger.Warn("Cannot auto-approve permission: password unavailable",
            "workspaceID", workspaceID, "requestID", requestID)
        return
    }

    targetPath := h.dialect.PermissionReplyPath(requestID)
    url := fmt.Sprintf("http://%s:%d%s", workspace.Status.PodIP, opencodePort, targetPath)

    body := []byte(`{"reply":"always"}`)
    req, err := http.NewRequestWithContext(ctx, http.MethodPost, url, bytes.NewReader(body))
    if err != nil {
        return
    }
    req.SetBasicAuth("opencode", password)
    req.Header.Set("Content-Type", "application/json")

    resp, err := h.httpClient.Do(req)
    if err != nil {
        h.logger.Warn("Auto-approve permission failed", "error", err,
            "workspaceID", workspaceID, "requestID", requestID)
        return
    }
    resp.Body.Close()

    h.logger.Info("Auto-approved permission",
        "workspaceID", workspaceID, "requestID", requestID)
}
```

### Why Workspace Setting (Not Subscriber Count)

The previous design used `SubscriberCount(workspaceID) == 0` as a proxy for "headless mode." This is wrong because:
1. MCP clients subscribe to SSE too (in `SendMessage`) — they'd count as subscribers
2. A browser could be connected but the user stepped away — still counts as subscriber
3. The intent is explicit: "I want this workspace to auto-approve" vs. "I happen to not have a browser open"

A workspace setting is:
- Explicit (user opts in)
- Deterministic (no race conditions)
- Inspectable (visible in CRD spec)
- Testable (set the field, verify behavior)

### Caching Consideration

`shouldAutoApprovePermissions` reads the workspace CRD on every permission event. Permissions are rare (only fire when configured), so this is acceptable. If it becomes a hot path, cache the setting in `wsConfig` (already cached for `maxActiveSessions`).
