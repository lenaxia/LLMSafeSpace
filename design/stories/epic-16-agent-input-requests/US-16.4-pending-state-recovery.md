# US-16.4: Pending State Recovery on SSE Connect

**Epic:** 16 - Agent Input Requests
**Priority:** High
**Depends on:** US-16.2, US-16.3
**Layer:** Backend

## User Story

As a user who refreshes the page while the agent is waiting for my answer, I need the question/permission prompt to re-appear immediately, so that I can answer it without the agent being stuck forever.

## Acceptance Criteria

- [ ] When a browser connects to `GET /workspaces/:id/events` (StreamEvents handler), the backend fetches pending questions and permissions from the pod
- [ ] For each pending question, a synthetic `agent.question` event is published to the newly-connected subscriber
- [ ] For each pending permission, a synthetic `agent.permission` event is published to the newly-connected subscriber
- [ ] The fetch happens asynchronously (does not block the SSE connection setup)
- [ ] If the workspace is not Active (no pod IP), the fetch is skipped (no error)
- [ ] If the fetch fails (pod unreachable, timeout), it is logged at Warn and does not affect the SSE connection
- [ ] The synthetic events are identical in shape to real-time events (frontend cannot distinguish them)
- [ ] If no questions/permissions are pending, no synthetic events are published

## Technical Details

### Implementation in `StreamEvents`

After the SSE connection is established and the subscriber channel is created, spawn a goroutine:

```go
func (h *ProxyHandler) StreamEvents(c *gin.Context) {
    // ... existing setup (workspace exists check, flusher, broker subscribe) ...

    ch := h.broker.Subscribe(workspaceID)
    defer h.broker.Unsubscribe(workspaceID, ch)

    // NEW: Emit any pending input requests so reconnecting browsers see them immediately
    go h.emitPendingInputRequests(workspaceID)

    // ... existing SSE loop ...
}
```

### `emitPendingInputRequests` Implementation

```go
func (h *ProxyHandler) emitPendingInputRequests(workspaceID string) {
    ctx, cancel := context.WithTimeout(context.Background(), 5*time.Second)
    defer cancel()

    workspace, err := h.k8sClient.LlmsafespaceV1().Workspaces(h.namespace).Get(workspaceID, metav1.GetOptions{})
    if err != nil || workspace.Status.Phase != phaseActive || workspace.Status.PodIP == "" {
        return
    }

    password, err := h.getPassword(ctx, workspaceID)
    if err != nil {
        return
    }

    podIP := workspace.Status.PodIP

    // Fetch and emit pending questions
    if body, err := h.fetchFromPod(ctx, podIP, password, h.dialect.QuestionListPath()); err == nil {
        for _, req := range h.parseQuestionList(body) {
            h.broker.Publish(workspaceID, WorkspaceSSEEvent{Type: "agent.question", Data: req})
        }
    }

    // Fetch and emit pending permissions
    if body, err := h.fetchFromPod(ctx, podIP, password, h.dialect.PermissionListPath()); err == nil {
        for _, req := range h.parsePermissionList(body) {
            h.broker.Publish(workspaceID, WorkspaceSSEEvent{Type: "agent.permission", Data: req})
        }
    }
}

func (h *ProxyHandler) fetchFromPod(ctx context.Context, podIP, password, path string) ([]byte, error) {
    url := fmt.Sprintf("http://%s:%d%s", podIP, opencodePort, path)
    req, err := http.NewRequestWithContext(ctx, http.MethodGet, url, nil)
    if err != nil {
        return nil, err
    }
    req.SetBasicAuth("opencode", password)

    resp, err := h.httpClient.Do(req)
    if err != nil {
        h.logger.Warn("Failed to fetch pending input requests", "error", err, "path", path)
        return nil, err
    }
    defer resp.Body.Close()

    if resp.StatusCode != http.StatusOK {
        return nil, fmt.Errorf("unexpected status %d", resp.StatusCode)
    }
    return io.ReadAll(resp.Body)
}
```

### Parse Helpers

```go
func (h *ProxyHandler) parseQuestionList(body []byte) []*agent.QuestionRequest {
    var raw []json.RawMessage
    if json.Unmarshal(body, &raw) != nil {
        return nil
    }
    var results []*agent.QuestionRequest
    for _, r := range raw {
        if req, err := h.dialect.ParseQuestionRequest("question.asked", r); err == nil {
            results = append(results, req)
        }
    }
    return results
}

func (h *ProxyHandler) parsePermissionList(body []byte) []*agent.PermissionRequest {
    var raw []json.RawMessage
    if json.Unmarshal(body, &raw) != nil {
        return nil
    }
    var results []*agent.PermissionRequest
    for _, r := range raw {
        if req, err := h.dialect.ParsePermissionRequest("permission.asked", r); err == nil {
            results = append(results, req)
        }
    }
    return results
}
```

### Timing Consideration

The goroutine runs after `h.broker.Subscribe(workspaceID)` returns the channel. Events published by the goroutine will be delivered to the channel. The SSE write loop (which reads from the channel) starts immediately after. So the synthetic events will be among the first events the browser receives.

Race condition: if a real `question.asked` event arrives between the subscribe and the fetch completing, the browser may receive a duplicate. This is handled by frontend deduplication (US-16.11 — dedup by `id` field).

## Test Cases

| # | Test | Scenario | Expected |
|---|------|----------|----------|
| 1 | No pending items | Pod returns `[]` for both endpoints | No synthetic events published |
| 2 | One pending question | Pod returns `[{id:"que_1",...}]` | `agent.question` event published with parsed request |
| 3 | Multiple pending questions | Pod returns 3 questions | 3 `agent.question` events published |
| 4 | One pending permission | Pod returns `[{id:"per_1",...}]` | `agent.permission` event published |
| 5 | Workspace not Active | Phase = Suspended | No fetch attempted, no error |
| 6 | Pod unreachable | HTTP timeout | Warning logged, SSE connection unaffected |
| 7 | Pod returns 500 | Server error | No events published, no crash |
| 8 | Concurrent real event + synthetic | Real `question.asked` arrives during fetch | Both events published (frontend deduplicates) |
