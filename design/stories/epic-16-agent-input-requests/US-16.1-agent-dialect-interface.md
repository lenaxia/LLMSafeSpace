# US-16.1: Agent Dialect Interface + OpenCode Implementation

**Epic:** 16 - Agent Input Requests
**Priority:** Critical
**Depends on:** None
**Layer:** Backend

## User Story

As a platform developer, I need an abstraction that encodes agent-specific HTTP routes and event formats, so that swapping the underlying agent (opencode → claude-code) requires implementing one interface rather than modifying proxy logic, router, MCP server, and frontend.

- [ ] `pkg/agent/dialect.go` defines the `Dialect` interface with methods for route mapping and event classification

- [ ] `pkg/agent/opencode/dialect.go` implements `Dialect` for opencode

- [ ] Event classification methods correctly identify question/permission events from the SSE stream
- [ ] `ParseQuestionRequest` correctly extracts the normalized type from raw opencode event JSON

- [ ] Unit tests cover all dialect methods with real event payloads captured in worklog 0069

## Technical Details

### Interface Definition (`pkg/agent/dialect.go`)

```go
package agent

import "encoding/json"

// Dialect encodes the HTTP API contract of the agent running inside a workspace pod.
// Implement this interface for each supported agent runtime.
type Dialect interface {
    // --- Session route paths (pod-internal, relative) ---
    SessionCreatePath() string
    SessionListPath() string
    SessionMessagePath(sessionID string) string
    SessionPromptAsyncPath(sessionID string) string
    SessionAbortPath(sessionID string) string
    SessionGetPath(sessionID string) string
    EventStreamPath() string

    // --- Input request route paths ---
    QuestionListPath() string
    QuestionReplyPath(requestID string) string
    QuestionRejectPath(requestID string) string
    PermissionListPath() string
    PermissionReplyPath(requestID string) string

    // --- SSE event classification ---
    IsQuestionAsked(eventType string) bool
    IsQuestionResolved(eventType string) bool
    IsPermissionAsked(eventType string) bool
    IsPermissionResolved(eventType string) bool
    IsSessionIdle(eventType string, properties json.RawMessage) bool
    IsSessionBusy(eventType string, properties json.RawMessage) bool

    // --- Event parsing (returns nil if event doesn't match) ---
    ParseQuestionRequest(eventType string, properties json.RawMessage) (*QuestionRequest, error)
    ParsePermissionRequest(eventType string, properties json.RawMessage) (*PermissionRequest, error)
    ParseSessionStatus(properties json.RawMessage) (sessionID string, status string, err error)
}
```

### Normalized Types (`pkg/agent/types.go`)

```go
package agent

// QuestionOption is a single selectable choice within a question.
type QuestionOption struct {
    Label       string `json:"label"`
    Description string `json:"description"`
}

// QuestionInfo is a single question with its options.
type QuestionInfo struct {
    Question string           `json:"question"`
    Header   string           `json:"header"`
    Options  []QuestionOption `json:"options"`
    Multiple bool             `json:"multiple"`
    Custom   bool             `json:"custom"` // Always true when absent from source event
}

// QuestionRequest is the normalized, agent-agnostic representation of a pending question.
type QuestionRequest struct {
    ID        string         `json:"id"`
    SessionID string         `json:"session_id"`
    Questions []QuestionInfo `json:"questions"`
    Tool      *ToolRef       `json:"tool,omitempty"`
}

// PermissionRequest is the normalized, agent-agnostic representation of a pending permission.
type PermissionRequest struct {
    ID         string                 `json:"id"`
    SessionID  string                 `json:"session_id"`
    Permission string                 `json:"permission"`
    Patterns   []string               `json:"patterns"`
    Metadata   map[string]interface{} `json:"metadata,omitempty"`
    Always     []string               `json:"always,omitempty"`
    Tool       *ToolRef               `json:"tool,omitempty"`
}

// ToolRef identifies the tool call that triggered the input request.
type ToolRef struct {
    MessageID string `json:"message_id"`
    CallID    string `json:"call_id"`
}
```

### OpenCode Implementation (`pkg/agent/opencode/dialect.go`)

Key implementation details:
- Route paths: `/question`, `/question/{id}/reply`, `/question/{id}/reject`, `/permission`, `/permission/{id}/reply`
- Event types: `question.asked`, `question.replied`, `question.rejected`, `permission.asked`, `permission.replied`
- Session status: `session.status` with `properties.status.type == "idle"|"busy"`
- `ParseQuestionRequest`: unmarshal `properties` as opencode's `QuestionRequest` shape, map to normalized type, set `Custom=true` when field is absent
- `ParsePermissionRequest`: unmarshal `properties` as opencode's `PermissionRequest` shape, map to normalized type

### ID Validation

opencode IDs use underscores (`que_abc`, `per_xyz`, `ses_123`). The existing `validID` regex in `pkg/mcp/client.go` does NOT match underscores. Add a separate validator:

```go
// validAgentID matches opencode-style IDs: prefix + underscore + alphanumeric
var validAgentID = regexp.MustCompile(`^(que|per|ses|msg|evt)_[a-zA-Z0-9]{1,253}$`)
```
