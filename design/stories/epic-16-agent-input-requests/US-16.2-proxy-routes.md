# US-16.2: Question/Permission Proxy Routes

**Epic:** 16 - Agent Input Requests
**Priority:** Critical
**Depends on:** US-16.1
**Layer:** Backend

## User Story

As a frontend or MCP client, I need REST endpoints to list pending questions/permissions and submit replies, so that I can interact with the agent's input request system through the LLMSafeSpace API.

## Acceptance Criteria

- [ ] 5 new routes registered in `registerProxyRoutes`:
  - `GET  /:id/question` â†’ proxy to dialect.QuestionListPath()
  - `POST /:id/question/:requestID/reply` â†’ proxy to dialect.QuestionReplyPath(requestID)
  - `POST /:id/question/:requestID/reject` â†’ proxy to dialect.QuestionRejectPath(requestID)
  - `GET  /:id/permission` â†’ proxy to dialect.PermissionListPath()
  - `POST /:id/permission/:requestID/reply` â†’ proxy to dialect.PermissionReplyPath(requestID)
- [ ] All routes go through existing auth middleware (same as session proxy routes)
- [ ] All routes use `proxyToWorkspace` pattern (workspace must be Active, pod IP resolved)
- [ ] Request body is forwarded as-is (no transformation)
- [ ] Response is forwarded as-is (no filtering, no buffering)
- [ ] `requestID` parameter is validated: must start with `que_` or `per_` respectively, max 253 chars, alphanumeric + underscore only
- [ ] Invalid `requestID` returns 400 without hitting the pod

## Technical Details

### Route Registration

In `api/internal/server/router.go`, add to `registerProxyRoutes`:

```go
func registerProxyRoutes(rg *gin.RouterGroup, proxyHandler *handlers.ProxyHandler) {
    // Existing session routes
    rg.POST("/:id/sessions/:sessionId/message", proxyHandler.SendMessage)
    rg.POST("/:id/sessions/:sessionId/prompt", proxyHandler.SendPromptAsync)
    rg.GET("/:id/sessions/:sessionId/message", proxyHandler.GetHistory)
    rg.GET("/:id/sessions/:sessionId", proxyHandler.GetSession)
    rg.POST("/:id/sessions/:sessionId/abort", proxyHandler.AbortSession)
    rg.GET("/:id/events", proxyHandler.StreamEvents)

    // New: Question/Permission input request routes
    rg.GET("/:id/question", proxyHandler.ListQuestions)
    rg.POST("/:id/question/:requestID/reply", proxyHandler.QuestionReply)
    rg.POST("/:id/question/:requestID/reject", proxyHandler.QuestionReject)
    rg.GET("/:id/permission", proxyHandler.ListPermissions)
    rg.POST("/:id/permission/:requestID/reply", proxyHandler.PermissionReply)
}
```

### Handler Implementation

Each handler follows the same pattern:

```go
func (h *ProxyHandler) QuestionReply(c *gin.Context) {
    requestID := c.Param("requestID")
    if !isValidQuestionID(requestID) {
        c.JSON(http.StatusBadRequest, gin.H{"error": "invalid request ID format"})
        return
    }
    targetPath := h.dialect.QuestionReplyPath(requestID)
    h.proxyToWorkspace(c, targetPath, false, "")
}
```

### ID Validation

```go
var questionIDPattern = regexp.MustCompile(`^que_[a-zA-Z0-9]{1,250}$`)
var permissionIDPattern = regexp.MustCompile(`^per_[a-zA-Z0-9]{1,250}$`)

func isValidQuestionID(id string) bool { return questionIDPattern.MatchString(id) }
func isValidPermissionID(id string) bool { return permissionIDPattern.MatchString(id) }
```

### ProxyHandler Changes

The `ProxyHandler` struct gains a `dialect` field:

```go
type ProxyHandler struct {
    dialect agent.Dialect  // NEW
    // ... existing fields
}
```

Injected via `NewProxyHandler` (add parameter). The existing session route handlers are migrated to use dialect paths in this story â€” replacing the hardcoded `/session/` string concatenation. This ensures the Dialect is the single source of truth for ALL agent route paths from day one.
