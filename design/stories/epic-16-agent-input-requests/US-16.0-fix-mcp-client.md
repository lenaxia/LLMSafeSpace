# US-16.0: Fix MCP Client — SSE Parsing + ID Validation

**Epic:** 16 - Agent Input Requests
**Priority:** Critical
**Depends on:** None
**Layer:** Backend

## User Story

As an MCP client user, I need `SendMessage` to correctly detect when the agent finishes (session idle), and I need all API methods to accept real opencode session IDs, so that the MCP integration actually works against a live cluster.

## Problem Statement

Two pre-existing bugs in `pkg/mcp/client.go` prevent the MCP client from working correctly against the real LLMSafeSpace API:

### Bug 1: SSE Event Parsing

`SendMessage` subscribes to `GET /api/v1/workspaces/:id/events` and looks for:
```go
if event.Type == "session.idle" { break }
```

But the real SSE stream emits:
```json
{"type":"session.status","session_id":"ses_abc","status":"idle"}
```

There is no event with `type: "session.idle"`. The function always times out and falls back to `fallbackHistory`. Tests pass only because the mock server emits a fake `{"type":"session.idle"}` event that doesn't exist on the real wire.

### Bug 2: ID Validation Regex

```go
var validID = regexp.MustCompile(`^[a-zA-Z0-9][a-zA-Z0-9.\-]{0,252}$`)
```

This rejects underscores. Real opencode IDs contain underscores:
- Session: `ses_18b28260affeoxXrX1iwPH8wFg`
- Question: `que_e74d7e6db001ZI3VDSHthsee0g`
- Permission: `per_1748012345000_xyz`
- Message: `msg_e74d7da37001Nw4A59Ndzegm3A`

`validateID` is called on `sessionID` in `GetHistory` and `SendMessage`, meaning these methods would fail for real opencode session IDs.

## Acceptance Criteria

- [ ] `SendMessage` correctly detects `{"type":"session.status","session_id":"...","status":"idle"}` and breaks out of the SSE loop
- [ ] `SendMessage` only breaks on idle for the CORRECT session (matches `sessionID` parameter)
- [ ] `validID` regex updated to accept underscores: `^[a-zA-Z0-9][a-zA-Z0-9._\-]{0,252}$`
- [ ] All existing tests updated to use the real wire format (not fake `session.idle` event type)
- [ ] New test: `SendMessage` ignores idle events for OTHER sessions
- [ ] New test: `GetHistory` accepts opencode-format session IDs (`ses_abc123`)
- [ ] Fallback to history still works when SSE connection fails

## Technical Details

### Fix 1: SSE Event Parsing

```go
// Before (broken):
var event struct {
    Type    string `json:"type"`
    Content string `json:"content"`
}
if json.Unmarshal([]byte(data), &event) == nil {
    if event.Type == "session.idle" {
        break
    }
}

// After (correct):
var event struct {
    Type      string `json:"type"`
    SessionID string `json:"session_id"`
    Status    string `json:"status"`
    EventType string `json:"event_type"`
    Data      json.RawMessage `json:"data"`
}
if json.Unmarshal([]byte(data), &event) == nil {
    // Direct session.status event (from broker)
    if event.Type == "session.status" && event.Status == "idle" && event.SessionID == sessionID {
        break
    }
    // Nested opencode.event wrapping session.status
    if event.Type == "opencode.event" && event.EventType == "session.status" {
        // Parse nested data for session idle
        // ...
    }
}
```

### Fix 2: ID Validation

```go
// Before:
var validID = regexp.MustCompile(`^[a-zA-Z0-9][a-zA-Z0-9.\-]{0,252}$`)

// After:
var validID = regexp.MustCompile(`^[a-zA-Z0-9][a-zA-Z0-9._\-]{0,252}$`)
```

### Test Updates

All `SendMessage` tests must emit the real wire format:
```go
// Before (fake):
fmt.Fprintf(w, "data: {\"type\":\"session.idle\"}\n\n")

// After (real):

```
