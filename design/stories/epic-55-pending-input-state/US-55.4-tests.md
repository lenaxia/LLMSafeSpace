# US-55.4: Tests — Precedence-While-Busy, Cross-Workspace, Reconnect Anti-Entropy, E2E

**Epic:** 55 - Pending-Input State as a First-Class Session-Activity Facet
**Priority:** Critical
**Phase:** 4 (tests)
**Depends on:** US-55.1, US-55.2, US-55.3

## User Story

As a maintainer, I need the coverage that would have caught this bug — the "pending *while* busy" case the existing suite missed (`Sidebar.test.tsx:413-463` all use `status: "idle"` + `mockIsSessionBusy=false`) — plus coverage for the new cross-workspace and anti-entropy paths, so the precedence fix and the ownership correction cannot silently regress.

## Context

Per README-LLM Rule 0 (TDD) and Rule 11 (adversarial self-review), this story is mandatory and not optional. The existing test gap (F7) is the reason the precedence bug shipped. The tests here are written first where they precede the implementation (US-55.1–55.3); this story consolidates the integration and e2e coverage that spans all three.

## Acceptance Criteria

- [ ] Every test case below passes.
- [ ] The "pending while busy" case is covered at unit, integration, and e2e levels (this is the regression that catches the bug class).
- [ ] No test depends on an external service without a mock/fake (Rule 0).
- [ ] Tests pass with `-race` (Go) and `-count=1` (frontend), with `-timeout` set.
- [ ] A regression test exists that fails if a sidebar-relevant control event type does not reach `PublishToUser` (the forgotten-publish bug class, Epic D7).
