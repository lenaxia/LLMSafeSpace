# US-24.16: Safe Mode File Download

**Domain:** Recovery UX
**Depends On:** US-24.13
**Priority:** Medium

## Summary

API endpoint to tar and stream a directory from a safe-mode workspace pod. Enables users to bulk-download their files when the workspace is in safe mode.

## API Endpoint

```
GET /workspaces/:id/files/download?path=/workspace
```

**Response**: `200 OK` with `Content-Type: application/gzip`, streaming tar.gz of the requested path.

**Error responses**:
- `403 Forbidden` — caller doesn't own the workspace
- `409 Conflict` — workspace not in safe mode (only available in safe mode)
- `400 Bad Request` — path traversal attempt (`..`), or path outside `/workspace`

## Implementation

```go
func (h *FileHandler) HandleDownload(c *gin.Context) {
    workspaceID := c.Param("id")
    path := c.DefaultQuery("path", "/workspace")

    // Auth: verify ownership (same as terminal handler)
    // Phase check: must be Active + SafeMode condition
    // Path validation: must start with /workspace, no ".."

    // Execute tar in the safe-mode pod
    execReq := clientset.CoreV1().RESTClient().Post().
        Resource("pods").Name(podName).Namespace(ns).
        SubResource("exec").
        VersionedParams(&corev1.PodExecOptions{
            Command: []string{"tar", "-czf", "-", "-C", "/", path},
            Stdout:  true,
        }, parameterScheme)

    // Stream response directly to HTTP client
    c.Header("Content-Type", "application/gzip")
    c.Header("Content-Disposition", fmt.Sprintf("attachment; filename=%s-files.tar.gz", workspaceID))
    // ... SPDY exec stream → HTTP response body
}
```

## Security

- Same ownership check as terminal handler (user must own workspace)
- Path validation: reject `..`, reject paths outside `/workspace`
- Only available when SafeMode condition is True (not for normal workspaces — use terminal instead)
- Pod label check (same as terminal handler F1.3.6): verify pod belongs to workspace

## Acceptance Criteria

1. `GET /workspaces/:id/files/download?path=/workspace` streams tar.gz
2. Only works when workspace is in safe mode (409 otherwise)
3. Ownership verified (403 for non-owner)
4. Path traversal rejected (400 for `..` or paths outside `/workspace`)
5. Streams without buffering entire tar in memory (suitable for large workspaces)
6. Frontend shows "Download files" button when SafeMode condition is True
