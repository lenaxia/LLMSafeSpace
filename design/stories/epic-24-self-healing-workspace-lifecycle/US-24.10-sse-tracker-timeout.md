# US-24.10: SSE Tracker Context Timeout

**Domain:** Health Management
**Depends On:** None
**Priority:** Low (prevents goroutine leak)

## Summary

The SSE session-status tracker in agentd opens a persistent HTTP connection to opencode's `/event` endpoint. If the connection hangs (half-open socket after pod restart), the goroutine blocks indefinitely. Add a 5-minute context deadline; on expiry, cancel and reconnect.

## Change

In `cmd/workspace-agentd/main.go`, `connectAndRead`:

```go
func (t *sessionStatusTracker) connectAndRead(ctx context.Context, client *OpenCodeClient) error {
    // Add 5-minute deadline to the connection context
    connCtx, cancel := context.WithTimeout(ctx, 5*time.Minute)
    defer cancel()

    req, err := http.NewRequestWithContext(connCtx, "GET", agentAddr+"/event", nil)
    // ... rest unchanged
}
```

## Acceptance Criteria

1. SSE connection has 5-minute maximum lifetime
2. On timeout: connection closed, `connectAndRead` returns, `subscribe` loop reconnects with backoff
3. Session status map preserved across reconnects (no "all idle" flash)
4. Backoff still applies on reconnect failure (existing behavior)
