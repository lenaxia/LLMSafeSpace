# US-24.12: CRD Schema Cleanup (Deferred)

**Domain:** All
**Depends On:** US-24.5 wiring complete, behavioral switch stable for 1+ week
**Priority:** Low (cleanup after behavioral switch is validated)
**Status:** Deferred

## Summary

Remove deprecated fields from the CRD once the new recovery system is stable. This is a **breaking CRD change** — existing workspace objects with these fields will lose them on the next controller write.

## What's Already Done (no action needed)

The following were shipped as part of US-24.3 and US-24.5:

- ✅ `CPULimit`, `MemoryLimit` fields added to `ResourceRequirements`
- ✅ `ConsecutiveFailures`, `LastFailureClass`, `LastFailureAt`, `NextRetryAt`, `LastStableAt`, `ControllerRestartCount`, `SafeMode` added to `WorkspaceStatus`
- ✅ Both CRD YAMLs (charts + pkg/crds) updated
- ✅ CRD schema test passes
- ✅ `removeCondition` helper added (US-24.1)

## What Remains (to do AFTER behavioral switch is stable)

### Remove from WorkspaceStatus:
```go
// TransientFailureCount     int32        // replaced by ConsecutiveFailures
// LastTransientFailureAt    *metav1.Time // replaced by LastFailureAt
```

### Remove from WorkspaceSpec:
```go
// MaxRetries int32  // kubebuilder:default=3, max=10
//                   // replaced by per-class policies in recovery_policy.go
```

### Remove from constants.go:
```go
// MaxTransientFailures = 3
// TransientFailureResetWindow = 5 * 60
```

### Deprecate (keep const, never set):
```go
// WorkspacePhaseFailed — const kept for API compatibility
// FailureReasonTransientPodLoss — enum kept, never used
```

### Add conditions (deferred to US-24.13/US-24.17):
```go
// WorkspaceConditionSafeMode     = "SafeMode"      // US-24.13
// WorkspaceConditionDiskPressure  = "DiskPressure"  // US-24.17
```

## Why Deferred

1. The old fields (`TransientFailureCount`, `LastTransientFailureAt`) are still actively written by `recoverFromTransientPodLoss` and `maybeResetTransientCounter`.
2. `MaxRetries` is still read by `recoverFromTransientPodLoss`.
3. These can only be removed AFTER US-24.5 wiring replaces all call sites.
4. Even then, removal should wait until the new behavior is validated in the running cluster for at least a week.
5. `ResolvedImageDigest` (originally in this story) depends on US-24.14 which is deferred.

## Acceptance Criteria (when executed)

1. All `markFailed` call sites already removed (prerequisite)
2. All `recoverFromTransientPodLoss` call sites already removed (prerequisite)
3. Removed fields absent from Go types and CRD YAMLs
4. `make generate` clean
5. `kubectl apply` on new CRD succeeds
6. Existing workspaces readable after update (fields simply disappear from status)
