# US-24.15: Secret Self-Healing

**Domain:** Resilience
**Depends On:** US-24.1
**Priority:** Medium

## Summary

Call `ensurePasswordSecret` in `handleCreating` before pod creation. If the password secret was deleted during recovery backoff (or by any external actor), it's recreated automatically.

## Current State

`ensurePasswordSecret` is called in:
- `handlePending` (L191) — before transitioning to Creating
- `handleResuming` (L446) — before transitioning to Creating

NOT called in:
- `handleCreating` — if secret is deleted while workspace is in Creating phase (e.g., during backoff), the next pod creation fails because the volume mount references a missing secret.

## Change

Add one line to `handleCreating`, before `buildPod`:

```go
func (r *WorkspaceReconciler) handleCreating(ctx context.Context, workspace *v1.Workspace) (ctrl.Result, error) {
    // ... restartGeneration check ...

    // Self-heal: ensure password secret exists (may have been deleted during backoff)
    if err := r.ensurePasswordSecret(ctx, workspace); err != nil {
        logger.Error(err, "Failed to ensure password secret")
        return ctrl.Result{}, err
    }

    // ... rest of handleCreating ...
}
```

`ensurePasswordSecret` is already idempotent (checks if secret exists before creating).

## Acceptance Criteria

1. `handleCreating` calls `ensurePasswordSecret` before any pod creation attempt
2. If secret exists: no-op (idempotent)
3. If secret missing: recreated with fresh random password
4. If secret creation fails (API error): reconcile returns error → requeue (standard pattern)
5. Works during backoff: when NextRetryAt elapses and pod creation is attempted, secret is ensured first
