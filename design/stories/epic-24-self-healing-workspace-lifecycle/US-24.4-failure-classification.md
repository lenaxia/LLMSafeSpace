# US-24.4: Failure Classification

**Domain:** Failure Classification
**Depends On:** US-24.1
**Priority:** High (enables recovery policies)

## Summary

Introduce a `FailureClass` enum and a pure `classifyFailure` function that maps pod observations to one of four classes. This drives per-class recovery policies.

## Types

```go
// recovery.go (or classification.go)

type FailureClass string
const (
    FailureClassNone           FailureClass = ""
    FailureClassInfrastructure FailureClass = "Infrastructure"
    FailureClassResource       FailureClass = "Resource"
    FailureClassProcess        FailureClass = "Process"
    FailureClassConfiguration  FailureClass = "Configuration"
)

// PodObservation captures the relevant signals from a pod for classification.
type PodObservation struct {
    Exists             bool
    Phase              corev1.PodPhase
    Reason             string   // pod.Status.Reason
    Message            string   // pod.Status.Message
    ContainerReason    string   // first terminated/waiting container's reason
    ContainerExitCode  int32
    ContainerOOMKilled bool     // terminated with reason "OOMKilled"
    CrashLoop          bool     // any container in CrashLoopBackOff
    Unschedulable      bool     // PodScheduled condition = False
    Scheduled          bool     // PodScheduled condition = True
}
```

## Classification Rules (priority order)

```go
func classifyFailure(obs PodObservation) FailureClass {
    // 1. Pod not found → Infrastructure (node lost the pod)
    if !obs.Exists {
        return FailureClassInfrastructure
    }

    // 2. OOM always wins → Resource (regardless of other signals)
    if obs.ContainerOOMKilled {
        return FailureClassResource
    }

    // 3. Eviction/preemption/node issues → Infrastructure
    switch obs.Reason {
    case "Evicted", "Preempting", "NodeShutdown", "NodeAffinity",
         "Terminated", "DeadlineExceeded", "GracefulNodeShutdown",
         "DisruptionTarget":
        return FailureClassInfrastructure
    }
    if obs.Unschedulable {
        return FailureClassInfrastructure
    }

    // 4. Ephemeral storage eviction (Evicted + message contains "ephemeral")
    if obs.Reason == "Evicted" && strings.Contains(obs.Message, "ephemeral") {
        return FailureClassResource
    }

    // 5. Image/config errors → Configuration
    switch obs.ContainerReason {
    case "ImagePullBackOff", "ErrImagePull", "InvalidImageName",
         "ErrImageNeverPull", "CreateContainerConfigError",
         "CreateContainerError":
        return FailureClassConfiguration
    }

    // 6. Process crashes → Process
    switch obs.ContainerReason {
    case "CrashLoopBackOff", "RunContainerError", "StartError",
         "ContainerCannotRun", "PostStartHookError", "BackOff":
        return FailureClassProcess
    }
    if obs.CrashLoop {
        return FailureClassProcess
    }

    // 7. Default: unknown reason → Process (conservative, non-terminal)
    return FailureClassProcess
}
```

## Acceptance Criteria

1. `classifyFailure` is a pure function (no side effects, no K8s API calls)
2. Every known K8s reason string has an explicit classification (see TESTPLAN.md)
3. Unknown reasons default to `FailureClassProcess` (F30)
4. `classifyFailure` NEVER returns `FailureClassNone` for a failed pod
5. OOM takes priority over all other signals (even if Evicted + OOM)
6. `observePod` extracts all relevant fields from `corev1.Pod`
7. Metrics: every classification increments `failure_classifications_total{class, pod_reason, container_reason}`
8. Metrics: unknown/default path increments `failure_classification_unknown_total`
9. Structured log emitted on every classification with full observation context
