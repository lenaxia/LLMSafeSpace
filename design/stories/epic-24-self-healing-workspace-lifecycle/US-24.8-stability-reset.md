# US-24.8: Stability Reset

**Domain:** Recovery State Machine
**Depends On:** US-24.5
**Priority:** Medium

## Summary

After 2 minutes of continuous healthy Active phase, reset all recovery state. This proves the last failure was transient and prevents historical failures from accumulating.

## Logic

```go
func (r *WorkspaceReconciler) maybeResetRecoveryState(ws *v1.Workspace) {
    if ws.Status.ConsecutiveFailures == 0 && ws.Status.ControllerRestartCount == 0 {
        return // nothing to reset
    }

    // Start the stability clock on first healthy reconcile
    if ws.Status.LastStableAt == nil {
        now := metav1.Now()
        ws.Status.LastStableAt = &now
        return
    }

    // Check if stable long enough
    if time.Since(ws.Status.LastStableAt.Time) < 2*time.Minute {
        return // not stable long enough yet
    }

    // Reset everything
    ws.Status.ConsecutiveFailures = 0
    ws.Status.ControllerRestartCount = 0
    ws.Status.LastFailureClass = FailureClassNone
    ws.Status.LastFailureAt = nil
    ws.Status.NextRetryAt = nil
    ws.Status.SafeMode = false
    // Remove Degraded condition if set
    r.removeCondition(ws, WorkspaceConditionDegraded)
    r.removeCondition(ws, WorkspaceConditionSafeMode)
}
```

## Suspend Clears Recovery State (F22)

`handleSuspending` must clear (note: currently only clears `TransientFailureCount = 0`; does NOT clear `LastTransientFailureAt`):
- `ConsecutiveFailures = 0`
- `ControllerRestartCount = 0`
- `LastFailureClass = ""`
- `LastFailureAt = nil`
- `NextRetryAt = nil`
- `LastStableAt = nil`
- Remove Degraded and DiskPressure conditions

**NOT cleared during suspend** (per US-24.13 design decision):
- `SafeMode` — preserved so `handleSuspended` can disable TTL for safe-mode workspaces
- `SafeMode` condition — preserved for same reason

**Cleared during resume** (`handleResuming`):
- `SafeMode = false` — resume is an implicit "retry normal mode"
- Remove SafeMode condition

Rationale: Suspend clears the recovery counters (fresh start on resume) but preserves the SafeMode flag so the system knows this workspace was in trouble. Resume is the user's signal that they want to try again.

## Acceptance Criteria

1. After 2 min healthy Active: `ConsecutiveFailures`, `ControllerRestartCount`, `LastFailureClass`, `NextRetryAt` all cleared
2. **F43**: Degraded condition removed on stability reset
3. SafeMode condition removed on stability reset (workspace recovered naturally from safe mode → normal mode → stable)
4. **F28**: `LastStableAt = nil` → starts clock on first healthy reconcile (no panic)
5. **F22**: `handleSuspending` clears ALL recovery state fields
6. Before 2 min: all state preserved (no premature reset)
7. Clock resets if workspace becomes unhealthy during the 2-min window
