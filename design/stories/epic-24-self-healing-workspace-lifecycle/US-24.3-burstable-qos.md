# US-24.3: Burstable QoS

**Domain:** Resource Management
**Depends On:** None (can ship independently as Phase 1)
**Priority:** Critical (immediate stability win)

## Summary

Change workspace pods from QoS=Guaranteed (request=limit) to QoS=Burstable (request < limit) with 4× default burst headroom. This eliminates OOM kills from normal usage spikes.

## CRD Changes

Add two new fields to `ResourceRequirements`:

```go
type ResourceRequirements struct {
    // Existing fields (unchanged):
    // +kubebuilder:validation:Pattern=^([0-9]+m|[0-9]+\.[0-9]+)$
    // +kubebuilder:default="500m"
    CPU              string `json:"cpu,omitempty"`              // request, default "500m"
    // +kubebuilder:validation:Pattern=^[0-9]+(Ki|Mi|Gi)$
    // +kubebuilder:default="512Mi"
    Memory           string `json:"memory,omitempty"`           // request, default "512Mi"
    // +kubebuilder:validation:Pattern=^[0-9]+(Ki|Mi|Gi)$
    // +kubebuilder:default="1Gi"
    EphemeralStorage string `json:"ephemeralStorage,omitempty"` // request=limit, default "1Gi"
    CPUPinning       bool   `json:"cpuPinning,omitempty"`       // existing, unchanged

    // NEW fields:
    // +kubebuilder:validation:Pattern=^([0-9]+m|[0-9]+\.[0-9]+)$
    CPULimit         string `json:"cpuLimit,omitempty"`         // limit, default 4× CPU request
    // +kubebuilder:validation:Pattern=^[0-9]+(Ki|Mi|Gi)$
    MemoryLimit      string `json:"memoryLimit,omitempty"`      // limit, default 4× Memory request
}
```

## Defaulting Logic (F10)

| `Memory` | `MemoryLimit` | Result |
|---|---|---|
| empty | empty | request=512Mi, limit=2Gi |
| "1Gi" | empty | request=1Gi, limit=4Gi |
| empty | "4Gi" | request=512Mi, limit=4Gi |
| "1Gi" | "4Gi" | request=1Gi, limit=4Gi |
| "2Gi" | "1Gi" | **REJECTED** (limit < request) |

Same logic applies to CPU/CPULimit.

## Webhook Changes (F48)

- `MaxCPUMillicores` flag validates against `cpuLimit` (not `cpu` request)
- `MaxMemoryMi` flag validates against `memoryLimit` (not `memory` request)
- New validation: reject if limit < request for either CPU or memory
- Ephemeral storage remains request=limit (no burst — eviction is the consequence)

## Acceptance Criteria

1. Default pod spec: request=500m/512Mi, limit=2000m/2Gi
2. Custom request with empty limit: limit = 4× request
3. Custom request + custom limit: both honored as-is
4. Webhook rejects limit < request
5. Webhook `MaxCPUMillicores` caps `cpuLimit`, `MaxMemoryMi` caps `memoryLimit`
6. Ephemeral storage: request = limit (unchanged, no burst)
7. CRD YAML updated with new fields + kubebuilder validation patterns
