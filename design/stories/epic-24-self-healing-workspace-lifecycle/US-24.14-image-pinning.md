# US-24.14: Image Pinning

**Domain:** Resilience
**Depends On:** US-24.1
**Priority:** Medium

## Summary

Pin the resolved runtime image digest at workspace creation time. Subsequent pod creations use the stored digest, preventing "someone pushed a broken tag" from breaking existing workspaces.

## Design

1. First time `buildPod` is called for a workspace:
   - `resolveRuntimeImage` resolves `spec.runtime` â†’ full image reference (e.g., `ghcr.io/lenaxia/workspace-python:3.11`)
   - Controller resolves the tag to a digest (if not already a digest): `ghcr.io/lenaxia/workspace-python@sha256:abc123...`
   - Stores digest in `status.resolvedImageDigest`
2. Subsequent `buildPod` calls:
   - If `status.resolvedImageDigest` is set, use it directly (skip `resolveRuntimeImage`)
3. `restartGeneration` bump:
   - Clears `status.resolvedImageDigest` â†’ next pod creation re-resolves (picks up new image)

## Digest Resolution

```go
func resolveAndPinImage(ctx context.Context, c client.Reader, runtime string, currentDigest string) (string, error) {
    if currentDigest != "" {
        return currentDigest, nil // already pinned
    }
    image, _, err := resolveRuntimeImage(ctx, c, runtime)
    if err != nil {
        return "", err
    }
    // If image already contains @sha256:, it's pinned
    if strings.Contains(image, "@sha256:") {
        return image, nil
    }
    // For tag-based images, use the tag as-is for now
    // (Full digest resolution requires registry API calls â€” follow-up)
    return image, nil
}
```

**Note**: Full digest resolution (tag â†’ sha256) requires calling the container registry API, which adds complexity and a network dependency to the reconcile loop. For the initial implementation, we store the full image reference (including tag) and rely on the RuntimeEnvironment operator to use digest-based references. A follow-up story can add registry-based resolution.

## Acceptance Criteria

1. First pod creation stores resolved image in `status.resolvedImageDigest`
2. Subsequent pod creations use stored image (don't re-resolve from RuntimeEnvironment)
3. `restartGeneration` bump clears `resolvedImageDigest` (re-resolves on next attempt)
4. RuntimeEnvironment update does NOT affect existing workspaces (only new ones)
5. Explicit image references (`spec.runtime` contains `/`) stored as-is
