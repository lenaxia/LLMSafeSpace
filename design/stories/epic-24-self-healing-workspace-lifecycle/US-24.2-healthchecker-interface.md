# US-24.2: HealthChecker Interface Extraction

**Domain:** Architecture
**Depends On:** US-24.1
**Priority:** Medium

## Summary

Extract a `HealthChecker` interface to decouple health-check HTTP calls from the reconciler. This enables unit testing of health-decision logic without real HTTP servers.

## Design

```go
// health.go

// HealthObservation is the result of a single health check call.
type HealthObservation struct {
    Reachable     bool
    Healthy       bool
    Version       string
    UptimeSeconds int
    Error         error
}

// HealthChecker makes HTTP calls to the workspace agent.
// Production uses httpHealthChecker; tests inject a stub.
type HealthChecker interface {
    CheckHealth(ctx context.Context, podIP string, port int) HealthObservation
}

type httpHealthChecker struct {
    client *http.Client
}

func (h *httpHealthChecker) CheckHealth(ctx context.Context, podIP string, port int) HealthObservation {
    // HTTP GET to /v1/healthz, parse response, return observation
}
```

## Acceptance Criteria

1. `HealthChecker` interface defined with single method
2. `httpHealthChecker` implements it (production)
3. `WorkspaceReconciler` accepts `HealthChecker` field (nil = use default httpHealthChecker)
4. Existing health tests refactored to inject a stub instead of overriding `agentdAdminPort`
5. `classifyFailure` and recovery policy logic remain plain functions (no interface — YAGNI per F38)

## Non-Goals

- No `PodObserver` interface (plain function)
- No `RecoveryEngine` interface (plain function)
- No changes to health-check behavior (just structural)

## Tests

| Test | What It Proves |
|---|---|
| Existing health tests pass with stub HealthChecker | Interface works |
| `TestHealthChecker_Timeout` | httpHealthChecker respects context timeout |
| `TestHealthChecker_ConnectionRefused` | Returns Reachable=false |
| `TestHealthChecker_UnhealthyResponse` | Returns Healthy=false |
