# US-24.6: Remove Failed Phase

**Domain:** Recovery State Machine
**Depends On:** US-24.5, US-24.13
**Priority:** High

## Summary

Remove the `Failed` phase from the workspace state machine entirely. All former Failed paths now enter the recovery loop â†’ safe mode. No workspace is ever permanently inaccessible.

## Changes

1. **Remove `markFailed` helper** â€” all 5 callsites replaced with `enterRecovery`
2. **Remove `handleFailed` case** from the reconcile switch (or keep as no-op for deprecated CRDs)
3. **Keep `WorkspacePhaseFailed` const** as deprecated (API compatibility) but controller never sets it
4. **API changes**:
   - `EnsureSession` (L736): remove `case v1.WorkspacePhaseFailed` from "not usable" switch. Instead: if workspace is in Creating phase with SafeMode=true, return structured safe-mode response (workspace IS accessible via terminal, just not via chat).
   - `waitForWorkspaceActive` (L782): remove `phase == Failed` early-exit. Instead detect SafeMode (phase=Active + condition) and return immediately with safe-mode info.
   - `RestartWorkspace` (L490): already only rejects Terminating/Terminated â€” no change needed. Works for safe-mode workspaces as-is.
5. **Proxy changes**: 
   - Chat/session routes (`/workspaces/:id/sessions/*`, `/workspaces/:id/event`): when workspace has SafeMode condition, return `{"error": "workspace in safe mode", "safeMode": true, "reason": "...", "diagnostic": "/workspaces/:id/diagnostic"}` with 503. Do NOT attempt to route to opencode (doesn't exist in safe mode).
   - Terminal routes (`/workspaces/:id/terminal/*`): work as-is (kubectl exec bypasses proxy, goes directly to pod). No change needed.
   - File download route (US-24.16): new endpoint, only available in safe mode.
6. **`cleanupFailedWorkspaceSecrets`**: remove or repurpose â€” no workspace enters Failed, so this never fires. Secrets are cleaned up in handleTerminating instead.
7. **`EnsureSession` for workspaces in backoff (not yet safe mode)**: when phase=Creating and `NextRetryAt` is set, return structured response immediately (don't spin): `{"status": "recovering", "nextRetryAt": "...", "consecutiveFailures": N, "message": "workspace recovering automatically"}`. This prevents the 30s timeout-spin for workspaces in backoff.

## Acceptance Criteria

1. `grep -r "markFailed\|WorkspacePhaseFailed" controller/` returns only the deprecated const definition
2. No code path in the controller sets `Phase = Failed`
3. API `EnsureSession` returns structured safe-mode response (not error) when workspace is in safe mode
4. API `waitForActive` returns immediately with safe-mode info when SafeMode condition is True
5. Proxy returns `{"safeMode": true, "reason": "...", "diagnostic": "..."}` for safe-mode workspaces
6. Existing tests that assert `Phase == Failed` are rewritten to assert safe mode entry
