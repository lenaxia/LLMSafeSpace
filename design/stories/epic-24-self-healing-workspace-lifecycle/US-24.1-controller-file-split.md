# US-24.1: Controller File Split

**Domain:** Architecture
**Depends On:** None
**Priority:** High (enables all other stories)

## Summary

Extract the 520-line `controller.go` monolith into focused files with single responsibilities. This is prerequisite for testability of the new recovery logic.

## Target File Structure

```
controller/internal/workspace/
├── reconciler.go          # Reconcile() dispatch, SetupWithManager, WorkspaceReconciler struct
├── phase_pending.go       # handlePending
├── phase_creating.go      # handleCreating
├── phase_active.go        # handleActive (minus health check logic)
├── phase_suspend.go       # handleSuspending, handleSuspended, handleResuming
├── phase_terminating.go   # handleTerminating, handleDeletion
├── health.go              # checkAgentHealth observation (HTTP call → result), enrichAgentStatus
├── recovery.go            # recoverFromTransientPodLoss, maybeResetTransientCounter, backoff calc
├── pod_builder.go         # buildPod, buildSafeModePod, buildCredentialSetupInit, buildWorkspaceSetupInit
├── secrets.go             # ensurePasswordSecret, cleanupFailedWorkspaceSecrets, deleteEphemeralSecretsSecret
├── pvc.go                 # buildPVC, isPVCStale, pvcUsesWaitForFirstConsumer, pendingTimedOut
├── network_policy.go      # (already separate — no change)
├── constants.go           # (already separate — no change)
├── helpers.go             # (already separate — no change)
└── types.go               # HostResolver interface, any shared types within the package
```

## Acceptance Criteria

1. All 83 existing controller tests pass without modification (beyond import path changes)
2. No import cycles between files within the package
3. **F46**: `checkAgentHealth` decomposed into:
   - `health.go`: makes HTTP call, parses response, returns a `HealthObservation` struct
   - `phase_active.go`: receives observation, decides whether to restart pod
   - No single file both makes HTTP calls AND modifies workspace status
4. Each file has a clear single responsibility (identifiable from filename)
5. No public API changes — all functions remain package-private (lowercase)
6. `git diff --stat` shows only renames/moves, no logic changes
7. Add `removeCondition` helper (needed by US-24.8, US-24.17) alongside existing `setCondition`

## Non-Goals

- No logic changes in this story (pure refactoring)
- No new tests (existing tests provide coverage)
- No interface extraction (that's US-24.2)

## Implementation Notes

- Use `git mv` where possible to preserve blame history
- The `WorkspaceReconciler` struct stays in `reconciler.go` (all files in the same package can access it)
- Method receivers (`func (r *WorkspaceReconciler) ...`) work across files in the same package
- Run `go vet ./...` and `golangci-lint run` after split to catch any issues
