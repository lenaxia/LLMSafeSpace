# US-24.13: Safe Mode

**Domain:** Recovery
**Depends On:** US-24.5
**Priority:** Critical (guarantees workspace always accessible)

## Summary

When normal recovery is exhausted (N consecutive failures), the controller builds a minimal "safe mode" pod instead of the normal runtime pod. Safe mode guarantees the workspace is always accessible — the user gets a shell and file access regardless of what's wrong with the runtime.

## Safe Mode Image

- **Base**: Alpine Linux (~5MB)
- **Added**: Static Go binary (~10MB) serving:
  - `GET /v1/healthz` → `{"healthy": true}` (always)
  - `GET /v1/diagnostic` → `{"reason": "...", "failures": N, "lastError": "...", "message": "..."}`
  - `/bin/sh` available for terminal exec
- **Total**: ~15MB
- **Pinned by SHA digest** — immutable, can't be broken by tag push
- **Stored as constant** in controller source: `const safeModeImage = "ghcr.io/lenaxia/llmsafespace/safe-mode@sha256:..."`

## Pod Spec

```go
func (r *WorkspaceReconciler) buildSafeModePod(ctx context.Context, ws *v1.Workspace) (*corev1.Pod, error) {
    pod := &corev1.Pod{
        // Same name, namespace, labels, owner reference as normal pod
        Spec: corev1.PodSpec{
            Containers: []corev1.Container{{
                Name:    "safe-mode",
                Image:   safeModeImage,
                Command: []string{"/safe-mode-server"},
                Env: []corev1.EnvVar{
                    {Name: "FAILURE_REASON", Value: string(ws.Status.LastFailureClass)},
                    {Name: "CONSECUTIVE_FAILURES", Value: strconv.Itoa(int(ws.Status.ConsecutiveFailures))},
                    {Name: "LAST_ERROR", Value: ws.Status.Message},
                },
                Ports: []corev1.ContainerPort{
                    {ContainerPort: 4098, Name: "diagnostic"},
                },
                Resources: corev1.ResourceRequirements{
                    Requests: corev1.ResourceList{
                        corev1.ResourceCPU:    resource.MustParse("50m"),
                        corev1.ResourceMemory: resource.MustParse("32Mi"),
                    },
                    Limits: corev1.ResourceList{
                        corev1.ResourceCPU:    resource.MustParse("100m"),
                        corev1.ResourceMemory: resource.MustParse("64Mi"),
                    },
                },
                // Same security context as normal pod
            }},
            // NO init containers
            // NO password secret mount
            Volumes: buildSafeModeVolumes(ws),
            // Same nodeSelector, securityContext, serviceAccount settings
        },
    }
    return pod, nil
}

func buildSafeModeVolumes(ws *v1.Workspace) []corev1.Volume {
    if ws.Status.PVCName == "" {
        return nil // PVC never bound — boot without volume
    }
    // Check if disk pressure triggered safe mode
    readOnly := true
    for _, c := range ws.Status.Conditions {
        if c.Type == WorkspaceConditionDiskPressure && c.Status == "True" {
            readOnly = false // user needs write access to free space
            break
        }
    }
    return []corev1.Volume{{
        Name: "workspace",
        VolumeSource: corev1.VolumeSource{
            PersistentVolumeClaim: &corev1.PersistentVolumeClaimVolumeSource{
                ClaimName: ws.Status.PVCName,
                ReadOnly:  readOnly,
            },
        },
    }}
}
```

## Entry Triggers

1. `ConsecutiveFailures >= policy.SafeModeAfter` (6 for Process/Resource, 3 for Configuration)
2. `ControllerRestartCount > 5` without stability (persistent unreachability, A13)

## Exit Triggers

1. User bumps `spec.restartGeneration` → clears SafeMode, retries normal mode
2. User resumes from Suspended → `handleResuming` checks SafeMode:
   - If SafeMode=true: clear SafeMode, attempt normal mode (resume is an implicit "retry")
   - If SafeMode=false: normal resume flow

**Key design decision**: `SafeMode` is NOT cleared during suspend. Only cleared by:
- `restartGeneration` bump (explicit retry)
- Resume (implicit retry — user is actively engaging)

This allows `handleSuspended` to check `SafeMode` for TTL disablement.

## Controller Behavior for Safe-Mode Workspaces

- `handleActive` early-return when `SafeMode=true`:
  ```go
  if workspace.Status.SafeMode {
      // Only verify pod is still Running. Skip everything else.
      if pod.Status.Phase != corev1.PodRunning {
          // Safe-mode pod lost — recreate it (still in safe mode)
          workspace.Status.Phase = v1.WorkspacePhaseCreating
          workspace.Status.PodIP = ""
          workspace.Status.Endpoint = ""
          return ctrl.Result{}, r.Status().Update(ctx, workspace)
      }
      return ctrl.Result{RequeueAfter: requeueActive}, nil
  }
  ```
- `checkAgentHealth`: **SKIP** (safe mode pod doesn't run agentd)
- `enrichAgentStatus`: **SKIP** (no statusz endpoint)
- Auto-suspend: **SKIP** (no activity tracking in safe mode)
- Timeout: **SKIP** (safe mode has no lifetime limit)
- `handleSuspended` TTL check: **DISABLED** when `SafeMode=true` (user needs time to download files)
- `handleSuspending`: does NOT clear `SafeMode` (only clears recovery counters)
- `handleResuming`: clears `SafeMode` (resume = implicit retry of normal mode)

## Safe-Mode Image Build (prerequisite)

The safe-mode image must be built before US-24.13 can ship:

```
cmd/safe-mode-server/
├── main.go          # ~50 lines: HTTP server on :4098
│                    #   GET /v1/healthz → {"healthy": true}
│                    #   GET /v1/diagnostic → {reason, failures, lastError, message}
│                    #   Reads env vars: FAILURE_REASON, CONSECUTIVE_FAILURES, LAST_ERROR
└── Dockerfile       # FROM alpine:3.20
                     # COPY safe-mode-server /safe-mode-server
                     # ENTRYPOINT ["/safe-mode-server"]
```

- Built as part of the existing CI pipeline (same Makefile targets as workspace-agentd)
- Pushed to same registry as runtime images
- SHA digest captured and hardcoded in controller source
- Image is ~15MB (alpine + static Go binary)

## Acceptance Criteria

1. Safe-mode image built, pushed, SHA-pinned as constant in controller source
2. Safe-mode pod has NO init containers
3. Safe-mode pod PVC mount: read-write if disk pressure, read-only if process crash, omitted if PVC never bound
4. Safe-mode pod minimal resources (50m/32Mi request, 100m/64Mi limit)
5. Safe-mode pod boots → phase=Active + `WorkspaceConditionSafeMode = True`
6. Terminal handler works (phase=Active, pod has /bin/sh)
7. `restartGeneration` bump clears SafeMode and retries normal mode
8. `handleActive` returns early for SafeMode pods: only verifies pod Running, skips health/auto-suspend/timeout/enrichment
9. `handleSuspending` does NOT clear SafeMode (only clears recovery counters)
10. `handleSuspended` disables TTL when SafeMode=true
11. `handleResuming` clears SafeMode (resume = implicit retry)
12. Triggered by ConsecutiveFailures threshold OR ControllerRestartCount threshold
13. Diagnostic endpoint returns failure reason, count, and last error
14. Safe-mode pod lost (evicted) → `handleActive` detects, sets phase=Creating → recreates safe-mode pod (stays in safe mode)
