# US-24.9: Readiness Gate

**Status: SUPERSEDED — Core behavior already implemented**
**Superseded by:** Commit `85ace63` added `allContainersReady()` gate in `handleCreating` — workspace no longer transitions to Active until all containers pass readiness probes. The minor remaining piece (skip `checkAgentHealth` for not-Ready pods) can be folded into US-24.7 if needed.

**Domain:** Health Management
**Depends On:** US-24.1
**Priority:** ~~Medium~~ N/A

## Summary

Don't run `checkAgentHealth` on pods that haven't passed kubelet's readiness probe. Prevents false-positive health failures during slow startup (>30s).

## Change

In `handleActive`, before calling `checkAgentHealth`:

```go
// Skip health checks if pod hasn't passed readiness yet
if !isPodReady(pod) {
    return // don't health-check a pod that isn't ready
}
r.checkAgentHealth(ctx, workspace)
```

```go
func isPodReady(pod *corev1.Pod) bool {
    for _, cs := range pod.Status.ContainerStatuses {
        if cs.Name == "workspace" && cs.Ready {
            return true
        }
    }
    return false
}
```

## Acceptance Criteria

1. Pod Running but not Ready → `checkAgentHealth` not called
2. Pod Running and Ready → `checkAgentHealth` called normally
3. No ContainerStatuses (edge case) → skip health check (conservative)
4. Safe-mode pods → skip health check entirely (separate check in US-24.13)
