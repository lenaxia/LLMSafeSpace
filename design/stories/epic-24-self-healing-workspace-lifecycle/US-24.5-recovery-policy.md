# US-24.5: Recovery Policy with Exponential Backoff

**Domain:** Recovery State Machine
**Depends On:** US-24.4
**Priority:** High (core recovery logic)

## Summary

Replace the current 3-strike `recoverFromTransientPodLoss` with per-class recovery policies that use exponential backoff and trigger safe mode instead of terminal failure.

## Recovery Policy Structure

```go
type RecoveryPolicy struct {
    MaxAttempts   int           // 0 = unlimited (safe mode is the cap)
    BackoffBase   time.Duration
    BackoffMax    time.Duration
    BackoffFactor int           // multiplier per attempt (2 = doubling)
    StabilityReset time.Duration
    SafeModeAfter int           // enter safe mode after this many failures
}

var recoveryPolicies = map[FailureClass]RecoveryPolicy{
    FailureClassInfrastructure: {0, 5*time.Second, 2*time.Minute, 2, 2*time.Minute, 0},
    FailureClassResource:       {0, 10*time.Second, 5*time.Minute, 2, 2*time.Minute, 6},
    FailureClassProcess:        {0, 10*time.Second, 5*time.Minute, 2, 2*time.Minute, 6},
    FailureClassConfiguration:  {0, 30*time.Second, 5*time.Minute, 2, 2*time.Minute, 3},
}
```

## Backoff Calculation

```go
func calculateBackoff(failures int32, policy RecoveryPolicy) time.Duration {
    if failures <= 0 {
        return 0
    }
    shift := int(failures - 1)
    if shift > 30 { // F39: prevent time.Duration overflow
        shift = 30
    }
    backoff := policy.BackoffBase * time.Duration(1<<shift)
    if backoff > policy.BackoffMax {
        backoff = policy.BackoffMax
    }
    return backoff
}
```

## handleCreating Flow (post-Epic-24)

```
handleCreating(ctx, workspace):
  1. Check restartGeneration (F19)
     → if bumped: clear recovery state, clear SafeMode, proceed to step 5
  2. Ensure password secret exists (US-24.15)
  3. Get pod by name
  4. IF pod exists:
     a. Init containers complete → delete ephemeral secrets
     b. Pod terminating (DeletionTimestamp) → requeue, wait
     c. Pod Running + IP → transition to Active ✓
     d. Pod Failed → observePod → classifyFailure → DELETE pod → enterRecovery
     e. Pod Pending + Unschedulable > 5min → classify Infra → enterRecovery (FN3)
     f. Pod Pending + Scheduled + ContainerCreating → requeue (C17: don't timeout pulls)
     g. Otherwise → requeue (pod still starting)
  5. IF pod NOT found:
     a. Check NextRetryAt (F1/F16)
        → if set and not elapsed: return RequeueAfter(remaining)
     b. Check SafeMode flag
        → if true: buildSafeModePod
        → if false: buildPod (normal)
     c. If buildPod fails → classify Configuration → enterRecovery
     d. Create pod → set PodName → requeue
```

## enterRecovery Function

```go
func (r *WorkspaceReconciler) enterRecovery(ctx context.Context, ws *v1.Workspace, class FailureClass) (ctrl.Result, error) {
    ws.Status.ConsecutiveFailures++
    ws.Status.LastFailureClass = class
    now := metav1.Now()
    ws.Status.LastFailureAt = &now

    policy := recoveryPolicies[class]

    // Check safe mode trigger
    if policy.SafeModeAfter > 0 && ws.Status.ConsecutiveFailures >= int32(policy.SafeModeAfter) {
        ws.Status.SafeMode = true
        // condition set, metrics emitted, log emitted
    }

    // Calculate backoff
    backoff := calculateBackoff(ws.Status.ConsecutiveFailures, policy)
    if backoff > 0 {
        nextRetry := metav1.NewTime(now.Add(backoff))
        ws.Status.NextRetryAt = &nextRetry
    }

    ws.Status.Phase = v1.WorkspacePhaseCreating
    ws.Status.PodIP = ""
    ws.Status.Endpoint = ""

    // Metrics + logging
    return ctrl.Result{RequeueAfter: backoff}, r.Status().Update(ctx, ws)
}
```

## Acceptance Criteria

1. **F1/F16**: NextRetryAt checked in "pod not found" branch (not at top)
2. **FN3**: Pod Pending + Unschedulable > 5min → Infrastructure backoff
3. **C17**: Pod Pending + Scheduled + pulling image → NO timeout (just requeue)
4. **F19**: restartGeneration bump clears all recovery state and creates pod immediately
5. **F34**: Counter is class-agnostic (always increments). SafeModeAfter check uses CURRENT class.
6. **F39**: Shift exponent capped at 30 (prevents time.Duration overflow)
7. **F49**: Failed pod DELETED before setting recovery state
8. Infrastructure class: SafeModeAfter=0 → never enters safe mode
9. Process/Resource class: SafeModeAfter=6 → safe mode after 6 failures
10. Configuration class: SafeModeAfter=3 → safe mode after 3 failures
11. `recoverFromTransientPodLoss` removed (replaced by `enterRecovery`)
12. `MaxTransientFailures` constant removed
13. `TransientFailureResetWindow` constant removed (replaced by StabilityReset in policy)
