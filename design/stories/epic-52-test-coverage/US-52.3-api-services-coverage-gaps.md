# US-52.3: API Service-Layer Coverage Gaps

**Epic:** 52 — Comprehensive Test Coverage
**Layer:** Unit
**Effort:** Medium (2–3d)
**Depends on:** US-52.6 (uses the standardised harness)
**Status:** Proposed

---

## Problem

`api/internal/services/` contains 17 subdirectories. Most have a single
test file with narrow scope; several critical services have no test file
at all. The cross-cutting services — the ones whose bugs cascade across
every handler — are the least tested:

| Service | Source files | Test files | Gap |
|---|---|---|---|
| `kubernetes/` | 1 | **0** | No tests at all |
| `eventbroker/` | 2 (`broker.go`, `user_broker.go`) | 1 (`user_broker_test.go` only) | `broker.go` — the multi-shard workspace-event broker — untested |
| `ratelimit/` | 2 | 1 | Distributed path + Redis-failure fallback untested |
| `metering/` | 2 | 2 | Edge cases only |
| `msgqueue/` | 1 | 1 | Concurrent producer/consumer + DLQ path untested |
| `policy/` | 1 | 1 | Policy eval matrix incomplete |
| `sessionindex/` | 1 | 1 | Concurrent updates + eviction untested |

Additionally, `api/internal/middleware/MISSINGTESTS.md` still lists residual
items (token expiry, distributed rate limiting) that should be closed here.

## Scope

**In:**
- Unit tests for the services above, using the `fixture{}` pattern from
  `api/internal/services/workspace/workspace_service_test.go:33`.
- Integration tests use the `//go:build integration` build tag, matching
  the convention in `pkg/secrets/` and `api/internal/handlers/`.
- Closing the two open items in `MISSINGTESTS.md`:
  - Auth middleware: token-expiry tests, multi-method auth (API key vs JWT
    vs session cookie)
  - Rate limiting: distributed-rate-limit path (multi-replica Redis-backed)
- Where a service touches Postgres or Redis, use the harness from US-52.6
  (`api/internal/testharness/`) — do not reinvent setup.

**Out:**
- Handler-layer tests (those live in `api/internal/handlers/` and are
  well-covered).
- Rewriting the services themselves. Bugs found while writing tests are
  fixed + regression-tested in the same PR.
- Database migration tests (covered by `migration-safety` CI workflow).

|---|---|
