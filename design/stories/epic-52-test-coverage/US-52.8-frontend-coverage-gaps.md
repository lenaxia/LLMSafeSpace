# US-52.8: Frontend Coverage Gaps

**Epic:** 52 — Comprehensive Test Coverage
**Layer:** Unit (vitest) + E2E (Playwright)
**Effort:** Medium (2–3d)
**Depends on:** —
**Status:** Proposed

---

## Problem

The frontend has 109 vitest unit tests for 170 components (~64%) and 9
Playwright e2e specs:

```
frontend/tests/e2e/
├── accessibility.spec.ts
├── auth.spec.ts
├── chat.spec.ts
├── context-bar.spec.ts
├── input-requests.spec.ts
├── relay-admin.spec.ts
├── session-activity.spec.ts
├── sidebar-hierarchy.spec.ts
└── streaming.spec.ts
```

**Missing e2e coverage** for entire feature areas:

| Feature area | Unit-tested? | E2E-tested? | Risk |
|---|---|---|---|
| Org admin (SSO, credentials, policies, billing) | partial | **none** | Multi-tenant config regressions ship to prod |
| Settings tabs (admin provider creds, secrets, API keys) | yes | **none** | Settings UI regression blocks every user |
| Account recovery (password reset, key rotation) | partial | **none** | Lockout recovery broken = locked-out users |
| Workspace creation flow (runtime select, storage) | partial | **none** | First-time-user flow broken |
| Model selector + model-set round-trip | yes | partial (in chat) | Model selection broken silently |

Unit-test gaps are smaller but real: error-boundary components, the
`useWorkspace` hook's reconnect logic, the `useEventStream` SSE reconnection
backoff, and the `RelaySetupWizard` multi-step state machine all lack
direct tests.

## Scope

**In:**
- New Playwright e2e specs for the 5 missing feature areas above.
- New vitest unit tests for the named component/hook gaps.
- A shared `frontend/tests/e2e/fixtures/` directory for test data (users,
  workspaces, credentials) so specs don't reinvent setup.
- A shared `frontend/tests/e2e/helpers/` for common actions (login, create
  workspace, wait for Active).

**Out:**
- Visual regression testing (would require Percy/Chromatic — out of scope).
- Mobile viewport testing (the UI is desktop-first; mobile is a future
  epic).
- Accessibility audit beyond the existing `accessibility.spec.ts` (which
  uses axe-core).
- Refactoring components for testability — if a hook isn't testable,
  surface the gap; don't rewrite the hook in this story.
