# US-52.5: cmd Binaries Tests

**Epic:** 52 — Comprehensive Test Coverage
**Layer:** Unit + integration
**Effort:** Medium (2d)
**Depends on:** —
**Status:** Proposed

---

## Problem

`cmd/` holds 8 entry-point binaries. Only 2 have substantial tests:

| Binary | Source | Tests | Risk |
|---|---|---|---|
| `cmd/workspace-agentd/` | 19 | 19 | ✅ Covered |
| `cmd/seal-key/` | 1 | 1 | ✅ Covered |
| `cmd/mcp/` | 1 | **0** | MCP server entry — flag parsing, signal handling untested |
| `cmd/redact/` | 1 | **0** | CLI for the redact pipeline — every release ships this binary |
| `cmd/repolint/` | 1 | **0** | CI gate — bug here breaks every PR |
| `cmd/relay-proxy/` | 3 | 2 | Some coverage; signal handling gap |
| `cmd/relay-router/` | 6 | 2 | Some coverage; reconcile loop untested |
| `cmd/relay-wireguard/` | 0 Go files (Dockerfile only) | 0 | Shell-based sidecar image (US-42.8 / Epic 42 Layer 2) — builds `render-wg.sh` into an Alpine image; no Go source by design. **Not testable via Go tests**; covered by the relay e2e in US-52.2's integration suite |

CLI entry points regress silently: a flag rename breaks every script that
uses the binary, and there is no test to catch it before release.

## Scope

**In:**
- Unit tests for `cmd/mcp`, `cmd/redact`, `cmd/repolint` flag parsing +
  signal handling + exit codes.
- Unhappy-path + signal-handling additions to `cmd/relay-proxy` and
  `cmd/relay-router`.
- A shared `cmd/cmdtest/main_test_helper.go` (or similar) with helpers for
  `exec.Command`-style invocation against the built binary — so every CLI
  test uses the same pattern.
- Decision: `cmd/relay-wireguard/` is a shell-based sidecar image
  (Dockerfile + `render-wg.sh` ConfigMap, US-42.8) with no Go source by
  design — it is **not empty** and must not be deleted. It is tested via
  the relay integration suite (US-52.2) and the relay e2e canaries
  (US-52.10), not via Go unit tests.

**Out:**
- End-to-end testing of the MCP server's tool surface (covered by
  `sdks/canary/mcp/`).
- Testing `cmd/workspace-agentd` internals (already well-covered).
- Performance benchmarks of CLI startup.

## Acceptance Criteria

- [ ] All listed tests merged.
- [ ] `go test -timeout 60s -race ./cmd/...` passes.
- [ ] Coverage of the 4 binaries above (mcp, redact, repolint, relay-proxy,
      relay-router) from ~10% → ≥60%.
- [ ] Shared `cmd/cmdtest/` helper exists and is used by all CLI tests; no
      test reinvents the `exec.Command` pattern.
- [ ] The repolint tests use *fixture repos* (under `testdata/`), not the
      real repo — so tests are deterministic.
- [ ] `cmd/relay-wireguard/` documented in worklog as a shell-based
      sidecar (Dockerfile only, US-42.8); **not deleted** and **not
      tested via Go tests** — covered by relay integration (US-52.2)
      and relay canaries (US-52.10).
- [ ] **Demonstrate-the-failure step (per second-pass audit X1):** at
      least one test in the PR is shown failing when the protection is
      removed (e.g., temporarily remove the redact rule, show
      `TestRedact_NoPIILeak` fails, restore) — documented in worklog.
- [ ] Adversarial review records zero real findings.
- [ ] Worklog entry.
