# US-52.7: Nightly E2E kind Expansion

**Epic:** 52 — Comprehensive Test Coverage
**Layer:** E2E
**Effort:** Large (3–4d)
**Depends on:** US-52.6 (uses harness patterns)
**Status:** Proposed

---

## Problem

The system-level e2e is `local/test.sh`: a 9-step shell script run by
`.github/workflows/e2e-nightly.yml`. It has structural problems:

1. **Opaque failures.** "Test 4/9 failed" tells you nothing. The script
   prints `kubectl describe` on failure but the failure mode is buried in
   the script's own log line.
2. **No isolation.** Steps share state: Test 5 depends on the workspace
   created in Test 4. A failure in Test 4 invalidates Tests 5–9 silently.
3. **No coverage.** Nothing reports what was tested, only that the script
   exited 0 or non-zero. There's no JUnit output, no HTML report, no
   per-scenario pass/fail.
4. **No parallelism.** All 9 steps run serially. A kind cluster can run
   many isolated scenarios in parallel.
5. **No LLM-keyed scenarios skipped silently.** Tests 6 and 8 (the LLM
   round-trips) silently skip when `LLM_API_KEY` is unset, so a regression
   in the prompt path can ship from a CI run that "passed."
6. **Disabled PR-level e2e.** `.github/workflows/e2e-pr.yml`'s "Full-stack
   E2E (docker-compose)" job is `if: false` — disabled. PRs merge without
   any full-stack signal.

The existing Go-level e2e is fragmented: `tests/epic26/` and
`tests/gharouter/` are epic-specific, not general coverage.

## Scope

**In:**
- A new Go-based e2e runner: `tests/runner/` that wraps the kind bootstrap
  + port-forward + scenario execution, replacing the structural role of
  `local/test.sh`.
- Migrated scenarios under `tests/<area>/` as Go tests with build tag
  `//go:build e2e`:
  - `tests/lifecycle/` — workspace create → active → suspend → resume →
    terminate (replaces Test 1–5 of `local/test.sh`)
  - `tests/proxy/` — session create → message → history (replaces Tests
    6–7)
  - `tests/streaming/` — SSE events (replaces Test 8 partially)
  - `tests/resume/` — suspend/resume with session continuity (replaces
    Test 8)
- Each scenario independent: own workspace, own cleanup, own pass/fail.
- CI publishes a JUnit XML + HTML report as an artifact on every nightly
  run.
- Re-enable a *subset* of e2e on PRs (fast scenarios only): the kind-based
  lifecycle scenarios run in `.github/workflows/e2e-pr.yml` against a
  fresh kind cluster per PR. LLM-requiring scenarios stay nightly-only.

**Out:**
- Replacing `local/test.sh` itself — kept as a smoke script for human
  debugging. The Go runner is the CI source of truth; the shell script is
  for `make e2e-local` interactive use.
- Performance benchmarking (still `hack/benchmark-*.sh`).
- Frontend Playwright tests (separate workflow, US-52.8).
- Cross-cloud relay e2e (would require real cloud accounts; out of scope).

## Intent-Declared Test Inventory

### `tests/runner/` infrastructure

| Helper | What | Value added |
|---|---|---|
| `BootstrapKind(t, name) *Cluster` | Create kind cluster, install CRDs, deploy API+controller | Single call replaces ~80 lines of `bootstrap.sh` |
| `(*Cluster) PortForward(t, svc, port) string` | Port-forward to API; returns URL; auto-cleanup | No manual PF_PID management |

| `JunitReport(t, out)` | Convert test results to JUnit XML | CI report |

## CI Integration

### Nightly (`.github/workflows/e2e-nightly.yml`)

- Runs *all* `tests/*` packages with `//go:build e2e`.
- Uploads JUnit XML + HTML report as artifact.
- On failure, posts a comment listing failing scenarios with log links.
- LLM-gated scenarios skip cleanly when `LLM_API_KEY` is unset, **and** the
  nightly summary reports the skip count prominently (no silent pass).

### PR (`.github/workflows/e2e-pr.yml` — re-enable)

- Runs *fast* subset only: `tests/lifecycle/`, `tests/webhooks/`,
  `tests/metrics/`, no LLM-gated.
- Replaces the disabled docker-compose job.
- Target: < 10 minutes wall-clock on a fresh kind cluster.

## Failure-Mode Coverage Matrix

| Failure mode | Covered by |
|---|---|
| Workspace never starts | lifecycle × 1 |
| Data loss on suspend | lifecycle × 1, resume × 1 |
| Slow resume (SLA regression) | lifecycle × 1, resume × 1 |
| Resource leak on terminate | lifecycle × 1 |
| Authz regression (cross-tenant) | proxy × 1 |
| Body-size OOM | proxy × 1 |
| SSE goroutine leak | streaming × 2 |
| Silent metric drift | metrics × 2 |
| Webhook bypass | webhooks × 3 |
| LLM integration broken | proxy × 2, resume × 1 (LLM-gated, skip when no key) |

## Acceptance Criteria

- [ ] `tests/runner/` exists with the helpers above.
- [ ] All listed scenario packages exist and pass on a fresh kind cluster.
- [ ] `local/test.sh` retained for human use; documented as such in
      `local/README.md` (new or updated).
- [ ] `.github/workflows/e2e-nightly.yml` runs the Go runner; publishes
      JUnit XML + HTML artifact.
- [ ] `.github/workflows/e2e-pr.yml` re-enabled with fast subset;
      docker-compose job removed.
- [ ] LLM-gated scenarios skip visibly — the nightly summary prints
      "SKIPPED: 3 (LLM_API_KEY unset)" so skips are not silent.
- [ ] Each scenario uses `tests/runner.NewUser(t)` for isolation; no
      scenario assumes state from another.
- [ ] Wall-clock: nightly < 30 minutes, PR < 10 minutes.
- [ ] Adversarial review records zero real findings.
- [ ] Worklog entry.
