# US-52.1: Controller Phase Reconciler Tests

**Epic:** 52 — Comprehensive Test Coverage
**Layer:** Unit + envtest
**Effort:** Large (4–5d)
**Depends on:** —
**Status:** Proposed

---

## Problem

The workspace controller is the product: it owns the
`Pending → Creating → Active → Suspending → Suspended → Resuming → Active`
lifecycle (per README-LLM.md §Workspace lifecycle). Yet of its 21 source
files, **9 have no direct test**:

| File | What it does | Why untested is dangerous |
|---|---|---|
| `reconciler.go` | Top-level Reconcile loop, dispatches to phase handlers | A bug here makes *every* phase wrong |
| `phase_pending.go` | PVC + pod creation from Pending | First user-visible step; failures look like "workspace never starts" |
| `phase_creating.go` | Pod readiness → Active transition | The Active↔non-Active boundary; misclassification bills users wrongly |
| `phase_active.go` | Health probing, drift detection, status refresh | The steady-state handler; bugs leak as silent metric drift |
| `phase_suspend.go` | Pod delete, PVC retain | A bug here destroys user data |
| `phase_terminating.go` | Finalizer-driven cleanup | A bug here orphans PVCs and Secrets |
| `recovery.go` | Recovery state machine after pod death | Bugs surface only under failure — invisible to happy-path CI |
| `secrets.go` | Workspace Secret materialization | A bug here breaks every credential-using workspace |
| `pvc.go` | PVC create/get/resize | The data persistence layer |
| `runtime_resolver.go` | RuntimeEnvironment → image lookup | A bug here makes every workspace fail to schedule |
| `helpers.go`, `network_policy.go`, `constants.go` | Shared utilities | Network policy regression = cross-tenant leak |

Existing coverage (`controller_test.go`, `controller_restart_test.go`,
`stale_pvc_test.go`, etc.) exercises the *reconciler as a whole* through the
fake clientset, but no test asserts the contract of an individual phase
handler in isolation. When a phase handler is changed, the test that breaks
is the wrong one — an unrelated e2e test, days later.

## Scope

**In:**
- Unit tests (one `<file>_test.go` per source file above) that exercise each
  phase handler against a `controller-runtime/pkg/client/fake` client in
  isolation, with explicit fixture setup of the *input* CRD state and
  assertion on the *output* CRD state + K8s objects created.
- Envtest tests (extending the pattern at
  `pkg/apis/llmsafespaces/v1/envtest_defaults_test.go`) that exercise the
  reconciler against a real apiserver for the cross-cutting flows that fake
  clients misrepresent:
  - Finalizer ordering on delete (fake client does not enforce GC)
  - Status subresource updates (fake client behaves differently)
  - Watch event delivery timing
- Adversarial-test cases per file: at least 2 unhappy paths per phase
  handler, covering the documented failure modes listed below.

**Out:**
- Refactoring the phase handlers. This story adds tests; it does not
  restructure code. If a test reveals a bug, file it under "Findings" and
  fix in the same PR with a regression test (per Rule 5).
- Cross-controller integration (controller + API server together). That is
  US-52.7.
- The `org_suspend.go` flow — it has its own test file already.

## Failure-Mode Coverage Matrix

This is the adversarial checklist (Rule 11 Phase 1) applied to the
controller. Each row is a failure mode; each cell names the test that
protects against it.

| Failure mode | Covered by |
|---|---|
| Stuck in Pending (PVC create race, missing RTE) | phase_pending × 2 |
| Stuck in Creating (premature Active, missing pod) | phase_creating × 2 |
| Zombie Active (pod died, agentd unhealthy, drift) | phase_active × 3, recovery × 2 |
| Data loss on Suspend (PVC deleted with pod) | phase_suspend × 1 |
| Resource leak on Terminate (finalizer removed early) | phase_terminating × 2 |
| Recovery storm (no backoff, wrong classification) | recovery × 2 |
| Credential staleness (secret not updated/deleted) | secrets × 3 |
| Cross-tenant leak (NetworkPolicy missing) | phase_pending × 1 |
| Multi-arch scheduling (amd64 image on arm64 node) | runtime_resolver × 1 |
| Goroutine leak (slow shutdown, context not honoured) | reconciler × 2 |

Every cell is a test that would fail loudly in CI if the protection is
removed. Tests with no cell ("documentation tests") are not added.

## Test Infrastructure Patterns Used

- **Fake client**: `sigs.k8s.io/controller-runtime/pkg/client/fake` with
  `.WithStatusSubresource(&v1.Workspace{})` (matches
  `controller/internal/workspace/controller_test.go:42`).
- **Scheme**: `testScheme(t)` helper registers `v1` + `corev1` (matches
  existing `controller_test.go:28`).
- **Reconciler construction**: `reconcilerFor(t, objs...)` helper (matches
  existing pattern at `controller_test.go:36`).
- **No mocks for K8s client** — the fake client *is* the fake; mocking it
  re-implements a fake badly (per P5).
- **Envtest**: build tag `//go:build envtest`; CI runs in
  `.github/workflows/envtest.yml`'s existing job.

## Acceptance Criteria

- [ ] One test file per source file in the inventory above, merged.
- [ ] Every test in the inventory passes under
      `go test -timeout 60s -race -count=1 ./controller/internal/workspace/...`.
- [ ] Envtest tests pass under
      `go test -tags envtest -timeout 120s ./controller/internal/workspace/...`.
- [ ] Coverage of `controller/internal/workspace/` increases from current
      ~58% to ≥80% (record before/after in worklog).
- [ ] At least one test demonstrates the failure-mode protected (e.g.
      temporarily revert the fix in a throwaway commit, show test fails,
      restore) — documented in worklog.
- [ ] No test uses `time.Sleep` for synchronization (use
      `Eventually`/`Consistently` from `testify` or re-reconcile loops).
- [ ] Adversarial review (Rule 11 Phase 2) on the test PR records zero real
      findings.
- [ ] Worklog entry.
