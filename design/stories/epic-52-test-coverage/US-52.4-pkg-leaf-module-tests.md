# US-52.4: pkg/ Leaf Module Tests

**Epic:** 52 — Comprehensive Test Coverage
**Layer:** Unit
**Effort:** Medium (2d)
**Depends on:** —
**Status:** Proposed

---

## Problem

`pkg/` holds the shared contracts imported across `api/`, `controller/`,
`cmd/`, and `sdks/`. Several leaf modules have zero or near-zero tests:

| Module | Source files | Test files | Gap |
|---|---|---|---|
| `pkg/email/` | 3 (provider, ses, noop) | 2 (8 tests, added by Epic 49 #306) | Already covered — SES + noop provider tests exist; this story verifies coverage is sufficient and adds edge cases only if gaps remain |
| `pkg/types/` | 13 | 2 | The DTO layer — every API call passes through it |
| `pkg/interfaces/` | 2 | **0** | Interface definitions; no tests of helper methods |
| `pkg/config/` | 1 | **0** | Config loading + validation |
| `pkg/kubernetes/` | 3 | 2 | Helper methods untested |

These modules are imported everywhere — a regression in `pkg/types/`
breaks every API consumer; a regression in `pkg/email/ses_provider.go`
silently drops every email.

## Scope

**In:**
- Unit tests for the 5 modules above.
- `pkg/email/` SES provider: unit tests using a stubbed SES client (the
  `aws-sdk-go-v2/service/ses` interface is stub-friendly).
- `pkg/types/`: round-trip JSON tests for every DTO (marshal → unmarshal →
  compare), validation tests for any DTO with `validate:` tags.
- `pkg/interfaces/`: test any non-trivial helper methods; interface
  satisfaction tests via `var _ Interface = (*Impl)(nil)` compile-time
  assertions.
- `pkg/config/`: Viper loading + env-var override + validation tests.
- `pkg/kubernetes/`: helper-method coverage (the K8s client itself is
  integration-tested elsewhere).

**Out:**
- Refactoring types. If a JSON tag is wrong, fix it + add the regression
  test; do not restructure the package.
- Generating OpenAPI/Swagger from types (existing `swaggo/swag` flow).
