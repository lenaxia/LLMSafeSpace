# US-7.1: System Daemon (Sidecar)

**Epic:** 7 — Runtime Interception Layer
**Status:** Planning
**Dependencies:** None

## Objective

Build a root-owned daemon that runs as a sidecar container in Kubernetes. It listens on a Unix socket (shared volume) and executes privileged operations (apt/apk installs) on behalf of the main container's UID 1000 agent. In Docker mode (no sentinel), the daemon binary is unused.

## Design

### Deployment Model

**Kubernetes:** Controller adds the daemon as a sidecar container sharing an emptyDir volume at `/run/llmsafespace/`. The main container's apt wrapper connects to the socket in that shared volume.

**Docker:** Daemon binary exists in the image but is never started. The entrypoint skips it (no sentinel = direct opencode).

### Sidecar Container Spec (generated by controller)

```yaml
- name: system-daemon
  image: same-as-main  # same base image
  command: ["/usr/local/bin/system-daemon"]
  securityContext:
    runAsUser: 0
    allowPrivilegeEscalation: false
    capabilities:
      drop: [ALL]
      add: [CHOWN, DAC_OVERRIDE, FOWNER, SETUID, SETGID]
    seccompProfile:
      type: RuntimeDefault
  volumeMounts:
    - name: daemon-socket
      mountPath: /run/llmsafespace
    - name: policies
      mountPath: /etc/llmsafespace
      readOnly: true
    - name: daemon-log
      mountPath: /var/log/llmsafespace
```

### Daemon Process

Simple event loop:
1. Load policy from `/etc/llmsafespace/daemon/policy.json`
2. Create socket at `/run/llmsafespace/system.sock` (permissions `0666` — socket is in a shared emptyDir, access control is via the volume mount, not file perms)
3. Accept connections, handle requests, stream output
4. On SIGTERM: close socket, exit

No child process management. No forking. No gosu. The daemon is a pure socket server. The main container runs opencode independently.

### Socket Protocol

JSON-over-Unix-socket, newline-delimited.

```go
type Request struct {
    ID      string   `json:"id"`
    Command string   `json:"command"`  // "apt", "apt-get", "apk"
    Args    []string `json:"args"`     // ["install", "-y", "python3"]
}

type StreamLine struct {
    ID     string `json:"id"`
    Stream string `json:"stream,omitempty"` // "stdout" or "stderr"
    Data   string `json:"data,omitempty"`
    Done   bool   `json:"done,omitempty"`
    Exit   int    `json:"exit,omitempty"`
    Error  string `json:"error,omitempty"`
}
```

### Policy Engine

Before executing, validates:
1. **Command allowlist** — only `apt`, `apt-get`, `apk`
2. **Subcommand allowlist** — `install`, `update`, `list`, `search` (not `remove` by default)
3. **Blocked flags** — `--allow-unauthenticated`, `--force-yes`
4. **Blocked packages** — configurable denylist
5. **Rate limiting** — max N installs per minute (default 10)

Policy from `/etc/llmsafespace/daemon/policy.json` (mounted ConfigMap):
```json
{
  "allowedCommands": ["apt", "apt-get", "apk"],
  "allowedSubcommands": ["install", "update", "list", "search", "show"],
  "blockedFlags": ["--allow-unauthenticated", "--force-yes"],
  "blockedPackages": [],
  "rateLimit": {"maxPerMinute": 10}
}
```

### Audit Log

Every request logged to `/var/log/llmsafespace/audit.jsonl`:
```json
{"ts":"2026-05-24T20:00:00Z","id":"abc","command":"apt","args":["install","-y","python3"],"decision":"allow","exitCode":0,"durationMs":3200}
```

### Readiness

The sidecar reports ready once the socket exists. The main container's wrapper retries connection briefly if the socket isn't ready yet (sidecar may start slightly after main container).

```go
// In wrapper (client side):
func connectWithRetry() (net.Conn, error) {
    for i := 0; i < 10; i++ {
        conn, err := net.Dial("unix", "/run/llmsafespace/system.sock")
        if err == nil { return conn, nil }
        time.Sleep(500 * time.Millisecond)
    }
    return nil, fmt.Errorf("daemon not available")
}
```

## Files Created

| File | Purpose |
|------|---------|
| `cmd/system-daemon/main.go` | Socket creation, accept loop, signal handling |
| `cmd/system-daemon/handler.go` | Request parsing, policy check, exec, stream response |
| `cmd/system-daemon/policy.go` | Policy loading and evaluation |
| `cmd/system-daemon/audit.go` | Audit log writer |

## Acceptance Criteria

1. Daemon starts, creates socket at `/run/llmsafespace/system.sock`
2. Accepts connection from another container via shared emptyDir volume
3. `apt install curl` succeeds when requested via socket
4. `apt install python3` succeeds AND wrapper at `/usr/bin/python3` is restored after install
5. Blocked flag (`--allow-unauthenticated`) rejected with clear error
6. Rate limit triggers after N rapid requests
7. Audit log written to `/var/log/llmsafespace/audit.jsonl`
8. SIGTERM → graceful shutdown (finish in-flight, close socket, exit)
9. Binary is <10MB static Go binary
10. Unit tests: policy allow/deny, rate limit, malformed request, concurrent requests, wrapper restoration

## Non-Goals

- No child process management (opencode runs in separate container)
- No gosu/privilege dropping (daemon stays root, that's its job)
- No Docker-mode logic (daemon is simply not started in Docker)
