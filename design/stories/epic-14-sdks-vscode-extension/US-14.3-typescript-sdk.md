# US-14.3: TypeScript SDK

**Epic:** 14 - Multi-Language SDKs & VS Code Extension
**Priority:** Critical
**Depends on:** US-14.1

## User Story

As a TypeScript/JavaScript developer, I want a typed SDK for LLMSafeSpace, so that I can manage workspaces and interact with sandbox agents from Node.js or browser environments with full type safety and autocompletion.

## Acceptance Criteria

- [ ] Package `@llmsafespace/sdk` exists in `sdks/typescript/`
- [ ] All REST endpoints have typed methods with request/response interfaces
- [ ] Authentication: supports both JWT token and API key
- [ ] Token refresh handled automatically (re-login on 401)
- [ ] Errors are typed (`LLMSafeSpaceError` with `status`, `message`, `code`)
- [ ] Works in Node.js 18+ and modern browsers (fetch-based, no node-specific deps)
- [ ] Zero runtime dependencies beyond `fetch` (uses native fetch)
- [ ] Generated types match OpenAPI spec exactly
- [ ] README with usage examples for common workflows
- [ ] `npm run build` produces ESM + CJS bundles
- [ ] `npm run test` passes (generated client + hand-written integration tests)

## Technical Details

**Generation approach:**

Use `openapi-typescript` to generate types, then `openapi-fetch` for the runtime client. This produces the most idiomatic TypeScript output with zero bloat.

```bash
# In sdks/Makefile
generate-ts:
    npx openapi-typescript sdks/openapi.yaml -o sdks/typescript/src/generated/api.d.ts
    # Hand-written client.ts wraps openapi-fetch with auth + error handling
```

**Public API surface:**

```typescript
import { LLMSafeSpace } from '@llmsafespace/sdk';

// Initialize with API key
const client = new LLMSafeSpace({
  baseUrl: 'https://llmsafespace.example.com',
  apiKey: 'lsp_...',
  timeout: 120_000, // ms — default for sendMessage (LLM responses can be slow)
});

// Or with credentials (auto-manages JWT)
const client = new LLMSafeSpace({
  baseUrl: 'https://llmsafespace.example.com',
  credentials: { email: 'user@example.com', password: '...' },
});

// Workspace operations
const workspace = await client.workspaces.create({ name: 'my-project', runtime: 'python:3.11', storageSize: '10Gi' });
const list = await client.workspaces.list();
const status = await client.workspaces.getStatus(workspace.id);
await client.workspaces.suspend(workspace.id);
await client.workspaces.resume(workspace.id);
await client.workspaces.activate(workspace.id); // resume + ensure session

// Session operations (ensureSession creates session + activates workspace if needed)
const session = await client.sessions.ensure(workspace.id);
// sendMessage is synchronous — blocks until LLM responds. Configurable timeout.
const response = await client.sessions.sendMessage(workspace.id, session.sessionId, 'Write a hello world in Python');
const history = await client.sessions.getHistory(workspace.id, session.sessionId);

// Secret + binding management (workspace-level credential injection)
const secret = await client.secrets.create({ name: 'anthropic-key', value: 'sk-...' });
await client.workspaces.setBindings(workspace.id, [{ secretId: secret.id, envVar: 'ANTHROPIC_API_KEY' }]);
// Or set env vars directly
await client.workspaces.setEnv(workspace.id, { ANTHROPIC_API_KEY: 'sk-...' });

// Terminal (used by VS Code extension — US-14.10)
const ticket = await client.terminal.getTicket(workspace.id);
// ticket.ticket is a one-time token for WebSocket upgrade (30s expiry)

// Auth
const me = await client.auth.me();
const keys = await client.auth.listApiKeys();
```

**Important: `sendMessage` response shape:**

The `POST /workspaces/:id/sessions/:sessionId/message` endpoint is a transparent proxy to opencode's `/session/:id/message`. The response is opencode's native format — NOT a LLMSafeSpace-defined type. The SDK must:

1. Define a `MessageResponse` type that matches opencode's response shape
2. Document that this type tracks opencode's API (version-coupled)
3. Extract the text content from opencode's `parts` array for convenience

```typescript
// opencode response shape (simplified)
interface OpenCodeMessageResponse {
  id: string;
  role: 'assistant';
  parts: Array<{ type: 'text'; text: string } | { type: 'tool-invocation'; /* ... */ }>;
  metadata?: Record<string, unknown>;
}

// SDK convenience wrapper
interface MessageResponse {
  raw: OpenCodeMessageResponse;  // Full opencode response
  content: string;               // Extracted text (concatenated text parts)
}
```

The SDK's `sendMessage` returns `MessageResponse` where `content` is a convenience field that concatenates all `text` parts. Callers needing tool invocation details or metadata use `raw`.

**Important: `sendMessage` timeout behavior:**

The `sendMessage` method proxies to opencode's synchronous message endpoint. The LLM may take 30-120+ seconds to respond. The SDK:
1. Uses a configurable timeout (default 120s, set via `timeout` option)
2. On timeout, throws `TimeoutError` with message explaining the prompt may still be processing
3. Caller can then poll `getHistory` to check if the response arrived

This is documented prominently in the SDK README.

**Error handling:**

```typescript
import { LLMSafeSpaceError, NotFoundError, AuthError } from '@llmsafespace/sdk';

try {
  await client.workspaces.get('nonexistent');
} catch (e) {
  if (e instanceof NotFoundError) { /* 404 */ }
  if (e instanceof AuthError) { /* 401/403 */ }
  if (e instanceof LLMSafeSpaceError) { /* any API error */ }
}
```
