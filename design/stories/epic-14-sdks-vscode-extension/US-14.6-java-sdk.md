# US-14.6: Java SDK

**Epic:** 14 - Multi-Language SDKs & VS Code Extension
**Priority:** Normal
**Depends on:** US-14.1

## User Story

As a Java/Kotlin developer building enterprise applications, I want a typed Java SDK for LLMSafeSpace, so that I can integrate sandbox management into JVM-based services with proper exception handling and builder patterns.

## Acceptance Criteria

- [ ] Maven artifact `com.llmsafespace:sdk` exists in `sdks/java/`
- [ ] All REST endpoints have typed methods
- [ ] Authentication: API key and credential-based (auto JWT refresh)
- [ ] Typed exceptions (`LLMSafeSpaceException` hierarchy)
- [ ] Java 17+ (records for DTOs, sealed interfaces for errors)
- [ ] Uses `java.net.http.HttpClient` (no third-party HTTP dependency)
- [ ] Generated models match OpenAPI spec
- [ ] `mvn test` passes
- [ ] README with usage examples
- [ ] Kotlin-friendly API (no `void` returns where `Unit` is expected, nullable annotations)

## Assumptions

| # | Assumption | Validation |
|---|-----------|-----------|
| A1 | `openapi-generator` Java template supports OpenAPI 3.1 | ⚠️ openapi-generator 7.x supports 3.1. Must use v7+. |
| A2 | Java 17+ is acceptable minimum (records, sealed classes) | ✅ Java 17 is LTS and widely adopted in enterprise |
| A3 | `java.net.http.HttpClient` is sufficient (no need for OkHttp) | ✅ Built-in since Java 11, supports async and HTTP/2 |

## Technical Details

**Generation approach:**

Use `openapi-generator` with the `java` generator and `native` HTTP library.

```bash
# In sdks/Makefile
generate-java:
    openapi-generator generate \
        -i sdks/openapi.yaml \
        -g java \
        -o sdks/java \
        --library native \
        --additional-properties=java8=false,dateLibrary=java8,useJakartaEe=true,apiPackage=com.llmsafespace.sdk.api,modelPackage=com.llmsafespace.sdk.model
```

**Package structure:**

```
sdks/java/
├── pom.xml
├── src/
│   └── main/java/com/llmsafespace/sdk/
│       ├── LLMSafeSpace.java            # Client facade (hand-written)
│       ├── LLMSafeSpaceConfig.java      # Builder-pattern config
│       ├── auth/
│       │   └── AuthManager.java         # Token management
│       ├── exceptions/
│       │   ├── LLMSafeSpaceException.java
│       │   ├── NotFoundException.java
│       │   ├── AuthException.java
│       │   └── RateLimitException.java
│       ├── api/                          # Generated API classes
│       └── model/                        # Generated model classes
├── src/test/java/com/llmsafespace/sdk/
│   ├── ClientTest.java
│   └── IntegrationTest.java
└── README.md
```

**Public API surface:**

```java
import com.llmsafespace.sdk.LLMSafeSpace;
import com.llmsafespace.sdk.model.*;

// Initialize with API key
var client = LLMSafeSpace.builder()
    .baseUrl("https://llmsafespace.example.com")
    .apiKey("lsp_...")
    .build();

// Workspace operations
var workspace = client.workspaces().create(CreateWorkspaceRequest.builder()
    .name("my-project")
    .runtime("python:3.11")
    .storageSize("10Gi")
    .build());

var list = client.workspaces().list();
var status = client.workspaces().getStatus(workspace.id());
client.workspaces().suspend(workspace.id());
client.workspaces().resume(workspace.id());
client.workspaces().activate(workspace.id());

// Session operations (ensure activates workspace + creates session if needed)
var session = client.sessions().ensure(workspace.id());
var response = client.sessions().sendMessage(workspace.id(), session.sessionId(), "Write hello world");
var history = client.sessions().getHistory(workspace.id(), session.sessionId());
```

**Error handling:**

```java
try {
    client.workspaces().get("nonexistent");
} catch (NotFoundException e) {
    System.out.println("Not found: " + e.getMessage());
} catch (RateLimitException e) {
    System.out.println("Retry after: " + e.getRetryAfter());
} catch (LLMSafeSpaceException e) {
    System.out.println("API error " + e.getStatus() + ": " + e.getMessage());
}
```

**Dependencies:**

- `java.net.http.HttpClient` (JDK built-in)
- `com.fasterxml.jackson` (JSON serialization — standard in Java ecosystem)
- No other runtime deps
