# US-14.8: VS Code Extension Core (Sidebar + Commands)

**Epic:** 14 - Multi-Language SDKs & VS Code Extension
**Priority:** Critical
**Depends on:** US-14.3 (TypeScript SDK)

## User Story

As a developer using VS Code, I want a sidebar panel showing my LLMSafeSpace workspaces with their status, so that I can manage sandbox lifecycles without leaving my editor.

## Acceptance Criteria

- [ ] Extension activates and registers in VS Code's extension host
- [ ] Sidebar "LLMSafeSpace" view container with workspace tree
- [ ] Tree shows: workspace name, runtime badge, status icon (🟢 active, 🟡 suspended, ⚪ terminated)
- [ ] Tree auto-refreshes every 30s and on workspace state changes
- [ ] **Offline/error state:** When API is unreachable, sidebar shows last-known state with "⚠️ Disconnected" indicator and "Retry" button. Stale data is visually distinct (dimmed).
- [ ] Commands registered: Create Workspace, Suspend, Resume, Activate, Terminate, Refresh
- [ ] "Create Workspace" command shows QuickPick for runtime selection, then input for name
- [ ] Right-click context menu on workspace items: Suspend, Resume, Activate, Terminate, Copy ID
- [ ] Status bar item shows active workspace count (or "Disconnected" if API unreachable)
- [ ] Settings: `llmsafespace.apiUrl`, `llmsafespace.apiKey` (stored in VS Code secrets)
- [ ] First-run experience: prompt for API URL and API key if not configured
- [ ] Extension works in VS Code 1.95+ (minimum version — Chat Participant API requirement)

## Assumptions

| # | Assumption | Validation |
|---|-----------|-----------|
| A1 | VS Code Chat Participant API is stable | ⚠️ Introduced in VS Code 1.93 as proposed API, stabilized in ~1.95. Set `engines.vscode` to `^1.95.0` for safety. |
| A2 | `@llmsafespace/sdk` can be bundled into extension without size issues | ⚠️ SDK has zero runtime deps (uses native fetch). Bundle size should be <50KB. Verify after build. |
| A3 | VS Code SecretStorage is available in all extension contexts | ✅ Available since VS Code 1.53 |
| A4 | TreeDataProvider refresh doesn't cause UI flicker | ✅ Standard pattern — VS Code handles incremental updates |
| A5 | Extension can make HTTP requests to arbitrary URLs (not blocked by VS Code) | ✅ Extensions run in Node.js host with full network access |

**TreeDataProvider implementation:**

```typescript
// src/providers/workspace-tree.ts
export class WorkspaceTreeProvider implements vscode.TreeDataProvider<WorkspaceItem> {
  private _onDidChangeTreeData = new vscode.EventEmitter<WorkspaceItem | undefined>();
  readonly onDidChangeTreeData = this._onDidChangeTreeData.event;

  constructor(private api: ApiService) {}

  async getChildren(): Promise<WorkspaceItem[]> {
    const workspaces = await this.api.client.workspaces.list();
    return workspaces.items.map(ws => new WorkspaceItem(ws));
  }

  getTreeItem(element: WorkspaceItem): vscode.TreeItem {
    return element; // WorkspaceItem extends TreeItem
  }

  refresh(): void {
    this._onDidChangeTreeData.fire(undefined);
  }
}
```

**Authentication flow:**

1. On activation, check `llmsafespace.apiUrl` setting
2. If not set, show "Configure LLMSafeSpace" welcome view
3. API key stored in `context.secrets` (VS Code's encrypted secret storage)
4. SDK client initialized with stored credentials
5. On 401, prompt user to re-enter API key

**Auto-refresh:**

- Timer-based refresh every N seconds (configurable, default 30)
- Event-driven refresh after any command execution (create/suspend/activate)
- Debounced to prevent rapid-fire API calls
