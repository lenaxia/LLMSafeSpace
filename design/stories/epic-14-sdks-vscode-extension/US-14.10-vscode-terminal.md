# US-14.10: VS Code Terminal Integration

**Epic:** 14 - Multi-Language SDKs & VS Code Extension
**Priority:** High
**Depends on:** US-14.2 (Terminal proxy endpoint), US-14.8 (Extension core)

## User Story

As a developer using VS Code, I want to open a terminal connected to my sandbox's shell, so that I can run commands, inspect files, and debug directly inside the sandbox environment from my IDE.

## Acceptance Criteria

- [ ] "Open Terminal" command available in workspace context menu and command palette
- [ ] Terminal opens as a VS Code integrated terminal (appears in terminal panel)
- [ ] Terminal connects via WebSocket to the API's terminal proxy endpoint
- [ ] Input/output works correctly (typing, command execution, output display)
- [ ] Terminal supports ANSI colors and cursor movement (full PTY)
- [ ] Terminal resize events sent when panel is resized
- [ ] Terminal auto-reconnects on transient WebSocket disconnection (with backoff)
- [ ] Terminal shows error message if workspace is not active (offers to activate)
- [ ] Multiple terminals can be opened to the same workspace
- [ ] Terminal title shows workspace name
- [ ] Terminal closes gracefully when workspace is suspended/terminated

## Assumptions

| # | Assumption | Validation |
|---|-----------|-----------|
| A1 | VS Code Pseudoterminal API is stable | ✅ Available since VS Code 1.37, widely used |
| A2 | `ws` package works in VS Code extension host (Node.js) | ✅ Standard Node.js WebSocket client |

## Technical Details

**Implementation approach:**

VS Code's `Pseudoterminal` API allows creating custom terminal backends. We implement a `Pseudoterminal` that bridges to the WebSocket terminal proxy (US-14.2).

The connection flow uses the ticket-based auth from US-14.2:
1. Call `POST /workspaces/:id/terminal/ticket` with API key → get one-time ticket
2. Connect WebSocket to `GET /workspaces/:id/terminal?ticket=<ticket>`
3. Ticket is consumed on connect (single-use, 30s expiry)

```typescript
// src/providers/terminal-provider.ts
export class SandboxTerminal implements vscode.Pseudoterminal {
  private writeEmitter = new vscode.EventEmitter<string>();
  private closeEmitter = new vscode.EventEmitter<number | void>();
  onDidWrite = this.writeEmitter.event;
  onDidClose = this.closeEmitter.event;

  private ws: WebSocket | undefined;
  private dimensions: { cols: number; rows: number } | undefined;
  private intentionallyClosed = false;

  constructor(
    private workspaceId: string,
    private api: ApiService,
  ) {}

  open(initialDimensions: vscode.TerminalDimensions | undefined): void {
    this.dimensions = initialDimensions;
    this.connect();
  }

  close(): void {
    this.intentionallyClosed = true;
    this.ws?.close();
  }

  handleInput(data: string): void {
    this.ws?.send(JSON.stringify({ type: 'input', data }));
  }

  setDimensions(dimensions: vscode.TerminalDimensions): void {
    this.dimensions = dimensions;
    this.ws?.send(JSON.stringify({ type: 'resize', cols: dimensions.columns, rows: dimensions.rows }));
  }

  private async connect(): Promise<void> {
    try {
      // Step 1: Get one-time ticket via authenticated REST call
      const ticket = await this.api.client.terminal.getTicket(this.workspaceId);

      // Step 2: Connect WebSocket with ticket (no auth header needed)
      const apiUrl = this.api.baseUrl.replace('http', 'ws');
      const url = `${apiUrl}/api/v1/workspaces/${this.workspaceId}/terminal?ticket=${ticket.ticket}`;
      this.ws = new WebSocket(url);

      this.ws.onopen = () => {
        this.reconnectAttempts = 0; // Reset on successful connect
        if (this.dimensions) {
          this.ws!.send(JSON.stringify({ type: 'resize', cols: this.dimensions.cols, rows: this.dimensions.rows }));
        }
      };

      this.ws.onmessage = (event) => {
        const msg = JSON.parse(event.data.toString());
        switch (msg.type) {
          case 'output':
            this.writeEmitter.fire(msg.data);
            break;
          case 'exit':
            this.intentionallyClosed = true; // Server-side exit — don't reconnect
            this.closeEmitter.fire(msg.code);
            break;
          case 'error':
            this.writeEmitter.fire(`\r\n\x1b[31mError: ${msg.message}\x1b[0m\r\n`);
            this.intentionallyClosed = true;
            this.closeEmitter.fire(1);
            break;
        }
      };

      this.ws.onclose = () => {
        if (!this.intentionallyClosed) {
          this.reconnectWithBackoff();
        }
      };

      this.ws.onerror = () => {
        this.writeEmitter.fire('\r\n\x1b[31mConnection error\x1b[0m\r\n');
      };
    } catch (e) {
      this.writeEmitter.fire(`\r\n\x1b[31mFailed to connect: ${e.message}\x1b[0m\r\n`);
      this.closeEmitter.fire(1);
    }
  }

  private reconnectAttempts = 0;
  private reconnectWithBackoff(): void {
    this.reconnectAttempts++;
    if (this.reconnectAttempts > 3) {
      this.writeEmitter.fire('\r\n\x1b[31mConnection lost. Use "Open Terminal" to reconnect.\x1b[0m\r\n');
      this.closeEmitter.fire(1);
      return;
    }
    const delay = Math.min(1000 * Math.pow(2, this.reconnectAttempts - 1), 30000);
    if (this.reconnectAttempts >= 3) {
      this.writeEmitter.fire('\r\n\x1b[33mReconnecting...\x1b[0m\r\n');
    }
    setTimeout(() => this.connect(), delay);
  }
}
```

**Command registration:**

```typescript
// src/commands/open-terminal.ts
export function registerOpenTerminalCommand(context: vscode.ExtensionContext, api: ApiService) {
  context.subscriptions.push(
    vscode.commands.registerCommand('llmsafespace.openTerminal', async (item?: WorkspaceItem) => {
      const workspaceId = item?.workspace.id ?? await selectWorkspace(api);
      if (!workspaceId) return;

      // Check workspace is active
      const status = await api.client.workspaces.getStatus(workspaceId);
      if (status.phase !== 'Active') {
        const activate = await vscode.window.showWarningMessage(
          `Workspace is ${status.phase}. Activate it?`,
          'Activate', 'Cancel'
        );
        if (activate !== 'Activate') return;
        await api.client.workspaces.activate(workspaceId);
        // Wait for active (poll status)
        await waitForPhase(api, workspaceId, 'Active', 30_000);
      }

      // Terminal uses ticket-based auth (no raw token on disk or in URL permanently)
      const pty = new SandboxTerminal(workspaceId, api);
      const terminal = vscode.window.createTerminal({
        name: `LLMSafeSpace: ${item?.workspace.name ?? workspaceId}`,
        pty,
        iconPath: new vscode.ThemeIcon('terminal'),
      });
      terminal.show();
    })
  );
}
```

**Reconnection strategy:**

| Attempt | Delay | Action |
|---------|-------|--------|
| 1 | 1s | Silent reconnect |
| 2 | 2s | Silent reconnect |
| 3 | 4s | Show "Reconnecting..." in terminal |
| 4+ | Give up | Show error, offer "Reconnect" button |

On workspace suspend/terminate events (detected via polling or future SSE), the terminal shows a message and closes cleanly rather than attempting reconnection.

**WebSocket library:**

Use `ws` package (Node.js WebSocket client) since the extension runs in Node.js extension host, not the browser.

**Dependencies:**

- `ws` (WebSocket client for Node.js)
- `@llmsafespace/sdk` (for workspace status checks)
