# US-14.9: VS Code Chat Participant (@llmsafespace)

**Epic:** 14 - Multi-Language SDKs & VS Code Extension
**Priority:** High
**Depends on:** US-14.8

## User Story

As a developer using VS Code with GitHub Copilot, I want to type `@llmsafespace` in Copilot Chat to interact with my sandbox agent, so that I can ask the agent to write code, run tests, or debug issues directly from the chat panel.

## Acceptance Criteria

- [ ] `@llmsafespace` chat participant registered and visible in Copilot Chat
- [ ] Typing `@llmsafespace write a hello world in Python` sends the prompt to the active sandbox agent
- [ ] Response renders in the chat panel as markdown (synchronous — waits for complete response, shows progress indicator while waiting)
- [ ] If no active workspace, prompts user to select or create one
- [ ] Slash commands: `/new-session`, `/switch-workspace`, `/history`, `/status`
- [ ] Chat participant shows workspace name in response header
- [ ] Handles agent errors gracefully (sandbox not running, timeout, rate limit)
- [ ] Conversation context maintained within a VS Code chat session
- [ ] MCP server auto-configured for Copilot's native MCP support (mcp.json)
- [ ] Works alongside the sidebar (selecting a workspace in sidebar sets the chat context)

## Technical Details

**Two integration paths (both implemented):**

### Path 1: Chat Participant API (primary UX)

The Chat Participant API gives us a custom `@llmsafespace` mention in Copilot Chat with full control over request/response handling.

**Timeout handling for long prompts:**

The chat participant must handle the fact that `sendMessage` can take 30-120+ seconds:
1. Show progress indicator immediately: `response.progress('Thinking...')`
2. Use configurable timeout (default 120s)
3. On timeout: show message "The agent is still processing. Check back with `/history`."
4. On success: render response as markdown

```typescript
// src/providers/chat-participant.ts
export function registerChatParticipant(context: vscode.ExtensionContext, api: ApiService) {
  const participant = vscode.chat.createChatParticipant('llmsafespace.agent', async (request, chatContext, response, token) => {
    const workspaceId = getActiveWorkspaceId();
    if (!workspaceId) {
      response.markdown('No active workspace. Use `/switch-workspace` to select one.');
      return;
    }

    // Get or create session
    const sessionId = await getOrCreateSession(workspaceId);

    // Show progress while waiting for LLM
    response.progress('Agent is thinking...');

    try {
      const result = await api.client.sessions.sendMessage(workspaceId, sessionId, request.prompt);
      response.markdown(result.content);
    } catch (e) {
      if (e instanceof TimeoutError) {
        response.markdown('⏳ The agent is still processing your request. Use `/history` to check for the response.');
      } else if (e instanceof NotFoundError) {
        response.markdown('⚠️ Workspace not found. It may have been terminated.');
      } else if (e instanceof RateLimitError) {
        response.markdown('⚠️ Rate limited. Please wait a moment and try again.');
      } else {
        response.markdown(`❌ Error: ${e.message}`);
      }
    }
  });

  participant.iconPath = vscode.Uri.joinPath(context.extensionUri, 'resources/icons/llmsafespace.svg');
}
```

**Slash commands:**

| Command | Action |
|---------|--------|
| `/new-session` | Creates a new opencode session in the active workspace |
| `/switch-workspace` | Shows QuickPick to select a different workspace |
| `/history` | Shows recent session history in chat |
| `/status` | Shows workspace phase, agent health, disk usage |

### Path 2: MCP Configuration (zero-code Copilot integration)

For users who prefer Copilot's native MCP tool-calling (VS Code 1.99+), the extension auto-generates `.vscode/mcp.json`:

```typescript
// src/mcp/config-provider.ts
export async function ensureMcpConfig(context: vscode.ExtensionContext) {
  const workspaceFolder = vscode.workspace.workspaceFolders?.[0];
  if (!workspaceFolder) return;

  const mcpConfigPath = vscode.Uri.joinPath(workspaceFolder.uri, '.vscode', 'mcp.json');
  const apiUrl = vscode.workspace.getConfiguration('llmsafespace').get<string>('apiUrl');

  // SECURITY: Never write raw API key to disk. Use env var interpolation.
  // User must set LLMSAFESPACE_API_KEY in their environment.
  const config = {
    servers: {
      llmsafespace: {
        type: "sse",
        url: `${apiUrl}/mcp/sse`,
        headers: {
          "Authorization": "Bearer ${env:LLMSAFESPACE_API_KEY}"
        }
      }
    }
  };

  await vscode.workspace.fs.writeFile(mcpConfigPath, Buffer.from(JSON.stringify(config, null, 2)));

  // Remind user to set the env var if not already set
  if (!process.env.LLMSAFESPACE_API_KEY) {
    vscode.window.showInformationMessage(
      'Set LLMSAFESPACE_API_KEY environment variable for MCP integration.',
      'Copy Command'
    ).then(choice => {
      if (choice === 'Copy Command') {
        vscode.env.clipboard.writeText('export LLMSAFESPACE_API_KEY="lsp_your_key_here"');
      }
    });
  }
}
```

**Security note:** The generated `mcp.json` uses `${env:LLMSAFESPACE_API_KEY}` interpolation — the raw key is NEVER written to disk. This file is safe to commit to git (though adding `.vscode/mcp.json` to `.gitignore` is still recommended as a defense-in-depth measure).

This gives Copilot access to LLMSafeSpace MCP tools (`workspace_create`, `workspace_activate`, `session_create`, `session_message`, `session_history`, `workspace_stop`) without the chat participant — Copilot calls them as tools during normal conversation.

**Session management:**

- One opencode session per VS Code chat thread
- Session ID stored in chat participant's `context.history`
- New `/new-session` creates a fresh session (new conversation with the agent)
- Sessions persist across VS Code restarts (stored in extension globalState)
