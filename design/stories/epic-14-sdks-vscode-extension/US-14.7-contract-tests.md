# US-14.7: SDK Contract Test Suite

**Epic:** 14 - Multi-Language SDKs & VS Code Extension
**Priority:** High
**Depends on:** US-14.3 (at least one SDK must exist to validate against)

## User Story

As a maintainer of 4 SDKs, I need a language-agnostic contract test suite, so that I can verify all SDKs produce correct HTTP requests and handle responses identically without writing duplicate tests in each language.

## Assumptions

| # | Assumption | Validation |
|---|-----------|-----------|
| A1 | Prism mock server correctly validates against OpenAPI 3.1 | ⚠️ Prism 5.x supports 3.1. Must use v5+. If issues arise, fallback to 3.0.3 spec variant for mock server only. |
| A2 | Hurl can express all needed HTTP interactions | ✅ Hurl supports headers, JSON bodies, status assertions, and response body validation |
| A3 | All SDKs can be pointed at a local mock server URL | ✅ All SDKs accept `baseUrl` configuration |
| A4 | Contract tests can run in CI without a real K8s cluster | ✅ Prism mock server is standalone — no backend needed |

## Acceptance Criteria

- [ ] Contract tests defined in `sdks/tests/contract/` using Hurl files
- [ ] Tests cover: auth flow, workspace CRUD, session lifecycle, error responses, pagination
- [ ] Mock server (`prism` or similar) validates requests against OpenAPI spec
- [ ] Each SDK has a thin test harness that runs the contract scenarios
- [ ] CI pipeline: `make sdk-contract-test` runs all SDKs against mock server
- [ ] Drift detection: if an SDK produces a request that doesn't match the spec, the test fails
- [ ] At least 30 contract test cases covering happy + unhappy paths

## Technical Details

**Approach:**

Use a two-layer testing strategy:

1. **Spec-level validation** (Prism mock server): Validates that SDK requests conform to the OpenAPI spec and that SDKs correctly parse mock responses.

2. **Behavioral contracts** (Hurl files): Define the expected HTTP interactions for key workflows. Each SDK must produce equivalent HTTP traffic.

**Architecture:**

```
sdks/tests/
├── contract/
│   ├── auth.hurl                 # Login, register, API key flows
│   ├── workspaces.hurl           # CRUD, suspend, activate
│   ├── sessions.hurl             # Create, message, history
│   ├── errors.hurl               # 401, 403, 404, 409, 429 handling
│   └── pagination.hurl           # List with limit/offset
├── mock-server/
│   ├── docker-compose.yaml       # Prism mock server from OpenAPI spec
│   └── fixtures/                 # Custom response fixtures for edge cases
├── harness/
│   ├── run-typescript.sh         # Runs TS SDK against mock, captures HTTP
│   ├── run-python.sh             # Runs Python SDK against mock
│   ├── run-go.sh                 # Runs Go SDK against mock
│   └── run-java.sh              # Runs Java SDK against mock
└── Makefile                      # Orchestrates all contract tests
```

**Contract test categories:**

| Category | Cases | What's Validated |
|----------|-------|-----------------|
| Auth | 6 | Login request body, API key header format, token refresh on 401 |
| Workspaces | 8 | Create request body, list pagination params, status response parsing |
| Sessions | 6 | Create, send message body format, history response parsing |
| Errors | 6 | 401/403/404/409/429 mapped to correct exception types |
| Edge cases | 4 | Empty lists, null fields, large responses, unicode content |

**Mock server setup:**

```yaml
# sdks/tests/mock-server/docker-compose.yaml
services:
  mock-api:
    image: stoplight/prism:5
    command: mock /spec/openapi.yaml --host 0.0.0.0 --port 4010
    volumes:
      - ../../openapi.yaml:/spec/openapi.yaml
    ports:
      - "4010:4010"
```

**Per-SDK test harness pattern:**

Each SDK has a `contract_test` file that:
1. Points the SDK client at `http://localhost:4010` (Prism mock)
2. Executes the same workflow sequence as the Hurl files
3. Asserts the SDK correctly parses responses and throws correct error types

```typescript
// sdks/typescript/tests/contract.test.ts
describe('Contract: Auth', () => {
  it('sends correct login request body', async () => {
    const client = new LLMSafeSpace({ baseUrl: MOCK_URL, credentials: { email: 'test@test.com', password: 'pass' } });
    const result = await client.auth.login();
    expect(result.token).toBeDefined();
    expect(result.user.email).toBe('test@test.com');
  });
});
```

**CI integration:**

```makefile
# sdks/Makefile
sdk-contract-test:
    docker compose -f tests/mock-server/docker-compose.yaml up -d
    sleep 2
    $(MAKE) -C typescript contract-test
    $(MAKE) -C python contract-test
    $(MAKE) -C go contract-test
    $(MAKE) -C java contract-test
    docker compose -f tests/mock-server/docker-compose.yaml down
```

**Drift detection:**

Prism in strict mode rejects any request that doesn't match the OpenAPI spec. If an SDK sends an unexpected header, wrong content-type, or malformed body, the mock returns a validation error and the test fails.
