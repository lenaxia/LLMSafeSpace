# US-37.5: Activity Spinner in Sidebar

**Epic:** 37 - Session Activity & Unread State UX
**Priority:** Critical
**Depends on:** US-37.4 (SessionActivityProvider)

## User Story

As a user, I need to see a spinning indicator on sessions that are actively processing, regardless of which workspace I'm viewing.

## Acceptance Criteria

- [ ] Animated spinner (Loader2) on each busy session row in the sidebar
- [ ] Replaces the existing dead blue dot
- [ ] Spinner disappears when session goes idle
- [ ] Collapsed workspaces show a spinner if any child session is busy
- [ ] No layout shift — spinner occupies same space as the blue dot

## Assumptions

| # | Assumption | Validation |
|---|-----------|-----------|
| A1 | Existing blue dot (`h-1.5 w-1.5`) never renders because REST always returns `"idle"` | ✅ `Sidebar.tsx:696-698`; `database.go:710` hardcodes `"idle"` |
| A2 | `Loader2` icon already imported in Sidebar | ✅ `Sidebar.tsx:20` |

## Implementation

### Session Row (replaces dead blue dot)

```tsx
// SessionTreeRow — replace the blue dot:
{isBusy ? (
  <Loader2 className="h-3 w-3 animate-spin text-blue-500 flex-shrink-0" />
) : null}
```

Where `isBusy = useIsSessionBusy(s.id)`.

### Collapsed Workspace Indicator

```tsx
// WorkspaceGroup — only when collapsed and has busy sessions:
{!expanded && busyCount > 0 && (
  <Loader2 className="h-3 w-3 animate-spin text-blue-500 flex-shrink-0" />
)}
```

Where `busyCount = useWorkspaceBusyCount(workspace.id)`.

### Files to Modify

| File | Change |
|------|--------|
| `frontend/src/components/layout/Sidebar.tsx` | Replace blue dot with spinner in `SessionTreeRow`; add collapsed workspace spinner in `WorkspaceGroup` |
