# US-37.2: Merge Active Session Status into Session List REST

**Epic:** 37 - Session Activity & Unread State UX
**Priority:** Critical
**Depends on:** None

## User Story

As a user loading the page, I need the session list to show which sessions are actively processing, so that the sidebar is accurate immediately on load without waiting for SSE events.

## Acceptance Criteria

- [ ] `GET /workspaces/:id/sessions` returns `status: "active"` for sessions in the in-memory `activeSess` map
- [ ] Sessions not in the active map return `status: "idle"` (default)
- [ ] If `proxyHandler` is nil, all sessions have `status: "idle"` (no crash)
- [ ] No new method needed — reuse existing `GetActiveSessions`

## Implementation

In `router.go`, after `sessions, err := wsSvc.ListWorkspaceSessions(...)`:

```go
if proxyHandler != nil {
    activeIDs := proxyHandler.GetActiveSessionIDs(workspaceID)
    activeSet := make(map[string]struct{}, len(activeIDs))
    for _, id := range activeIDs {
        activeSet[id] = struct{}{}
    }
    for i := range sessions {
        if _, ok := activeSet[sessions[i].ID]; ok {
            sessions[i].Status = "active"
        }
    }
}
```

**Note:** `GetActiveSessions` already returns `[]string` of active session IDs. The router just builds a set and stamps the status. This is O(n) where n ≤ 10 sessions per workspace (stated constraint).
