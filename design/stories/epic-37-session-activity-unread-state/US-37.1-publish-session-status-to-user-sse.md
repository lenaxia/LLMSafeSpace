# US-37.1: Publish session.status to User-Scoped SSE Stream

**Epic:** 37 - Session Activity & Unread State UX
**Priority:** Critical
**Depends on:** None

## User Story

As a user viewing workspace A, I need to see in the sidebar when a session in workspace B becomes active or idle, so that I have awareness of activity across all my workspaces without switching.

## Acceptance Criteria

- [ ] When a session transitions to busy in any workspace, a `session.status` event with `status: "busy"` is delivered to the user-scoped SSE stream
- [ ] When a session transitions to idle in any workspace, a `session.status` event with `status: "idle"` is delivered to the user-scoped SSE stream
- [ ] The event includes `workspace_id`, `session_id`, and `status` fields
- [ ] The workspace-scoped SSE stream continues to receive events unchanged (no regression)
- [ ] If `WorkspaceOwner` returns `""`, the event is silently skipped (no error, no log spam)
- [ ] If `userBroker` is nil, no panic

## Assumptions

| # | Assumption | Validation |
|---|-----------|-----------|
| A1 | `ProxyHandler` has both `broker` and `userBroker` fields | ✅ `proxy.go:79-80` |
| A2 | `userBroker.WorkspaceOwner(workspaceID)` returns userID or `""` | ✅ `event_broker_user.go:284-289` |
| A3 | `RecordWorkspaceOwner` is called on every workspace phase change | ✅ `proxy.go:901-902` |
| A4 | User-scoped replay buffer (128 events) is sufficient — session status changes are low-frequency | ✅ One event per prompt cycle, seconds to minutes apart |

## Implementation

After the existing `h.broker.Publish` call in both `onSessionIdle` and `onSessionActive`, add:

```go
if h.userBroker != nil {
    if userID := h.userBroker.WorkspaceOwner(workspaceID); userID != "" {
        h.userBroker.PublishToUser(userID, WorkspaceSSEEvent{
            Type:        "session.status",
            WorkspaceID: workspaceID,
            SessionID:   sessionID,
            Status:      "idle", // or "busy" in onSessionActive
        })
    }
}
```

**Key detail:** The workspace-scoped event does NOT include `workspace_id` (implicit — subscribers are per-workspace). The user-scoped event MUST include it so the frontend can route the event to the correct workspace.
