# US-37.6: Unread Pulsation in Sidebar

**Epic:** 37 - Session Activity & Unread State UX
**Priority:** High
**Depends on:** US-37.4 (SessionActivityProvider), US-37.3 (hasUnread from REST)

## User Story

As a user who stepped away while a session was processing, I need the session name and icon to pulsate so that I'm drawn back to sessions with unseen responses.

## Acceptance Criteria

- [ ] Session icon and title pulsate when `isSessionUnread` is true and session is not selected
- [ ] Pulsation stops immediately on navigate-to
- [ ] Pulsation is suppressed while session is busy (spinner takes priority)
- [ ] Subtle opacity pulse — not aggressive flashing
- [ ] Respects `prefers-reduced-motion`
- [ ] Persists across page refresh (driven by REST `hasUnread`)

## Assumptions

| # | Assumption | Validation |
|---|-----------|-----------|
| A1 | `SessionTreeRow` knows if its session is selected (`s.id === selectedSessionId`) | ✅ `Sidebar.tsx:661` |
| A2 | Tailwind CSS custom keyframes can be added to `index.css` | ✅ `index.css` already has custom styles |

## Implementation

### CSS Animation (`styles/index.css`)

```css
@keyframes unread-pulse {
  0%, 100% { opacity: 1; }
  50% { opacity: 0.55; }
}

.animate-unread-pulse {
  animation: unread-pulse 2s ease-in-out infinite;
}

@media (prefers-reduced-motion: reduce) {
  .animate-unread-pulse {
    animation: none;
  }
}
```

### Session Row

Apply pulsation to icon and title only (not timestamp, kebab, or spinner):

```tsx
const showPulse = isUnread && !isSelected && !isBusy;

<MessageSquare className={cn("h-3.5 w-3.5 flex-shrink-0", showPulse && "animate-unread-pulse")} />
<span className={cn("flex-1 truncate", showPulse && "animate-unread-pulse")}>{title}</span>
```
