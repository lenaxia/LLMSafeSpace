# US-37.4: SessionActivityProvider (React Context)

**Epic:** 37 - Session Activity & Unread State UX
**Priority:** Critical
**Depends on:** US-37.1 (SSE delivers session.status), US-37.2 (REST returns active status)

## User Story

As the sidebar component, I need a centralized source of truth for session busy state across all workspaces, updated in real-time by SSE and initialized from REST data.

## Acceptance Criteria

- [ ] A `SessionActivityProvider` React Context provides busy/unread state to all descendants
- [ ] Initializes from REST session list query data (correct on first load)
- [ ] Updates in real-time from user-scoped SSE `session.status` events
- [ ] Clears busy state for a workspace when a `workspace.phase` event arrives with a non-Active phase
- [ ] Tracks `pendingUnread` — sessions that went idle while user wasn't viewing them
- [ ] Clears `pendingUnread` on navigate-to
- [ ] Respects `hasUnread` from REST data
- [ ] Does NOT create orphan query cache entries for collapsed workspaces
- [ ] Mounts in `AppShell` (inside the router), not `App` (outside)

## Implementation

### `useUserEventStream` Change

Add an optional `onEvent` callback:

```typescript
export function useUserEventStream(options?: { onEvent?: (event: unknown) => void }) {
  // ... existing code ...
  const onEventRef = useRef(options?.onEvent);
  useEffect(() => { onEventRef.current = options?.onEvent; });

  // In the existing onEvent handler, after processing workspace.phase/resync:
  onEventRef.current?.(data);
}
```

This is a minimal change — just forwards every event to an optional callback. The provider uses this to receive `session.status` events.

### Provider State

```typescript
interface SessionActivityState {
  busySessions: Map<string, string>;     // sessionId → workspaceId
  pendingUnread: Set<string>;            // sessionIds that went idle while not viewed
}
```

### Provider Logic

**From SSE `session.status` events:**
- `status: "busy"` → add to `busySessions`
- `status: "idle"` → remove from `busySessions`; if not currently viewed, add to `pendingUnread`

**From SSE `workspace.phase` events:**
- Non-Active phase (`Suspending`, `Suspended`, `Terminating`, `Terminated`, `Failed`) → clear all `busySessions` and `pendingUnread` entries for that workspace

**From REST session list queries:**
- On `["sessions", wsId]` query resolve: for each session with `status: "active"`, add to `busySessions`. For each with `hasUnread: true`, ensure in `pendingUnread`. For each with `hasUnread: false`, remove from `pendingUnread`.

**From navigation:**
- `clearPendingUnread(sessionId)` — called when user navigates to a session

**Guard: `queryClient.setQueryData`** — only update `["sessions", wsId]` cache if `queryClient.getQueryData(["sessions", wsId])` is not undefined. Prevents orphan entries for collapsed workspaces.

### Hook API

```typescript
function useIsSessionBusy(sessionId: string): boolean;
function useIsSessionUnread(sessionId: string): boolean;
function useWorkspaceBusyCount(workspaceId: string): number;
```

Each hook subscribes to the context and returns the requested value. At ~100 sessions max, re-rendering on any change is acceptable — no need for granular subscriptions.

### Mount Point

```tsx
// AppShell.tsx
export function AppShell() {
  return (
    <SessionActivityProvider>
      {/* existing AppShell content */}
    </SessionActivityProvider>
  );
}
```

`SessionActivityProvider` wraps the existing content, calls `useUserEventStream` internally with its `onEvent` callback, and uses `useParams` to read the current session.
