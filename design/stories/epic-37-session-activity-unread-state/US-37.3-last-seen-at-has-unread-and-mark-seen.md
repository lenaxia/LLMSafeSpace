# US-37.3: Add `last_seen_at`, Compute `hasUnread`, Mark-Seen Endpoint

**Epic:** 37 - Session Activity & Unread State UX
**Priority:** Critical
**Depends on:** None

## User Story

As a user returning to the app after being away, I need the system to remember which sessions I've already viewed, so it can accurately show which sessions have new responses.

## Acceptance Criteria

- [ ] `last_seen_at TIMESTAMPTZ` nullable column added to `session_index`
- [ ] `GET /workspaces/:id/sessions` includes `last_seen_at` and `has_unread` in response
- [ ] `has_unread` computed in SQL: true when `last_message_at IS NOT NULL AND (last_seen_at IS NULL OR last_message_at > last_seen_at)`
- [ ] `has_unread` is false when `last_seen_at IS NULL` (never visited = not unread — user may have seen responses when creating the session)
- [ ] `PUT /workspaces/:id/sessions/:sessionId/seen` sets `last_seen_at = NOW()`, returns 204
- [ ] Endpoint requires auth and verifies workspace ownership
- [ ] Idempotent — multiple calls are safe
- [ ] Go and TypeScript types updated
- [ ] Contract test fixtures and test updated

## Implementation

### Migration

```sql
-- 000019_session_last_seen.up.sql
ALTER TABLE session_index ADD COLUMN IF NOT EXISTS last_seen_at TIMESTAMPTZ;

-- 000019_session_last_seen.down.sql
ALTER TABLE session_index DROP COLUMN IF EXISTS last_seen_at;
```

### Go Type — `SessionListItem` (`pkg/types/types.go`)

Add two fields:
```go
LastSeenAt *time.Time `json:"lastSeenAt,omitempty"`
HasUnread  bool       `json:"hasUnread"`
```

### `ListSessionIndex` SQL (`database.go:681`)

Replace the current query:
```sql
SELECT
    session_id, title, parent_session_id, last_message_at, message_count,
    last_seen_at,
    (last_message_at IS NOT NULL
     AND (last_seen_at IS NULL OR last_message_at > last_seen_at)) AS has_unread
FROM session_index
WHERE workspace_id = $1
ORDER BY last_message_at DESC NULLS LAST
LIMIT 100
```

Update the scan to read `last_seen_at` (nullable) and `has_unread` (bool).

### `UpdateSessionLastSeen` (`database.go`)

New method:
```sql
INSERT INTO session_index (workspace_id, session_id, last_seen_at, message_count, updated_at)
VALUES ($1, $2, NOW(), 0, NOW())
ON CONFLICT (workspace_id, session_id) DO UPDATE SET
  last_seen_at = NOW(),
  updated_at = NOW()
```

The INSERT case handles brand-new sessions with no row yet. `message_count = 0` is corrected on the next `RecordMessage`.

### Service Layer

Add `UpdateLastSeen(ctx, workspaceID, sessionID) error` to `SessionIndexService` interface and `sessionindex.Service`.

Add `MarkSessionSeen(ctx, userID, workspaceID, sessionID) error` to `workspace.Service` — calls `verifyOwner` then `sessionIndex.UpdateLastSeen`.

### Router Endpoint

```go
rg.PUT("/:id/sessions/:sessionId/seen", func(c *gin.Context) {
    userID := authSvc.GetUserID(c)
    if userID == "" {
        c.JSON(http.StatusUnauthorized, gin.H{"error": "authentication required"})
        return
    }
    if err := wsSvc.MarkSessionSeen(c.Request.Context(), userID, c.Param("id"), c.Param("sessionId")); err != nil {
        respondWithError(c, err)
        return
    }
    c.Status(http.StatusNoContent)
})
```
