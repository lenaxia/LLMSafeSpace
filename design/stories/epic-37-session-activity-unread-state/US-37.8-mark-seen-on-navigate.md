# US-37.8: Mark-Seen on Navigate

**Epic:** 37 - Session Activity & Unread State UX
**Priority:** High
**Depends on:** US-37.3 (mark-seen endpoint), US-37.7 (divider needs lastSeenAt)

## User Story

As a user navigating between sessions, I need the system to track when I've viewed a session so that the unread indicator and divider are accurate.

## Acceptance Criteria

- [ ] Navigate-to calls `markSessionSeen` immediately (not debounced)
- [ ] Navigate-away calls `markSessionSeen` (1s debounced)
- [ ] Both calls are fire-and-forget — no loading state, no blocking
- [ ] Failed calls are silently ignored
- [ ] No call when workspace is not Active
- [ ] After mark-seen, session list query invalidated so `hasUnread` refreshes

## Assumptions

| # | Assumption | Validation |
|---|-----------|-----------|
| A1 | `workspacesApi.markSessionSeen` added in US-37.3 | ✅ Part of that story |
| A2 | `useEffect` cleanup fires on `sessionId` change | ✅ Standard React |

## Implementation

### Navigate-To (Immediate)

```typescript
useEffect(() => {
  if (!workspaceId || !sessionId || !isReady) return;

  clearPendingUnread(sessionId);

  workspacesApi.markSessionSeen(workspaceId, sessionId).catch(() => {});

  queryClient.invalidateQueries({ queryKey: ["sessions", workspaceId] });
}, [sessionId, workspaceId, isReady]);
```

### Navigate-Away (Debounced)

```typescript
const prevSessionRef = useRef<{ wsId: string; sId: string } | null>(null);
const debounceRef = useRef<ReturnType<typeof setTimeout>>();

useEffect(() => {
  if (prevSessionRef.current) {
    const { wsId, sId } = prevSessionRef.current;
    if (debounceRef.current) clearTimeout(debounceRef.current);
    debounceRef.current = setTimeout(() => {
      workspacesApi.markSessionSeen(wsId, sId).catch(() => {});
    }, 1000);
  }

  prevSessionRef.current = workspaceId && sessionId ? { wsId: workspaceId, sId: sessionId } : null;

  return () => {
    if (debounceRef.current) clearTimeout(debounceRef.current);
  };
}, [sessionId, workspaceId]);
```

**Why double mark-seen:** Navigate-to clears unread immediately. Navigate-away captures messages that arrived while the user was viewing (e.g., a queued message completed). The second call sets `last_seen_at` to a later NOW(), ensuring the divider is accurate on the next visit.
