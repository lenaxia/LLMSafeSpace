# US-22.4: Document `/v1/statusz` as Expensive

**Epic:** 22 — agentd Health Endpoint Redesign
**Priority:** Low (documentation-only; no behavior change)
**Depends on:** none
**Blocks:** none
**Layer:** workspace-agentd

## User Story

As a future maintainer or new consumer of the workspace-agentd HTTP API, I need an unambiguous code-level marker that `/v1/statusz` is the expensive deep-introspection endpoint (intended for slow-cadence polling and human inspection), so I don't accidentally wire it into a hot-path probe and recreate the worklog 0096 starvation incident.

## Acceptance Criteria

- [ ] `cmd/workspace-agentd/main.go` `/v1/statusz` handler (line 406-441 today) gains a doc comment explaining:
  - The endpoint makes multiple opencode calls under a mutex (`cache.mu`).
  - Latency is unbounded — depends on opencode's response time. Callers must use a generous timeout (≥ 30s).
  - It is NOT intended for kubelet liveness or readiness probes; those use `/v1/healthz` and `/v1/readyz` respectively.
  - It IS intended for: controller deep-status enrichment (sessions list, disk usage), API service status responses, ad-hoc operator inspection.
- [ ] No behavioral change. Tests do not need to change.
- [ ] (Optional) Add a runtime warning log line when the handler observes its own latency exceeding 5 seconds, so post-deploy diagnostics surface SSE-load contention.

## Technical Details

### Doc comment

```go
// /v1/statusz returns a deep snapshot of the workspace's state including
// agent connectivity, configured providers, sessions list, and disk usage.
//
// COST: This endpoint makes up to 4 HTTP calls to opencode (cachedState
// fetches providers, configured count, and sessions; IsHealthy is also
// called). On cache miss, all 4 are sequential under cache.mu. Total
// latency is dominated by opencode's response time, which can exceed
// 30 seconds under SSE load (worklog 0096). Callers MUST use a generous
// HTTP timeout (≥ 30s recommended).
//
// USE WHEN: you need a full snapshot of workspace state for status
// enrichment, dashboarding, or operator inspection on a slow cadence
// (≥ 60 seconds between calls per workspace).
//
// DO NOT USE FOR:
//   - Kubelet liveness probes (use /v1/healthz, US-22.1)
//   - Kubelet readiness probes (use /v1/readyz, US-22.3)
//   - Any health/probe-style polling at < 30s cadence
//
// Misusing this endpoint as a fast probe was the root cause of the
// 2026-05-31 incident on workspaces 6d36952e-…, c98963e7-…, 8e80afc4-…
// (worklogs 0099, 0100, 0102) — controller's checkAgentHealth probe at
// 5s timeout × 15s rate-limit produced 3-strike terminal failures
// when opencode was merely under SSE load.
```
