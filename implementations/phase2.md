# Phase 2: Minimal Controller and API Server

## Overview
This phase focuses on implementing a minimal version of both the controller and API server components. The goal is to have a working end-to-end flow where the API server can create a Sandbox resource and the controller can reconcile it to create a pod.

## Steps

### 1. Controller: Set up the controller project structure using `kubebuilder`

**Files for Context:**
- README.md (controller section)

**Core Task:**
Set up the controller project structure using `kubebuilder` to generate the boilerplate code for a Kubernetes operator.

**Requirements:**
- Use `kubebuilder` to generate the controller project structure
- Configure the controller to watch the `Sandbox` and `RuntimeEnvironment` resources
- Set up the manager to run the controller

**Implementation Details:**
```bash
# Install kubebuilder
curl -L -o kubebuilder https://go.kubebuilder.io/dl/latest/$(go env GOOS)/$(go env GOARCH)
chmod +x kubebuilder
sudo mv kubebuilder /usr/local/bin/

# Initialize the controller project
cd controller
kubebuilder init --domain llmsafespace.dev --repo github.com/lenaxia/llmsafespace/controller

# Create API for Sandbox
kubebuilder create api --group llmsafespace --version v1 --kind Sandbox --resource --controller

# Create API for RuntimeEnvironment
kubebuilder create api --group llmsafespace --version v1 --kind RuntimeEnvironment --resource --controller
```

     - Test that the controller handles pod deletion correctly
     - Test that the controller handles errors correctly

   - **Purpose:** Verify that the runtime environment controller correctly validates runtime images

5. **Kubernetes Client Tests**
